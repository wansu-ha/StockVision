## 프론트엔드

- **주목 종목 섹션**: 스크리닝 결과를 대시보드 하단에 표시 (거래량 급증, 조건 근접 종목 등)
- **전략 템플릿 → 종목 일괄 적용**: 하나의 규칙 세트를 여러 종목에 동시 적용
- **커뮤니티 전략 공유**: 규칙 복사본 공유/다운로드
- **백테스팅 UI**: 과거 데이터 기반 규칙 시뮬레이션
- **다크 모드**
- **모바일 반응형**

## 규칙 모델

- **비율/금액 기반 수량**: `qty_type: "RATIO"` (예산 비율), `"AMOUNT"` (금액 기반)
- **쿨다운 트리거**: N분 후 재실행 허용
- **무제한 트리거**: 조건 충족 시 매번 실행
- **조건 그룹 중첩**: `(A AND B) OR (C AND D)` 형태
- **사용자 정의 수식**: `custom_formula` (안전한 파서 필요)
- **규칙 체이닝**: 규칙 A 체결 → 규칙 B 활성화
- **시간 조건**: 특정 시간대만 활성화
- **손절/익절**: 수익률 기반 매도 조건

## 클라우드 서버

- **전 종목 스크리닝**: 키움 API 한계(초당 5건) → KRX 공공데이터 / FinanceDataReader 등 별도 데이터 소스 필요
  - KOSPI+KOSDAQ ≈ 2,500+ 종목, 종목별 REST 조회는 8분+ → 벌크 수집 필수
  - 현실적 조합: KRX 일괄 (1~5분 주기) + 키움 WS 200종목 (조건 근접 종목만)
  - 과거 데이터 축적 → 백테스팅용 DB 필요 (별도 연구)
- **AI 분석 (Claude API)**: 뉴스 감성 점수, 시장 요약 → 대시보드 컨텍스트 제공
- **유니버스 설정**: KOSPI 200, KOSDAQ 150 등 스크리닝 대상 그룹

## 로컬 서버

- **내장 프론트엔드**: 로컬 서버가 동일 SPA를 정적 서빙 → 클라우드/CDN 없이도 UI 접근 가능
  - 웹 접속 시 리다이렉트 불가 (도메인 다름) → localhost:8765에서 직접 서빙
  - 프론트엔드 코드 공유 (빌드 산출물 동일)
- **체결 보고 (서비스 통계)**: 종목/금액 미포함, 성공/실패 건수만 클라우드에 보고 (opt-in)
- **opt-in 수익 동기화**: 사용자가 명시적 동의 시 수익률을 클라우드에 전송 (커뮤니티 통계용)
- **포트폴리오 리밸런싱**: 보유 종목 전체를 대상으로 목표 비중 설정 → 주기적/임계치 기반 자동 리밸런싱
  - 잔고·보유 종목 데이터 필요 (키움 API 조회)
  - 종목 단위 규칙과 별개의 포트폴리오 레벨 전략
  - 목표 비중 설정 UI, 리밸런싱 트리거 조건 (주기/편차%), 시뮬레이션
- **규칙 우선순위**: 높은 순위 규칙 먼저 주문
- **가격 검증 캐시**: 동일 종목 1초 내 재조회 불필요
