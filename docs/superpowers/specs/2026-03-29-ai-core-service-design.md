# AI 코어 서비스 — Brainstorming 설계 문서

> 작성일: 2026-03-29 | 상태: 설계 완료 (brainstorming)

## 1. 개요

사용자가 자연어로 전략을 설명하면 AI가 v2 DSL 코드를 생성/수정/설명하고, 기본 비서가 초보자를 안내하는 기능.

### 스코프

| 포함 (1차) | 제외 (나중에) |
|-----------|-------------|
| 전략 빌더 (Sonnet, 운영자 키) | 프리미엄 비서 (사용자 키, 개인화, 사이트 조작) |
| 종목 분석기 강화 (기존 Sonnet) | 대화 요약 자동 생성 |
| 기본 비서 (Haiku, 무료) | thinking 자동 판단 |
| 크레딧제 + BYO Key | |
| 코드 인텔리전스 | |
| SSE 스트리밍 + thinking 토글 | |
| 대화 히스토리 DB 저장 | |
| 전략 버전 스냅샷 | |

## 2. AI 4계층 아키텍처

```
┌─ 프리미엄 비서 (나중에, 사용자 키) ──────────────┐
│  개인화, 사이트 전역 에이전트                      │
│  tool 호출로 아래 계층 사용                        │
├──────────────────────────────────────────────────┤
│  ┌────────────┬────────────┬────────────────┐    │
│  │ 종목 분석기  │ 전략 빌더   │ 기본 비서       │    │
│  │ Sonnet     │ Sonnet     │ Haiku          │    │
│  │ 운영자 키   │ 운영자 키   │ 운영자 키       │    │
│  │ 크레딧 차감  │ 크레딧 차감  │ 무료 (거의)    │    │
│  └────────────┴────────────┴────────────────┘    │
├──────────────────────────────────────────────────┤
│  비서 없이도 모든 기능 직접 사용 가능                │
│  (버튼, 프리셋, DSL 직접 편집, 백테스트 등)         │
└──────────────────────────────────────────────────┘
```

원칙:
- 비서 없이 모든 기능 사용 가능
- 분석기/빌더는 플랫폼 기능 (운영자 키, 크레딧제)
- 기본 비서는 무료 (Haiku 비용 무시 가능, ~$0.001/턴)
- 프리미엄 비서는 편의 레이어 (사용자 키, 선택적)

## 3. 전략 빌더 — 상세 설계

### 3.1 아키텍처 (접근법 C: 하이브리드)

```
프론트엔드
  ├─ 대화 UI (전체 히스토리 표시)
  └─ POST /api/v1/ai/chat {conversation_id, message, current_dsl}
         ↓ SSE 스트리밍
클라우드 서버
  ├─ 크레딧 차감 (토큰 기반)
  ├─ API 키 분기 (운영자 / BYO)
  ├─ 대화 히스토리 DB 조회 (전체 보관)
  ├─ LLM 전송용 윈도우 구성
  │   ├─ 시스템 프롬프트 (~3K 토큰, 서버 관리)
  │   ├─ 현재 DSL (항상 최신)
  │   └─ 최근 N턴 (토큰 초과 시 오래된 턴 제거)
  ├─ Claude API (streaming, thinking 옵션)
  ├─ DSL 추출 → parse_v2 검증
  │   └─ 실패 시 에러 피드백 포함 재시도 (최대 3회, 재시도 과금 안 함)
  ├─ 대화 히스토리 DB 저장
  └─ SSE 응답: 단계 표시 + 스트리밍 텍스트 + 검증된 DSL + 크레딧 잔량
```

핵심 결정:
- **시스템 프롬프트 서버 관리**: 튜닝 → 서버 배포만으로 반영 (프론트 빌드 불필요)
- **대화 저장은 전체, LLM 전송은 윈도우**: DSL이 이전 대화의 결과물이므로 "코드가 요약"
- **검증은 서버**: parse_v2 로직 중복 없음

### 3.2 UI 레이아웃

**데스크톱 (≥1024px)**
```
┌──────────────────┬───────────────────────────┐
│   AI 대화 패널    │   DSL 편집기               │
│                  │   (하이라이팅 + 자동완성)     │
│  [단계 표시]      │                           │
│  [thinking 접기]  │   실시간 반영 (생성 시)      │
│  [응답 스트리밍]   │   디프 하이라이트 (수정 시)   │
│                  │                           │
│  크레딧 바        │   버전 히스토리             │
├──────────────────┴───────────────────────────┤
│  파라미터 슬라이더                              │
└──────────────────────────────────────────────┘
```

**모바일 (<1024px)**
```
┌────────────────────┐
│  [대화]  [편집기]   │  ← 탭 전환
│                    │
│   전체 화면 표시    │
│                    │
└────────────────────┘
```

**종목 페이지**: 동일 컴포넌트를 모달로 표시

### 3.3 대화 패널 UX

```
┌──────────────────────────────┐
│ 🔍 전략 분석 중...            │  SSE 단계 표시
│ ✍️ DSL 생성 중...             │
│ ✅ 문법 검증 완료              │
├──────────────────────────────┤
│ ▶ 사고 과정 보기              │  접힌 상태 (기본)
│                              │
│ 골든크로스 + 3% 손절 전략을    │  스트리밍 응답
│ 만들었습니다:                 │
│                              │
│ ┌──────────────────────┐     │
│ │ RSI(14) < 30 AND     │     │  DSL 코드 블록
│ │ 골든크로스 → 매수 100% │     │
│ └──────────────────────┘     │
├──────────────────────────────┤
│ AI 크레딧  ████████░░  78%   │
│ 약 12회 대화 가능 (예상)      │
│ 매일 자정 초기화              │
└──────────────────────────────┘
```

대화 범위 (코파일럿):
- 생성: "골든크로스 매수 전략 만들어줘"
- 수정: "트레일링 추가해줘", "손절 3%로 바꿔줘"
- 설명: "이 전략 설명해줘", "리스크가 뭐야"

### 3.4 편집기 변경 반영

| 상황 | 동작 |
|------|------|
| 새 전략 생성 | 스트리밍과 함께 편집기에 실시간 타이핑 |
| 기존 전략 수정 | 디프 하이라이트 (추가=초록, 삭제=빨강) + [적용] 버튼 |

### 3.5 코드 인텔리전스

- **구문 하이라이팅**: 키워드(매수/매도/→/BETWEEN), 상수, 숫자, 함수 색 구분
- **인라인 에러**: parse_v2 실패 위치 빨간 밑줄
- **자동완성**: builtins.py 필드/함수 목록 기반, prefix 매칭 드롭다운

### 3.6 전략 버전 스냅샷

```
전략 히스토리
├─ v3  "트레일링 추가" (방금)        [현재]
├─ v2  "손절 3%로 변경" (10분 전)    [되돌리기]
├─ v1  "AI 초기 생성" (15분 전)      [되돌리기]
```

- AI 수정 시 자동 스냅샷 + 변경 요약
- 두 버전 간 디프 비교 가능
- 되돌리기로 이전 버전 복원
- 정리 정책: 1차에서는 없음, 나중에 데이터 보고 판단

## 4. 기본 비서

| 항목 | 내용 |
|------|------|
| 모델 | Haiku |
| API 키 | 운영자 (비용 ~$0.001/턴, 무시 가능) |
| 개인화 | 없음 |
| 대화 저장 | 안 함 |
| 역할 | 범용 안내, FAQ, 기능 설명, 온보딩 도움 |

기본 비서가 **못 하는 것**: 개인화 추천, 사이트 조작, 매매 분석

## 5. 비용 모델

### 5.1 크레딧제

```
내부: 일일 토큰 한도 (예: 50K input+output)
표시: ████████░░ 크레딧 78% 남음 / 약 12회 대화 가능
초기화: 매일 자정
```

| 작업 | 대략 소모 |
|------|----------|
| 전략 생성 1회 | ~3K 토큰 (~6%) |
| 수정 요청 1턴 | ~1.5K 토큰 (~3%) |
| 설명/분석 1턴 | ~2K 토큰 (~4%) |

### 5.2 과금 원칙

- 검증/백테스트: 무료 (parse_v2, 로컬 엔진)
- 재시도 (AI 생성 실패): 과금 안 함 (AI 오류는 우리 책임)
- 기본 비서 (Haiku): 무료
- BYO API Key: 무제한, 크레딧 차감 안 함

### 5.3 Extended Thinking

- 기본 OFF (빠름, 저렴)
- 설정에서 토글 ON (오래 생각, 정확도↑, 토큰 추가)
- thinking 내용은 접기/펴기로 확인 가능

---

## 6. 정확도 전략

- 시스템 프롬프트 ~3K 토큰: DSL 문법 요약 + 필드/함수 목록 + few-shot 프리셋 3~4개
- builtins.py 전체 목록 주입 (환각 방지)
- parse_v2 에러 메시지로 재시도 (자가 수정)
- 잘못된 패턴 블랙리스트
- **출시 후 지속 튜닝** — 실패 패턴 수집 → 프롬프트 개선

## 7. 데이터 모델

```sql
-- 대화 히스토리 (전체 보관, 멀티 디바이스)
ai_conversations (
    id, user_id, strategy_id NULLABLE,
    messages JSON,  -- 전체 대화
    current_dsl,
    created_at, updated_at
)

-- 크레딧 추적
ai_usage (
    id, user_id, date,
    tokens_used, tokens_limit,
    updated_at
)

-- 전략 버전 스냅샷
strategy_versions (
    id, rule_id, version,
    script TEXT,
    message VARCHAR,  -- 변경 요약
    created_at
)
```

## 8. API 엔드포인트

| 메서드 | 경로 | 설명 |
|--------|------|------|
| POST | /api/v1/ai/chat | AI 대화 (SSE 스트리밍) |

| GET | /api/v1/rules/{id}/versions | 전략 버전 목록 |
| POST | /api/v1/rules/{id}/versions/{v}/restore | 버전 되돌리기 |

## 9. 로드맵 (AI 전체)

```
1차 (이번):
  ├─ 전략 빌더 (Sonnet, SSE, 코파일럿)
  ├─ 기본 비서 (Haiku, 무료)
  ├─ 크레딧제 + BYO Key
  ├─ 코드 인텔리전스
  ├─ 대화 히스토리 DB
  └─ 전략 버전 스냅샷

2차:
  ├─ 프리미엄 비서 L1 (사용자 키, 대화 + API tool 호출)
  └─ 대화 요약 자동 생성

3차:
  ├─ 프리미엄 비서 L2+ (사이트 조작, 네비게이션)
  └─ thinking 자동 판단
```
