## 제1조 개요

StockVision(이하 "회사")은 사용자의 개인정보를 보호하기 위해 **개인정보보호법**을 준수하며, 이 방침에서 개인정보 수집, 이용, 보관, 삭제에 대한 원칙을 명시합니다.

**핵심 원칙**: 개인정보 최소화 및 로컬 저장
- 회사는 서비스 제공에 필수적인 최소한의 정보만 수집합니다.
- 금융 정보(API Key, 거래 기록, 잔고)는 절대로 수집하지 않습니다.

## 제2조 수집하는 개인정보

### 2.1 필수 수집 항목

| 항목 | 유형 | 수집 목적 | 보관 위치 | 보관 기간 |
|------|------|---------|---------|---------|
| 이메일 주소 | 개인정보 | 회원 식별, 계정 접근, 비밀번호 복구 | 클라우드 DB | 회원 탈퇴 후 30일 |
| 비밀번호(해시) | 개인정보 | 계정 보안 인증 | 클라우드 DB | 회원 탈퇴 후 30일 |

### 2.2 선택 수집 항목

| 항목 | 유형 | 수집 목적 | 보관 위치 | 보관 기간 |
|------|------|---------|---------|---------|
| 닉네임 | 개인정보 | 커뮤니티 표시(v2) | 클라우드 DB | 회원 탈퇴 후 30일 |
| 전략 규칙(JSON) | 개인정보 | 규칙 저장 및 동기화 | 클라우드 DB + 로컬 캐시 | 회원 탈퇴 후 30일 |

### 2.3 자동 수집 항목 (운영 통계)

회사는 서비스 운영 목적으로 다음을 자동 수집합니다:

| 항목 | 설명 | 개인정보 여부 | 용도 |
|------|------|------------|-----|
| 익명 UUID | 사용자를 구별하기 위한 임의의 식별자 | 아님 (익명화됨) | 운영 통계 |
| 앱 버전 | StockVision 버전 번호 | 아님 | 호환성 분석 |
| OS | Windows 버전 정보 | 아님 | 플랫폼 통계 |
| 키움 연결 상태 | "연결됨", "끊김" 등 상태값 | 아님 | 서비스 안정성 모니터링 |
| 타임스탬프 | 정보 수집 시간 | 아님 | 타임존 독립 기록 |

**개인정보 미해당 사유**:
- 익명 UUID만으로는 실명, 주소, 연락처 등 개인을 식별할 수 없음
- 개인정보보호법 제2조 정의상 "개인정보"에 해당하지 않음
- 따라서 이 데이터의 수집/이용에는 동의 요청 불필요

## 제3조 수집하지 않는 개인정보 (구조적 미수집)

### 3.1 금융 정보 미수집

회사는 다음 금융 정보를 **절대로 수집하지 않습니다**:

| 항목 | 이유 | 저장 위치 |
|------|------|---------|
| 증권사 API Key | 제3자 위임 금지(증권사 약관) | 사용자 PC (Credential Manager) |
| 증권사 API Secret | 사용자 소유 개인키 | 사용자 PC (Credential Manager) |
| 증권사 계좌번호 | 금융정보 보호 | 사용자 PC |
| 현금 잔고 | 금융정보 | 사용자 PC (로컬 로그) |
| 보유 종목 및 수량 | 금융정보 | 사용자 PC (로컬 로그) |
| 체결 거래(매수/매도 내역) | 개인 거래 기록 | 사용자 PC (로컬 SQLite) |
| 수익률, 손익액 | 개인 재무정보 | 사용자 PC (로컬 SQLite) |
| 거래 시간, 거래량 | 개인 거래 기록 | 사용자 PC (로컬 SQLite) |

### 3.2 구조적 미수집 설계

이 정보들을 수집하지 않기 위해 아키텍처 수준에서 설계했습니다:

1. **로컬 서버만 증권사 API 호출**
   - 사용자 API Key는 사용자 PC에만 저장
   - 클라우드 서버는 API Key 접근 불가

2. **금융 정보는 로컬 SQLite에만 저장**
   - 거래 기록(logs.db), 거래 결과는 사용자 PC의 로컬 데이터베이스에만 저장
   - 클라우드 서버로 전송되지 않음

3. **클라우드는 시세와 규칙만 저장**
   - 시세: 분석 목적의 공개 데이터 (개인정보 아님)
   - 규칙: 사용자가 정의한 논리(매매 기준), 개인정보 아님

## 제4조 개인정보 보안 조치

### 4.1 클라우드 서버 (규칙, 계정 정보)

#### 전송 보안
- **HTTPS/TLS 1.3**: 모든 통신은 암호화 (AES-256 대칭키)
- IP 화이트리스트: 로컬 서버와 프론트엔드 호스팅 IP만 허용

#### 저장 보안
- **비밀번호**: Argon2id 단방향 해싱 (솔트 포함, 반복 횟수 3회)
  - 개인정보보호법 안전성확보조치기준 §8 준수
- **민감 정보** (설정, 규칙): AES-256-GCM 암호화 저장
  - 개인정보보호법 안전성확보조치기준 §10 준수
- 암호화 키: AWS KMS 또는 HashiCorp Vault에서 관리

#### 접근 제어
- 데이터베이스 접근은 IAM 역할 기반
- API 서버만 데이터베이스 접근 권한 보유
- 관리자 직접 접근은 로깅 및 감시

#### 감시 및 로깅
- **접속 기록 보관**: 최소 6개월 (안전성확보조치기준 §9)
- 접속 기록: 사용자 ID, 접근 시간, 접근 IP, 작업 내용
- 비정상 접근 자동 탐지 및 차단
- 정기적 보안 감사 및 취약점 스캔

### 4.2 로컬 서버 (금융 정보는 수집 안 하므로 해당 없음)

#### 저장 보안
- **로컬 SQLite 데이터베이스**:
  - NTFS 파일 권한: 소유자(사용자) 읽기/쓰기만, 다른 사용자 접근 차단
  - BitLocker 또는 VeraCrypt: 디스크 전체 암호화 (권장)

#### 키 관리
- **Windows Credential Manager** (OS DPAPI 암호화):
  - 키움 API Key/Secret 저장
  - DPAPI: 로그인한 사용자만 복호화 가능
  - 다른 사용자, 원격 접근자는 접근 불가

#### 네트워크 보안
- 로컬 서버: localhost에만 바인드, 외부 접근 불가
- 증권사 API 통신: 사용자의 ISP 인터넷 사용 (VPN 등은 선택)

### 4.3 백업 및 복구

#### 클라우드 데이터
- 매일 자동 백업 (AWS S3)
- 백업도 암호화 상태로 저장
- 재해 복구 시나리오: RTO 1시간, RPO 1시간

#### 로컬 데이터
- 사용자가 직접 관리 (우리 책임 없음)
- .exe 삭제 시: `%APPDATA%\StockVision\` 에 데이터 남음 (사용자가 수동 삭제)
- 복구: 사용자가 백업본에서 복원

## 제5조 개인정보 이용

### 5.1 이용 목적

수집한 개인정보는 다음 목적으로만 이용됩니다:

| 정보 | 이용 목적 |
|------|---------|
| 이메일, 비밀번호 해시 | 회원 인증, 계정 관리, 비밀번호 복구 |
| 전략 규칙 | 규칙 저장, 기기 간 동기화, 백업 |
| 닉네임 | 커뮤니티 표시(향후 기능) |
| 운영 통계 | 서비스 안정성 모니터링, 호환성 분석, 사용 패턴 분석 |

### 5.2 제3자 제공

회사는 다음의 경우를 제외하고 개인정보를 제3자에게 제공하지 않습니다:

#### 법적 근거 제공
- **법원 명령**: 법원 출석 명령(소장)이 있을 경우
- **검찰 요청**: 수사 관련 정보 제공 요청
- **정부 부처 요청**: 세무, 금융감독 등 공식 조사

제3자 제공 시:
- 사용자에게 **사전 통지** (법원 비밀유지 명령 제외)
- 제공한 정보, 제공처, 제공 사유, 제공 일시 기록
- 제공 후 1개월 내 사용자에게 보고

#### 계약 기반 제공 금지
- 광고 회사, 마케팅 업체로 제공 불가
- 데이터 분석 회사로 제공 불가
- 커뮤니티 공유 데이터는 사용자의 **자발적 선택**이며, 공유 철회 시 삭제

### 5.3 커뮤니티 데이터 활용 (v2 이상)

향후 커뮤니티 기능이 추가될 때:

- 사용자가 자신의 전략/수익을 **공개로 공유**하면, 익명화된 통계에 포함될 수 있습니다.
- 공유 철회 시, 30일 내에 모든 데이터가 삭제됩니다.
- 이미 생성된 통계에서 개별 데이터는 제거되지만, 이전 통계 보고서는 유지됩니다.

## 제6조 개인정보 삭제

### 6.1 회원 탈퇴 시

사용자가 회원 탈퇴를 신청하면:

1. **즉시 삭제**:
   - 계정 활성 상태 → 비활성 상태 변경
   - 모든 API 토큰 무효화
   - 프론트엔드 및 로컬 서버 접근 불가

2. **지연 삭제 (30일 유예)**:
   - 이메일, 비밀번호 해시, 규칙, 닉네임 → 30일 후 삭제
   - 유예 기간 내 탈퇴 취소 가능
   - 30일 후 자동 완전 삭제

3. **바로 삭제 (불가역)**:
   - 접속 로그, 하트비트 기록
   - 커뮤니티 공유 데이터 (통계에 포함된 것은 제외)

### 6.2 로컬 데이터 삭제

회사는 로컬 데이터를 수집하지 않으므로, 삭제 책임도 없습니다:

- `%APPDATA%\StockVision\logs.db` — 사용자가 수동 삭제
- `%APPDATA%\StockVision\strategies.json` — 사용자가 수동 삭제
- Windows Credential Manager의 증권사 API Key — 사용자가 수동 삭제
- 앱 삭제 후 남은 로컬 파일은 사용자 책임

**삭제 안내**:
- 설정 페이지에서 "앱 데이터 폴더 열기" 버튼 제공
- 앱 삭제 매뉴얼: "남은 파일을 수동으로 삭제하려면..." 안내

### 6.3 특정 데이터 삭제 요청

사용자는 언제든 다음의 삭제를 요청할 수 있습니다:

- 닉네임
- 커뮤니티 공유 전략

요청 시:
- 요청일로부터 3일 내 삭제
- 이미 제공된 통계/분석 데이터는 제거되지 않음 (개인정보 아님)

## 제8조 사용자 권리

### 8.1 정보 열람 권리
사용자는 언제든 자신의 개인정보를 요청하여 열람할 수 있습니다:

- 요청 방법: 설정 페이지 → "개인정보 다운로드" 클릭
- 응답 시간: 요청일로부터 5일 내
- 형식: JSON 또는 CSV 다운로드

### 8.2 정정 권리
부정확한 개인정보를 정정할 수 있습니다:

- 닉네임 변경: 설정 페이지에서 즉시 변경
- 이메일 변경: 새 이메일로 인증 후 변경
- 정정된 정보는 모든 시스템에 즉시 적용

### 8.3 삭제 권리
제6조의 방식에 따라 개인정보 삭제를 요청할 수 있습니다.

### 8.4 이의 제기 권리
개인정보 처리에 이의가 있으면 support@stockvision.kr로 문의할 수 있습니다.

## 제9조 쿠키 및 로컬 스토리지

### 9.1 쿠키
프론트엔드는 다음 쿠키를 사용합니다:

| 쿠키명 | 목적 | 유효 기간 | 필수/선택 |
|------|------|---------|---------|
| `session_token` | 로그인 상태 유지 | 7일 | 필수 |
| `ui_preference` | UI 설정(다크모드 등) | 1년 | 선택 |

### 9.2 로컬 스토리지
브라우저 로컬 스토리지에 저장되는 정보:

- 마지막 로그인 이메일 (자동입력용)
- UI 설정
- 규칙 임시 저장(임시 드래프트)

사용자는 브라우저 설정에서 언제든 삭제할 수 있습니다.

## 제10조 미성년자 개인정보

### 10.1 이용 제한
StockVision은 **만 18세 이상 성인만 이용**할 수 있습니다.

- 미성년자가 회원가입하면 계정은 즉시 삭제됩니다.
- 부모/보호자가 발견하면 support@stockvision.kr로 신고할 수 있습니다.

### 10.2 법적 근거
- 금융감독청 금융투자소비자보호규정 (만 18세 이상 투자자)
- 자본시장과 금융투자업에 관한 법률(자본시장법) (미성년자 투자 제한)

## 제11조 해외 사용자

### 11.1 서비스 범위
StockVision은 **대한민국 법규에 따라 운영**되며:

- 대한민국 증권시장 데이터만 제공 (한국 주식)
- 키움증권/한국투자증권 API 지원
- 한국 법원의 관할

### 11.2 해외 이용자
해외에서 접속하면:

- 서비스는 정상 이용 가능 (기술적 제한 없음)
- 모든 법적 책임은 대한민국 법률을 따름
- 분쟁 발생 시 서울중앙지방법원 관할

### 11.3 개인정보 국외이전

회사의 클라우드 서버가 해외 데이터센터에 위치할 경우:

- **이전 대상 정보**: 이메일, 비밀번호 해시, 전략 규칙
- **이전받는 자**: 클라우드 인프라 제공자 (AWS 등)
- **이전 목적**: 서비스 제공을 위한 데이터 저장 및 처리
- **보호 조치**: HTTPS 전송 암호화, AES-256-GCM 저장 암호화
- **금융 정보 미이전**: API Key, 거래 기록, 잔고 등 금융 정보는 국외이전 대상이 아닙니다 (사용자 PC에만 저장)

> 클라우드 서버 소재지가 확정되면 구체적인 국가명과 이전 일시를 본 방침에 갱신합니다.
