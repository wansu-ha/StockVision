# 아이디어 노트

향후 기능 아이디어 모음. 착수 시 `/spec`으로 발전시킨다.

## AI 전략 리뷰 시스템

- Claude API로 매매 전략 성과를 분석하고 개선안을 제안받는 기능
- 장 마감 후 거래 결과(승률, 수익률, 손실 종목 패턴)를 Claude에 전달
- Claude가 스코어링 가중치 조정, 지표 추가/제거 등을 제안
- 사용자가 승인/거절하여 전략에 반영 (자동 수정 아님)
- 전용 페이지: `/trading/strategy` (성과 요약, AI 분석 요청, 제안 내역, 승인/거절)
- 참고: `virtual-auto-trading` spec 완료 후 진행

## 로컬 서버 직접 접속 (클라우드 우회)

- 클라우드 서버 장애 시 원격 킬스위치가 불가한 SPOF를 우회하는 방안
- SSH 터널링으로 로컬 서버에 직접 접속하여 제어하는 방식 연구
- 또는 네이티브 앱(모바일/데스크탑)에서 로컬 서버에 직접 연결
- Tailscale/ZeroTier 같은 메시 VPN으로 NAT 뒤 로컬 서버 접근 가능성 검토
- 장기적으로 앱 개발 시 클라우드 relay 없이 직접 연결 옵션 제공 가능
- 보안 리뷰(2026-03-17)에서 원격 킬스위치 SPOF 지적으로 기록

## yfinance 대체/폴백 데이터 소스

- IndicatorProvider가 yfinance에 100% 의존 → yfinance 장애 시 지표 계산 불가
- 폴백 옵션 1: KIS REST API 일봉 조회 (사용자 본인 키, 법적 문제 없음)
- 폴백 옵션 2: 클라우드 서버 일봉 API (이미 yfinance로 수집/저장 중, 약관 검토 필요)
- 배치 요청 최적화: `yf.download(tickers="SYM1 SYM2")` → 호출 수 감소
- 아키텍처 리뷰(2026-03-17)에서 SPOF로 지적
