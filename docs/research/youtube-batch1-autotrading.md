## 영상 1: 코딩 못해도 따라하는 복붙 주식 자동화 0강 (키움 REST API)

- **URL**: https://www.youtube.com/watch?v=y4km5VRwV24
- **채널**: 얄팍한 코딩사전 (얄코)
- **길이**: 30:40
- **내용 유형**: 입문자 대상 환경 설정 강의

### 핵심 내용 요약

**API 개념 설명 방식**
- REST API를 "네가 이렇게 요청하면 내가 이렇게 해 줄게라고 정해둔 설명서"로 쉽게 설명
- 키움 서버가 실제 일 처리(시세조회, 매매 실행)를 하고, 개발자는 API 명세서대로 요청만 만들면 됨
- 비유: 매표소(토큰 발급), 방번호(기능별 URL), 건물 주소(도메인)

**다루는 기술 스택**
- 언어: Python
- IDE: Cursor (VS Code 기반 AI 강화 에디터) — VS Code 대신 Cursor 채택 이유: AI 코딩 보조
- 증권사 API: 키움 REST API
- 키움 REST API 도메인: 실전용/모의투자용 분리

**환경 설정 단계 (상세)**
1. 키움증권 계좌 개설 + 영웅문S 앱 설치
2. 키움 REST API 웹페이지 접속 → 보안 프로그램 설치
3. API 사용 신청 (실전 + 모의투자 각각)
4. IP 주소 등록 (IP 기반 접근 제어)
5. 계좌 등록 → 앱키(APP KEY) + 앱시크릿(APP SECRET) 발급
6. Python 설치 (PATH 환경변수 추가 필수)
7. Cursor IDE 설치 + Python 플러그인 + Material Icon 테마
8. `requests` 라이브러리 pip 설치

**코드 아키텍처 (config 패턴)**
- `config.py`: 앱키, 시크릿키, 호스트 URL 중앙 관리
  - `IS_PAPER_TRADING = True/False`로 모의/실전 전환
  - 모의투자용과 실전투자용 키/URL을 파일 하나에서 관리
- `login.py`: config에서 키를 가져와 토큰 발급
- `main.py`: login에서 토큰 가져와 종목 정보 조회 등 실제 기능

**모의투자 지원**
- 키움 REST API: 모의투자 전용 별도 도메인 제공
- 모의투자 계좌 신청 → 가상 계좌로 개발/테스트 가능

### StockVision 관련 인사이트

1. **IP 화이트리스트 관리**: 키움 API는 IP 기반 접근 제어 필수 → StockVision local_server도 IP 변경 시 자동 감지/안내 기능 고려
2. **config 패턴**: IS_PAPER_TRADING 플래그 하나로 모의/실전 전환 → StockVision의 설정 구조와 일치, 좋은 UX 패턴
3. **앱키 보안**: "딱 한 번만 다운로드 가능"하고 안전한 폴더 보관 강조 → keyring 기반 보안 저장 중요성 재확인
4. **Cursor(AI IDE) 채택**: 코딩 입문자 강의에서도 AI IDE를 기본으로 추천하는 트렌드 → StockVision 전략 작성에 AI 코딩 보조 기능 중요성

## 영상 2: 트레이딩 봇을 만들고 100만원을 넣었다

- **URL**: https://www.youtube.com/watch?v=63bdCwJmRP4
- **채널**: 노마드 코더 (Nomad Coders)
- **길이**: 8:59
- **언어**: 영어 (한국계 개발자 채널, 영어로 진행)
- **내용 유형**: 실전 트레이딩 봇 제작 + 실패 분석 케이스 스터디

### 핵심 내용 요약

**전략: 단순 이동평균선(SMA) 크로스오버**
- 지표: Simple Moving Average (SMA, 단순 이동평균)
- 매수 신호: 가격이 SMA를 상향 돌파
- 매도 신호: 가격이 SMA를 하향 돌파
- 자산: 비트코인 (Binance API)

**백테스팅 결과**
- 테스트 기간: 1,000시간 (약 42일, 2020.08.30 ~ 10.08)
- 기준 시간봉: 1시간봉
- 시뮬레이션 결과: $1,000 투자 → +$33 수익 (약 3.3% 수익률)
- "나쁜 수익률이지만 그래도 플러스"

**실전 테스트 (1분봉, $1,000 투입)**
- 결과: 손실
- 원인 분석:
  1. **수수료 문제**: 1분 변동폭이 너무 작아 거래소 수수료가 수익을 초과
     - 예: $570에 매수 → $571에 매도 = 이론상 수익, 실제는 수수료로 -$0.08
  2. **시간봉 불일치**: 백테스팅은 30분봉으로 했지만 실전은 1분봉으로 실행
  3. **주문 지연 문제**: Python REST API → 주문 전송 중 가격 변동
     - 해결 방향: Go + WebSocket으로 전환 (더 빠른 실행)

**핵심 교훈**
- "항상 히스토리컬 데이터로 먼저 테스트하라" — 실전 전 백테스팅 필수
- 백테스팅 기간과 실전 실행 기간(타임프레임)을 반드시 일치시켜야 함
- 수수료는 전략 수익성을 결정하는 핵심 변수

### StockVision 관련 인사이트

1. **백테스팅 기능의 필요성 명확화**: "히스토리컬 데이터로 먼저 테스트"라는 교훈 → StockVision 백테스트 모듈(Phase E 예정)의 핵심 가치 재확인
2. **수수료 시뮬레이션**: 백테스팅 시 수수료를 반드시 포함해야 실전과 같은 결과 → 백테스트 엔진에 수수료 모델 필수
3. **타임프레임 일관성**: 전략 파라미터(K값, 시간봉)와 실전 실행 타임프레임 불일치 → 전략 설정 UI에서 명확히 표시 필요
4. **WebSocket 실시간 체결**: REST API보다 WebSocket이 실시간 매매에 유리 → StockVision의 실시간 가격 모니터링에 WebSocket 방식 검토 가치 있음
5. **수익/손실 투명한 공개**: 실패를 솔직하게 공개하는 콘텐츠가 높은 신뢰도 형성 → 플랫폼 내 전략 성과 공개 기능의 UX 중요성

## 영상 3: 작년에만 자동매매로 5억번 직장인 (feat. 공돌투자자)

- **URL**: https://www.youtube.com/watch?v=A5VfkpvroSE
- **채널**: 강민우 돈깡TV
- **길이**: 21:00
- **내용 유형**: 실전 시스템 트레이더 인터뷰/밀착 취재

### 인터뷰 대상: 공돌투자자

- 10년 경력 시스템 트레이더 (직장인 겸업 → 현재 유직 중)
- 운용 금액: 2개 계좌 합산 약 5억 원
- 연 수익: 2023년 5억 원 이상 (한 계좌), 다른 계좌는 마이너스
- 매매 스타일: 단기/당일 + 종가 배팅, 국내 주식 단기 + 미국 주식 장기

### 핵심 내용 요약

**시스템 트레이딩 도입 계기**
- 직장인이라 9시~10시(장 초반)에 부장님 미팅, 3시에 또 미팅
- 업무 중 주식 계속 모니터링 불가 → 자동화 필요성 절감
- 프로그래밍 가능한 배경 → 직접 개발

**시스템 구성**
- 집에 PC를 켜두고 시스템이 자동 매매 실행
- 사무실에서는 구글 원격 데스크탑으로 원격 접속하여 모니터링
- 초록색 = 금일 매수 종목, 빨간색 = 금일 매도 종목 직관적 UI

**투자 전략 (수동 + 자동 혼합)**
- 핵심 전략: "눌림 매매" — 이미 시세가 난 종목이 조정받을 때 매수
- 종목 선정 기준:
  1. 거래대금 상위 종목 스크리닝 (돈이 들어오는 종목)
  2. 등락률 높은 종목 (많은 사람이 상방으로 보는 검증된 종목)
  3. 재료 있는 종목 (뉴스 검색으로 연속성 있는 테마인지 확인)
  4. 메가트렌드 섹터 선호 (AI, 2차전지, 로봇 등)
  5. 대기업이 진입하는 사업 영역
- 손절 원칙: 감정에 흔들리지 않게 장 마감 후 냉정하게 손절가 세팅
- 매수 타이밍: 5일선/10일선 이탈 시 일시적 투매 구간을 확률적으로 매수

**시스템 트레이딩의 현실적 한계**
- 체결 미달 문제: 자금 규모가 커지면 주문이 다 체결되지 않음 (직접 언급)
- 종목 선정(스크리닝)은 현재 수동으로 진행 — AI 자동화 희망하지만 미완성
  - 이유: 중복 뉴스, 허위 뉴스 구분이 어려움 → "아직 인간이 필요한 영역"
- 시스템 트레이딩만으로 완전 자동화 불가 — 수동 판단과 병행 필수

**지표 활용**
- 이동평균선 (5일선, 10일선)
- 볼린저 밴드 (급락 시 투매 깊이 측정)

**운용 원칙**
- 확률적 접근: "70% 성공률이면 기대값 양수"
- 리스크 관리: 확신 낮은 종목은 소액(200만 원)만 진입
- 단기 매매 비중 높고 스윙은 연습 중
- 국내 주식: 단기 (코스피/코스닥은 장기투자 부적합 경험)
- 미국 주식: 장기 투자

**조언: 시스템 트레이딩 시작 전 필요한 것**
1. 먼저 수동 매매로 자신만의 수익 나는 로직 확립
2. 그 로직이 심리/환경적 방해를 많이 받는다고 느낄 때 시스템화
3. 증권사 HTS/MTS 내장 자동 매매 기능으로 먼저 검증
4. 직장인은 월급이라는 안전 마진을 깔고 소액으로 시작

### StockVision 관련 인사이트

1. **원격 모니터링 UI**: 구글 원격 데스크탑으로 집 PC를 모니터링 → StockVision의 원격 제어/모니터링 기능이 핵심 니즈임을 재확인. 초록/빨간 직관적 포지션 UI 중요
2. **종목 스크리닝 자동화 수요**: "거래대금 + 등락률 + 뉴스 재료" 기반 스크리닝을 자동화하고 싶지만 현재 수동 → AI 기반 종목 스크리닝이 고가치 기능
3. **수동+자동 하이브리드 모델**: 완전 자동화가 아닌 자동화 + 수동 판단 결합 → StockVision이 "완전 자동화"보다 "효율적인 반자동화"를 지향해야 함
4. **체결률 모니터링**: 큰 자금에서 주문 체결 미달 문제 → 체결 현황 실시간 모니터링 기능 필요
5. **손절 자동화**: 감정에 흔들리지 않는 자동 손절 — 시스템의 핵심 가치
6. **뉴스 + AI 통합**: "AI가 뉴스 재료를 분석해서 테마 파악"을 원함 → Claude API 통합 방향과 일치

## 영상 4: 누구나 쉽게 주식 투자 자동화 — 20분만에 자동매매 시스템 완성

- **URL**: https://www.youtube.com/watch?v=2Hxfb5HT4kE
- **채널**: 조코딩 JoCoding
- **길이**: 23:00
- **내용 유형**: 한국투자증권 API 기반 자동매매 완성 실습 (KIS 후원 영상)

### 핵심 내용 요약

**사용 기술 스택**
- 언어: Python
- 증권사 API: 한국투자증권 KIS (Korea Investment & Securities) REST API + WebSocket
- 알림: Discord Webhook
- IDE: Visual Studio Code
- 코드 공유: GitHub (https://github.com/youtube-jocoding/koreainvestment-autotrade)

**KIS API 특징 (다른 증권사 대비)**
- REST API + WebSocket 동시 지원 (국내 유일 당시)
- 윈도우/맥/리눅스 모두 지원 (HTS 설치 불필요)
- 해외 주식 API 지원 (국내 유일): 미국, 일본, 상해, 홍콩
- 통합증거금: 원화만 넣으면 자동 환전으로 해외 주식 매수 가능

**전략: 변동성 돌파 전략 (Larry Williams)**
- 공식: 목표가 = 오늘 시가 + (전일 고가 - 전일 저가) × K
- K값 기본: 0.5 (조정 가능)
- 매수 조건: 현재가 > 목표가
- 매도: 당일 장마감(3시 15분~3시 20분) 일괄 매도 (당일 청산)

**코드 구조 (3파일)**
```
config.yaml         # APP_KEY, APP_SECRET, 계좌번호, Discord Webhook URL
KoreaStockAutotrade.py   # 국내 주식 자동매매 로직
UsaStockAutoTrade.py     # 미국 주식 자동매매 로직
```

**국내 자동매매 실행 흐름**
1. 액세스 토큰 발급 (앱키+앱시크릿 → 토큰)
2. 심볼 리스트: 매수 희망 종목 코드 목록 입력
3. 타겟 매수 개수 설정 (예: 4개 중 3개만 매수)
4. 종목당 투자 비율 설정 (예: 33% → 100만원이면 33만원/종목)
5. 메인 루프:
   - 9:00~9:05: 전날 잔여 주식 전량 매도
   - 9:05~15:15: 변동성 돌파 전략으로 자동 매수
   - 매시간 30분: 잔고 현황 로그 + Discord 알림
   - 15:15~15:20: 전량 일괄 매도
   - 15:20: 프로그램 종료
6. 주말 자동 휴무 처리

**해외 주식 차이점**
- 타임존: 뉴욕 시간 기준 (datetime 모듈)
- 시장 분류: 나스닥/NYSE/AMEX 별도 리스트로 관리
- 기타 로직은 국내와 동일

**Discord 알림 구성**
- 매수/매도 발생 시 실시간 알림
- 매시간 30분마다 잔고 현황 알림 (헬스체크 역할)
- 스마트폰 Discord 앱으로 받을 수 있음

**설정 방법 (계좌 세팅)**
1. 한국투자증권 앱 설치 → 뱅키스 계좌 개설
2. 통합증거금 신청 (원화로 해외 주식 자동 환전)
3. 해외ETP 거래 신청
4. KIS Developer 사이트에서 API 신청 → APP KEY + APP SECRET 발급

### StockVision 관련 인사이트

1. **변동성 돌파 전략 구현 참고**: K값 파라미터 조정 가능한 전략 → StockVision 전략 엔진의 파라미터 설정 UI 설계 시 참고
2. **Discord Webhook 알림**: 매수/매도 이벤트 + 정기 헬스체크 알림의 분리 구성 → StockVision의 알림 시스템 (alert store)과 설계 방향 일치
3. **일괄 당일 청산 로직**: 3:15~3:20 강제 전량 매도 → 리스크 관리를 코드 레벨에서 하드코딩 → 전략 엔진에 "당일 청산" 옵션 중요
4. **GitHub 코드 공개**: 오픈소스로 코드 공개하여 신뢰 형성 + 커뮤니티 기여 → StockVision도 샘플 전략 코드 공개 고려
5. **잔고 모니터링 주기**: 매시간 30분마다 자동 잔고 보고 → 단순하지만 효과적인 헬스체크 패턴
6. **KIS API**: StockVision의 KIS 어댑터 이미 구현 중 — 이 영상의 API 구조(토큰 발급, 종목 조회, 매수/매도)와 일치 확인

### StockVision에 적용 가능한 핵심 발견

**1. 백테스팅은 필수 기능**
영상 2의 핵심 교훈: 백테스팅 없이 실전 → 실패. 수수료 포함, 타임프레임 일관성 필수.

**2. 원격 모니터링 + 알림 시스템이 핵심 가치**
공돌투자자는 구글 원격 데스크탑으로 집 PC를 원격 관리. StockVision의 클라우드+로컬 아키텍처가 이 니즈를 직접 해결.

**3. 수동+자동 하이브리드 모델**
5억 수익 트레이더도 종목 스크리닝은 수동. "완전 자동화"보다 "효율적 반자동화"가 현실적 목표.

**4. 변동성 돌파 전략이 입문자에게 가장 많이 사용되는 전략**
영상 3, 4 모두 변동성 돌파(Larry Williams) 전략을 예시로 사용. StockVision의 기본 제공 전략으로 적합.

**5. Discord/알림 채널 통합 수요 강함**
조코딩, 공돌투자자 모두 알림 시스템 강조. StockVision의 알림 기능 고도화 방향 확인.

**6. AI 기반 뉴스/재료 스크리닝 미해결 니즈**
공돌투자자: "AI가 뉴스 재료 분석해서 테마 파악하면 좋겠지만 아직 미완성". Claude API 통합의 명확한 사용 사례.

**7. 수수료 모델 포함 시뮬레이션**
영상 2의 실패 원인: 수수료 미고려. 백테스팅 엔진에 수수료, 슬리피지 반드시 포함 필요.

**8. 설정의 단순성이 UX의 핵심**
모든 영상에서 `config.py`/`config.yaml` 파일 하나로 모든 설정 집중. StockVision의 설정 페이지 UX도 단순성 유지 중요.
