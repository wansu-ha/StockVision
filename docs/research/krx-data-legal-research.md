# KRX 시세 데이터 지연 재배포 법적 조사

> 작성일: 2026-03-15 | 상태: 조사 완료 | 법률 자문 아님 — 상업 출시 전 전문가 확인 필요

## 핵심 결론

한국에는 미국처럼 "15분 지연이면 무료 재배포 가능"이라는 법적 기준이 **존재하지 않는다**.
증권사 API로 받은 시세를 DB에 저장하고 다수 사용자에게 제공하는 것은 **원칙적으로 약관 위반 가능성이 높다**.

---

## 1. KRX 시세 데이터 구조

| 구분 | 설명 | 재배포 |
|------|------|--------|
| 실시간 시세 | 코스콤(독점 배포) → 데이터 벤더 → 최종 사용자 | 유료 계약 필수 |
| 지연 시세 (~20분) | KRX 웹사이트에서 무료 조회 가능 | 조회는 가능, **재배포는 별도 계약 필요** |
| EOD (장 마감 후) | KRX 데이터포털(data.krx.co.kr) CSV 다운로드 | 비상업적/연구 목적 허용, 상업적 사용은 계약 필요 |

배포 체인:
```
KRX → 코스콤(독점 라이선스) → 언론사/증권사/데이터벤더
                              → 네이버, 카카오 등 포털 (계약 후 표시)
```

법적 근거: 자본시장법 제294조~제296조 — KRX 시세 데이터는 KRX의 지적재산으로 보호.

## 2. 증권사 API 약관 제약

### 키움증권 Open API+
- 발급 대상: **개인 투자자 본인의 자동매매 목적**에 한정
- 수신 데이터의 **타인 제공 및 재배포 명시적 금지**
- 다수 사용자 서비스에 중계하는 SaaS 구조 = 약관 위반

### 한국투자증권(KIS) Developers API
- 기업용/개인용 구분 존재
- **제3자 제공 및 재배포 금지** 조항 포함
- 장기 이력 DB 구축은 원칙적 불허

### 행위별 허용 여부

| 행위 | 허용 여부 |
|------|-----------|
| API 호출 → 즉시 화면 표시 | 허용 |
| 단기 서버 캐싱 (수분~수십분) | 회색지대 |
| DB에 이력 저장 후 제공 | 원칙적 금지 |
| 가공(차트)하여 다수 사용자 제공 | 재배포 해당, 금지 |
| 본인 전용 자동매매 활용 | 개인 API 허용 범위 |

## 3. 미국(NYSE/NASDAQ) 비교

미국은 명시적 규제 체계가 있음 (SEC Regulation NMS):
- **실시간 시세**: Redistributor Fee 납부 의무
- **15분 지연 시세**: 거래소가 자발적으로 무료 재배포 허용 (법적 의무가 아닌 상업적 정책)
- Yahoo Finance, Google Finance 등이 이 정책에 기반하여 지연 시세 무료 제공

**한국은 이런 정책이 없다** — 지연 시세도 재배포 시 계약 필요.

## 4. 핀테크/로보어드바이저의 합법적 방법

### A. 코스콤 직접 계약 (주요 방법)
- 코스콤 금융 데이터 이용계약 체결
- 비용: 월 수십만원~수백만원 (서비스 규모 비례)
- 합법적 재배포 권한 확보

### B. KRX 데이터포털 API (data.krx.co.kr)
- EOD 데이터 중심, 실시간 미제공
- 비상업 단계 무료, 상업화 시 계약 전환

### C. 외부 데이터 벤더
- 뉴지스탁(Quantit), 씽크풀 API, FnDataGuide
- 코스콤/KRX와 계약 후 sublicensing 형태

### D. 투자일임업 허가 + 증권사 제휴
- 금융투자업 등록 후 증권사 파트너십
- 증권사의 기존 KRX/코스콤 계약을 활용

## 5. StockVision 선택지

| 방안 | 내용 | 비용 | 리스크 |
|------|------|------|--------|
| **A. EOD만 사용** | yfinance 일봉만 제공, 분봉/실시간 제거 | 무료 | 최저 |
| **B. KRX 데이터포털** | EOD API 신청, 상업화 시 계약 전환 | 무료→유료 | 낮음 |
| **C. 코스콤/벤더 계약** | 월정액 계약 후 합법적 재배포 | 월 수십만원~ | 없음 |
| **D. 프론트 직접 호출** | 서버 캐싱 없이, 사용자 브라우저에서 증권사 API 직접 호출 | 무료 | 회색지대 |
| **E. 현행 유지** | yfinance 일봉 + 로컬 서버 실시간(본인 키) | 무료 | 낮음 (본인 데이터) |

### 현재 구조 (E안) 분석
- 일봉: yfinance (Yahoo Finance 데이터) → 서버 캐싱 → 제공 — **Yahoo ToS 위반 가능성 있으나 관행적 허용**
- 실시간: 로컬 서버에서 사용자 본인 브로커 키로 조회 → 본인에게만 표시 — **약관 범위 내**
- 클라우드에서 서비스 키로 수집 → DB 저장 → 다수 사용자 제공 — **약관 위반 가능성 높음**

---

## 6. 권장 방향

1. **현 단계 (비공개 베타)**: E안 유지 — 일봉(yfinance) + 로컬 실시간(본인 키)
2. **분봉/시봉 추가 시**: 로컬 서버에서 본인 키로 조회하는 구조 유지 (서버 캐싱 X)
3. **상업화 시**: 코스콤 또는 데이터 벤더와 계약 후 전환
4. **Unit 7 (legal)**: 이용약관에 "시세 데이터는 참고용" 면책 조항 명시
