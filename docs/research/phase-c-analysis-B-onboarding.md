# B. 온보딩 재설계 분석

## B-1. 첫 번째 성공 경험 정의

### 후보별 평가

| 후보 | 도달 난이도 | 소요 시간 | 만족도 | 구현 난이도 |
|------|------------|----------|--------|------------|
| 차트 보기 | 낮음 (2클릭) | 30초 | 낮음 — "구글 파이낸스랑 뭐가 달라?" | 없음 (구현됨) |
| 전략 만들기 | 중간 | 3~5분 | 중간 — 성취감 있으나 "그래서?" 의문 | 없음 (구현됨) |
| 백테스트 | 중간 | 5~10분 | 높음 — "작년에 수익 났네" | 높음 (미구현) |
| 모의투자 체결 | 높음 (로컬+mock) | 20~30분 | 매우 높음 | 중간 |
| 실전 첫 체결 | 매우 높음 | 45분+ | 최고 | 없음 (셋업 장벽) |

### 권장: 두 단계 성공 경험

1. **즉각 성공** (가입 직후, 0분): 규칙 만들기 + "오늘 이 규칙이 발동했을 종목 N개" 시뮬레이션
2. **완전 성공** (로컬+증권사 후): 실제 자동 주문 체결

(1)은 클라우드 API만으로 가능. 현재 코드베이스에서 구현 비용 최소.

## B-2. 온보딩 순서 재검토

### 현재 문제
```
가입 → 위험고지(맨 앞!) → 로컬 설치(5분 대기) → 증권사 → 대시보드
```
- 위험고지가 맨 앞 → 제품 이해 전 법률 텍스트 → 이탈
- BridgeInstaller가 "연결 대기 중... (7회 시도)"만 표시 → 불안
- 온보딩 끝나도 대시보드에서 아무 일도 안 일어남

### 제안 플로우
```
가입/로그인
  ↓
[즉각 체험 — 클라우드만]
  종목 검색 → 차트 → 첫 규칙 생성 → "오늘 이 규칙 발동 신호 N건"
  (5분 이내, 소유감 형성)
  ↓
[설치 동기]
  "이 규칙을 실제 실행하려면 로컬 서버 필요합니다"
  ↓
위험고지 (이 시점엔 제품 이해한 상태)
  ↓
로컬 서버 설치
  ↓
증권사 API 등록
  ↓
대시보드
```

### 단계별 이탈 완화

| 단계 | 이탈 원인 | 완화 |
|------|---------|------|
| 가입 | 귀찮음 | 소셜 로그인 or "둘러보기" |
| 즉각 체험 | 없음 (장벽 없음) | 첫 규칙 템플릿 제공 |
| 설치 동기 | 체험 후라 낮음 | "이미 만든 규칙이 기다리고 있습니다" |
| 위험고지 | 법률 텍스트 피로 | 3줄 요약 + 펼침 |
| 로컬 설치 | 기술적 장벽 | 타이머+진행 표시, "나중에" 허용 |
| 증권사 | 계정 미보유 | "모의투자 키로 시작" 강조 |

### 전환 메시지
- 체험→설치: "방금 만든 규칙, 자동 실행하려면 한 단계 더 (5분)"
- 설치→위험고지: "시작 전 알아야 할 것들 (1분)"
- 설치→증권사: "거의 다 왔습니다. 증권사 키만 등록하면 완료"

## B-3. 상태별 CTA 설계

### 상태 A: 가입만 (localReady=false)
- 위치: MainDashboard 상단 persistent 배너
- 메시지: "자동매매 실행을 위해 로컬 서버 설치 필요 [5분 설치 시작] [나중에]"
- 톤: 중립 안내 (경고 아님)

### 상태 B: 로컬 연결, 증권사 미등록
- 위치: 잔고/보유종목 카드 내부 overlay
- 메시지: "계좌 정보를 보려면 증권사를 연결하세요 [증권사 연결]"
- 톤: 카드 흐림 + 잠금 아이콘

### 상태 C: 증권사 연결, 엔진 미실행
- 위치: OpsPanel 엔진 상태 옆 인라인
- 메시지: "엔진: 정지 [시작]"
- 별도 배너 불필요

### 상태 D: 규칙 없음 (rules.length === 0)
- 위치: ListView 빈 상태 화면
- 메시지: "아직 매매 규칙이 없습니다. [규칙 템플릿으로 시작]"
- 톤: 긍정적 행동 유도

### CTA 우선순위 (동시 시 하나만)
로컬 미연결 > 증권사 미연결 > 규칙 없음

### `useOnboarding` 플래그 재정의
현재 localStorage만 보므로 실제 상태와 괴리. "위험고지 동의 여부"로 의미 좁히고, 나머지는 `localReady`/`brokerConnected` 서버 상태로 판단.

## B-4. 음영/잠금 UI 패턴

### 잠금 대상 분류

| 구분 | 기능 | 해제 조건 |
|------|------|----------|
| 완전 사용 | 종목 검색, 차트(일봉), 규칙 CRUD, 관심종목, 시장 컨텍스트 | 가입만 |
| 로컬 해금 | 설정 > 증권사 키 등록 폼 | +로컬 서버 |
| 증권사 해금 | 잔고, 보유종목, 실시간 시세, 미체결 주문 | +증권사 |
| 완전체 | 자동매매 시작/중지, 체결 로그 | +엔진 실행 |

### 음영 원칙
- 잠금(lock)이 아닌 의도적 음영: "기능이 있는데 비활성 상태"
- 불투명도 50% + 흐린 배경 + 해제 방법 한 줄 안내
- 0원 표시 대신 "증권사 연결 후 확인 가능" 상태 카드

### 혼합 데이터 카드
잔고 카드: 0원 → "잔고 확인을 위해 증권사를 연결하세요 [연결하기]"

### 데이터 소스 배지
차트 우상단: "일봉 (실시간 미연결)" — 로컬+증권사 연결 시 자동 사라짐

## B-5. 위저드 vs 인라인

### 현재 위저드 문제
- Step 자동 스킵 시 화면 점프 UX
- `useOnboarding` localStorage = 실제 상태와 분리
- `/onboarding` 격리 → 대시보드 가치 경험 전 이탈

### 권장: 하이브리드
```
위저드 (2단계만, 경량):
  Step 1: 즉각 체험 (종목 검색 + 첫 규칙)
  Step 2: 위험고지 동의
  → 대시보드 이동

인라인 (대시보드 내):
  → 로컬 설치 배너
  → 증권사 연결 카드 overlay
  사용자가 준비됐을 때 완료
```

핵심: 위험고지만 위저드에서 반드시 완료. 기술적 설치는 인라인.

## B-6. "왜 로컬인가" 메시지

### 권장 메시지

**설치 동기 단계:**
> 주문은 내 PC에서 직접 실행됩니다.
> 키가 외부로 전송되지 않고, 내 컴퓨터가 주문을 보냅니다.

**위험고지 직후:**
> 이 설정 이후 당신의 API 키는 이 PC의 암호화 저장소에만 보관됩니다.

**증권사 키 입력 폼 위 (BrokerKeyForm):**
> 입력한 키는 Windows 자격증명 관리자에 암호화 저장됩니다.
> 클라우드 서버로 전송되지 않습니다.

Windows Credential Manager 명시 → IT 리터러시 있는 타겟에게 강한 신뢰.

## 종합 권장 (우선순위)

### 즉시 (비용 낮음, 효과 높음)
1. 위험고지를 온보딩 맨 앞에서 제거 → 체험 후 이동
2. `useOnboarding` → "위험고지 동의" 플래그로 재정의
3. MainDashboard에 `localReady === false` 배너 CTA 추가
4. 잔고 카드 빈 상태 → "증권사 연결 후 확인" 상태로 교체

### 중기 (비용 중간)
5. 하이브리드 온보딩 (위험고지만 위저드, 나머지 인라인)
6. 차트 데이터 소스 배지 추가

### 유지 (변경 불필요)
- BridgeInstaller 폴링+딥링크 (UX 텍스트 수정만)
- BrokerKeyForm showHints 패턴
- useAccountStatus 5초 폴링
