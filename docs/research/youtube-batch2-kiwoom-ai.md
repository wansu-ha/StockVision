## 영상 1: 왕초보도 가능한! 내 조건검색식으로 자동매매를 해주는 프로그램을 하루 만에 만들어 보자!

- **채널**: 2026 키움영웅전
- **URL**: https://www.youtube.com/watch?v=_BAdI9THvYc
- **길이**: 15분 52초
- **조회수**: 137,302회 (2025-11-28 업로드)
- **태그**: #키움증권 #조건검색 #자동매매

### 요약

키움증권 공식 채널에서 제작한 영상. 키움 REST API + Python으로 조건검색식 기반 자동매매 프로그램을 만드는 방법을 시연. 실제 개발자(얄코)와 키움 API 담당자(이재혁 주임)가 협력하여 제작.

### 핵심 내용

#### 시스템 구조
- **입력**: 영웅문 HTS의 조건검색식 (사용자가 미리 설정)
- **처리**: Python 코드가 실시간으로 조건검색 결과를 모니터링 → 종목 발생 시 자동 매매
- **제어**: 텔레그램 봇을 리모콘으로 사용 (start/stop/report/help 명령어)
- **알림**: 매수/매도 완료 시 텔레그램으로 실시간 알림

#### 기술 스택
- **언어**: Python (비개발자도 노션 페이지에서 코드 복사·붙여넣기로 조립)
- **API**: 키움 REST API (앱키/앱시크릿으로 인증)
- **코드 실행기**: Cursor (코드 에디터)
- **제어 채널**: 텔레그램 봇

#### 키움 REST API 특장점 (영상에서 언급)
- API당 초당 5회 조회 (유량 제어)
- 조건검색 ID당 10개 등록 가능
- 웹소켓 실시간 등록 최대 100개
- 영웅문 전 기능 API화 (0198 실시간 조회 순위 등 키움 전용 데이터 포함)

#### 프로그램 기능
- 조건검색식에서 종목 발생 → 자동 매수
- 수익률 목표 달성 시 자동 익절
- 손절선 도달 시 자동 손절
- 예수금 사용 비율 설정
- 텔레그램으로 익절/손절 비율 실시간 변경 가능
- `report` 명령: 보유 종목 + 수익률 리포트
- `start`/`stop` 명령: 매매 프로세스 제어

#### 사용자 세팅 과정
1. 영웅문 HTS에서 조건검색식 설정
2. 키움 REST API 사이트에서 앱키/앱시크릿 발급
3. Python 설치
4. Cursor(코드 에디터) 설치
5. 노션에서 코드 복사·조립
6. 텔레그램 봇 연결

### StockVision 관련 인사이트

| 관찰 | StockVision 적용 아이디어 |
|------|--------------------------|
| 텔레그램을 리모콘으로 사용 | 원격 제어 채널 (하트비트 command 필드 vs 텔레그램 봇) 비교 - 텔레그램 방식이 사용자에게 익숙함 |
| 조건검색 → 매매 파이프라인 | 로컬 서버 전략 엔진의 "신호 감지 → 주문" 플로우와 동일 구조 |
| 익절/손절 실시간 변경 | 실행 중인 전략의 파라미터 hot-reload 기능 필요성 확인 |
| 코드 노출 없이 배포 (노션 복붙) | StockVision은 GUI로 이를 대체 - UX 방향성 확인 |
| 국내주식만 지원, 미국주식은 계획 중 | StockVision도 동일 단계 - Phase 순서 참고 |

## 영상 2: 구글 제미나이 3.0: 8분 만에 만드는 AI 주식 자동매매 시스템 (소스코드 무료 배포)

- **채널**: 호두의 AI 분석실
- **URL**: https://www.youtube.com/watch?v=8BsHVuJdJgE
- **길이**: 11분 39초
- **조회수**: 15,835회 (2025-12-05 업로드)
- **태그**: #주식자동매매 #AI코딩 #구글Antigravity #퀀트투자

### 요약

Google Antigravity(Cursor의 Google 버전, AI 코딩 도구)와 MD 기획서 파일 하나로 "주식 자동 분석 대시보드"를 8분 만에 만드는 과정 시연. 실제 매매 자동화보다는 **AI 기반 주식 분석 대시보드** 구축에 초점.

### 핵심 내용

#### 시스템 구조
- **입력**: MD 파일(기획서) 1개
- **AI 도구**: Google Antigravity (Cursor와 유사한 AI 코딩 에이전트)
- **결과물**: Flask 기반 웹 대시보드 (로컬 실행)
- **데이터 소스**: Yahoo Finance API (실시간 환율, 달러 인덱스, 주가)

#### 대시보드 기능
- 야후 파이낸스 API로 환율/달러 인덱스 등 거시 지표 실시간 표시
- AI 알고리즘으로 종목 분석 → S/A/B 등급 분류 후 추천 리스트 생성
- 성과 추적: 과거 추천 종목의 추천 당시 주가 vs 현재 주가 비교
- 스타일 박스 계산

#### 미완성 기능 (영상에서 솔직하게 공개)
1. **AI 분석 로직**: 파동 이론/시장 국면 탐지 대신 무작위 점수 생성 (초안)
2. **데이터 수집 범위**: 10개 종목만 → 코스피/코스닥 전종목 확대 필요
3. **성과 추적 로직**: 미구현
4. **스타일 박스 계산**: 하드코딩 상태

#### 개발 과정
1. MD 기획서 파일을 Antigravity 폴더에 넣기
2. AI에게 "대시보드 생성해줘" 요청 (Plan 모드)
3. 중간중간 Accept 클릭
4. Flask 서버 실행 요청 → 로컬 URL 생성
5. 버그 발생 시 AI에게 "데이터가 안 뜨니 고쳐줘" 메시지 → AI가 자동 디버깅

#### AI 자동 디버깅 사례
- 데이터 연결 오류 → AI가 화면 확인 후 자동 수정
- 그래프 미표시 → AI에게 "왼쪽 그래프가 안 나오니 고쳐줘" → 1분 내 수정

### StockVision 관련 인사이트

| 관찰 | StockVision 적용 아이디어 |
|------|--------------------------|
| MD 파일 기획서 → AI가 코드 생성 | spec 문서를 AI 코딩에 직접 활용하는 방식 - 현재 StockVision 워크플로와 유사 |
| S/A/B 등급 종목 추천 시스템 | 클라우드 서버 AI 서비스의 종목 추천 등급화 인터페이스 참고 |
| 성과 추적 (추천 → 실제 수익률) | 전략 백테스트 + 추천 이력 기록 기능의 사용자 가치 확인 |
| 야후 파이낸스 API 활용 | 글로벌 거시 지표 데이터 소스로 참고 (국내: KRX, 해외: Yahoo Finance) |
| 초안 단계 AI 로직의 솔직한 공개 | 사용자에게 AI 분석 신뢰도 투명하게 표시하는 UX 방향성 |

## 영상 3: AI 활용해 경제학 유튜브 쉽게 만드는 방법 | 프로그램 무료 제공

- **채널**: AI 실전 수익화 클래스 | 게으른 유튜버
- **URL**: https://www.youtube.com/watch?v=BLTdSwwJPNc
- **길이**: 9분 1초
- **조회수**: 20,331회 (2026-03-10 업로드)
- **태그**: #경제학 #ai #유튜브강의 #부업

### 요약

경제학/심리학 유튜브 채널을 AI 툴로 자동화하는 방법 소개. 주식 자동매매와는 직접적 관련 없으나, **Gemini API를 활용한 커스텀 앱 제작 방법론**과 **Gemini Canvas 기능**이 StockVision의 AI 기능 개발에 참고할 만한 내용 포함.

### 핵심 내용

#### 이미지 자동화 프로그램 (직접 제작 배포)
- 유튜브 대본 입력 → AI가 각 장면에 맞는 이미지 생성
- 화면 비율 선택 (9:16, 16:9 등)
- 비주얼 테마: 현실적/한국 카툰/본 캐릭터 + 커스텀 프롬프트
- 스타일 레퍼런스 이미지 첨부 가능
- 전체 이미지 일괄 생성/다운로드
- Gemini API 키 필요 (Google AI Studio에서 발급)

#### Gemini Canvas 기능 (핵심 인사이트)
- Gemini 도구 메뉴 → Canvas 진입
- **자연어 명령으로 HTML/JS 앱 즉시 생성**
- 예시: "유튜브 API 키로 실시간 급등 동영상 찾아줘, 조회수 정렬, 기간 필터 포함"
- 생성된 코드를 메모장에 복사 → `.html`로 저장 → 브라우저에서 실행
- 빠른 모델보다 **사고 모델(thinking model)** 권장 (품질 향상)
- 무료 tier는 사용량 매우 제한적 → 유료(Tier 1) 권장

#### 영상 제작 워크플로
1. 이미지 자동화 프로그램으로 스토리보드 이미지 생성
2. Kling AI로 첫 3개 이미지를 동영상으로 변환 (Gemini보다 저렴+성능 좋음)
3. Typecast(타이캐스트)로 AI 음성 생성
4. CapCut으로 최종 편집 + 줌인(Ken Burns) 효과 적용

### StockVision 관련 인사이트

| 관찰 | StockVision 적용 아이디어 |
|------|--------------------------|
| Gemini Canvas로 HTML 앱 즉시 생성 | 프론트엔드 프로토타이핑에 활용 가능한 AI 도구 |
| Gemini API 키 관리 (외부 노출 금지, 재발급) | API 키 보안 관리 UX 참고 (StockVision 증권사 API 키 관리 화면) |
| 사고 모델(thinking) vs 빠른 모델 선택 | AI 서비스에서 분석 깊이/속도 트레이드오프 설정 옵션 아이디어 |
| 자연어로 기능 추가 (반복 수정) | StockVision 전략 빌더의 자연어 입력 방식 참고 |

## 종합 분석: StockVision 관련 주요 발견

### 1. 자동매매 UX 패턴 확인

영상 1에서 확인된 실용적 UX 패턴:
- **텔레그램 리모콘**: start/stop/report/help 명령어 세트가 단순하고 직관적
- **실시간 파라미터 변경**: 장 중 익절/손절 비율 즉시 변경 → StockVision의 전략 hot-reload 기능의 사용자 가치 확인
- **원클릭 세팅 후 방치**: 초기 세팅은 복잡하지만 이후는 텔레그램으로만 운영

### 2. AI 주식 분석 대시보드 구조

영상 2에서 확인된 구현 구조:
```
Yahoo Finance API → 거시 지표 (환율, 달러 인덱스)
KOSPI/KOSDAQ 전종목 데이터 → AI 분석 엔진
→ S/A/B 등급 종목 추천
→ 성과 추적 (추천 당시 vs 현재 주가)
→ Flask 웹 대시보드
```

### 3. Gemini API 활용 패턴

영상 3에서 확인된 활용 패턴:
- MD 기획서 파일 → AI 코딩 도구 → 완성 앱
- Gemini Canvas로 HTML 단일 파일 앱 생성
- 반복적 자연어 수정으로 기능 추가

### 4. 키움 REST API vs 오픈 API Plus

영상 1에서 언급:
- REST API 방식: 키움 REST API (앱키/앱시크릿 인증)
- 기존 오픈 API Plus와 병존
- 초당 5회 유량 제어, 웹소켓 100개 실시간 등록
- **StockVision local_server의 키움 브로커 어댑터 설계에 직접 참고 가능**
