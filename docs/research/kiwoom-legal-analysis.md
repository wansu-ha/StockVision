## 2. 약관 문서 분석

### 2.1 전자금융거래 이용에 관한 기본약관
> 제정 2007.03.30 / 최종 개정 2025.07.10

**성격**: 키움증권 전자금융거래 전반에 적용되는 기본약관. OpenAPI 전용 약관 아님.

**StockVision 관련 핵심 조항**

**제2조 6항 — 접근매체 정의**:
> "회사에 등록된 고객번호, 전자서명생성정보 및 인증서, 비밀번호"

→ API 키, 계좌번호, 인증서, 비밀번호 **전부 접근매체**에 해당.

**제14조②2 — 금지행위**:
> "**대가를 수수·요구 또는 약속하면서** 접근매체를 **대여받거나 대여하는 행위** 또는 보관·전달·유통하는 행위"

→ 유료 SaaS가 사용자 API 키를 서버에 저장하면 이 조항에 저촉될 수 있음.
→ 단, 젠포트가 키움 공식 서비스로 운영되는 것을 볼 때 실무적으로는 묵시적 허용.
→ **OpenAPI 별도 약관 확인 필요** (이 약관은 일반 전자금융거래 기준).

**제8조①2 — 회사 면책**:
> "고객이 접근매체를 **제3자에게 대여하거나 사용을 위임**하거나 양도 또는 담보 목적으로 제공한 경우"

→ 키움이 이 상황(제3자에게 접근매체 위임)을 인지하고 면책 조항으로 처리.
→ 즉 완전 금지가 아니라 "발생 가능한 상황"으로 인정.

### 2.2 자본시장 공동 핀테크 오픈플랫폼 금융거래정보조회 서비스 이용 약관
> 제정 2019.06.19 / 최종 개정 2025.07.10

**성격**: 증권 계좌 **읽기(조회) 권한**을 핀테크 기업에 부여하는 공식 경로.
**운영기관**: ㈜코스콤 (`http://open.koscom.co.kr`)

**핵심 구조**:
```
키움증권 (회사)
    ↕ 데이터 제공 (사용자 동의 기반)
코스콤 오픈플랫폼 중계센터
    ↕ API 중계
핀테크기업 (StockVision이 등록 가능)
    ↕ 서비스 제공
사용자
```

**제2조 5항 — 핀테크기업 정의**:
> "자본시장 공동 핀테크 오픈플랫폼에 **연계된** 핀테크서비스를 제공하는 기업"

→ StockVision이 코스콤에 등록하면 공식 "핀테크기업" 지위 획득 가능.

**제6조 — 제공 가능한 데이터 종류**:
1. 사용자 본인명의 계좌의 **실시간 잔고** 내역
2. 사용자 본인명의 계좌의 **실시간 거래** 내역
3. 사용자 본인명의 계좌의 **자산별 구성** 내역
4. 사용자 본인명의 계좌에 설정된 **관심종목** 내역

→ 포트폴리오 대시보드, 수익률 추적, 잔고 기반 전략 조정 등 구현 가능.

**제3조③ — 동의 유효기간**:
> "동의의 유효기간은 **1년**이며, 계속 이용하고자 하는 사용자는 **1년 단위로 재동의**"

→ 운영 요건: 매년 사용자 재동의 플로우 필요.

**이 약관의 커버 범위**: 읽기 전용. 자동매매 실행은 별도 경로(OpenAPI+).

### 2.3 오픈뱅킹 서비스 이용약관
> 제정 2020.12.22 / 최종 개정 2025.07.10

**성격**: **은행 계좌** 연동 이체/조회 서비스. 증권 자동매매와 직접 관련 없음.
**운영기관**: 사단법인 금융결제원

**문서 2(코스콤/증권)와의 차이**:

| 구분 | 문서 2 (코스콤) | 문서 3 (금융결제원) |
|------|----------------|-------------------|
| 대상 계좌 | 증권 계좌 | 은행 계좌 |
| 주요 기능 | 잔고·거래내역 조회 | 이체, 출금, 자금모으기 |
| StockVision 활용 | 포트폴리오 조회 | 투자금 입금 자동화 |

**제10조 — 서비스 종류 (주목할 항목)**:
- `재무관리`: 고객의 자산을 관리하는 서비스 ← 자산관리 기능
- `자동출금이체`: 고객의 **사전 지시에 따라 지정일에 자동 출금이체** ← 정기 투자금 자동 이체 가능
- `자금모으기`: 다수의 신청계좌에서 하나의 계좌로 자금 집중

→ StockVision 활용 시나리오: 사용자 은행 계좌 → 키움 증권 계좌 자동 이체 (정기 투자)

### 2.4 오픈 API 서비스 이용약관
> 제정 2025.02.20 (최신 제정)

**성격**: 키움증권 OpenAPI+(HTS 기반 자동매매 API) 전용 약관. **StockVision에서 가장 직접적으로 적용되는 핵심 약관.**

---

**제3조 — 적용범위**:
> \"고객이 회사에서 제공하는 오픈 API 서비스(조회 및 **매매주문업무** 포함)를 전자금융거래 방식으로 이용하는 것에 적용한다\"\

→ 조회 뿐 아니라 **매매주문**이 공식 API 범위에 명시. 자동매매 주문 실행 자체는 API 설계 의도에 포함.

---

**제5조② — 고객 의무 (가장 중요)**:
> \"고객은 자신의 보안코드(앱키, 시크릿키)가 유출 및 도용되지 않도록 관리해야 하고, **제3자에게 대여, 위임, 양도 또는 본인 이외의 제3자에게 누설해서는 아니 된다.**\"

→ **클라우드 SaaS 구조 명시적 위반.**
→ 사용자 API 키를 StockVision 서버에 저장하거나 서버가 사용자 대신 API를 호출하는 것은 이 조항 위반.
→ 문서 1 제14조②2의 그레이존보다 훨씬 명확한 금지.

---

**제5조③ — 시세 정보 이용 제한**:
> \"고객은 회사에서 제공하는 시세 정보를 **고객이 직접 사용할 목적의 프로그램 개발 등 개인의 업무에 한하여** 이용해야 하며, **제3자에게 제공해서는 아니 된다.**\"

→ **StockVision 서버에서 시세 데이터를 여러 사용자에게 중계하는 것은 약관 위반.**
→ 각 사용자가 자신의 키움 API를 직접 호출해야 함 (로컬 브릿지 필수).
→ 코스콤 오픈플랫폼(문서 2)이 별도로 필요한 이유 — 읽기 데이터는 그쪽 경로로.

---

**제11조 — API 호출 제한**:
- 조회: 초당 **5건**
- 주문: 초당 **5건**
- 실시간 조건검색: 로그인 1개당 **10건**

→ 단일 서버에서 수백 사용자 계좌를 관리하면 제한에 즉시 걸림.
→ 로컬 브릿지 아키텍처에서 각 사용자 PC가 제한을 각자 부담 → 문제 없음.

---

**제14조 — 약관 위반 시 제재**:
> 서비스 이용을 제한하거나 중지할 수 있다

→ 클라우드 SaaS가 발각되면 **계정 전체 정지** 가능성.

---

**아키텍처 결론**:
```
❌ 금지 구조 (클라우드 SaaS):
  사용자 → StockVision 서버 (API 키 보관) → 키움 API 직접 호출

✅ 합법 구조 (로컬 브릿지):
  StockVision 서버 (전략 신호만) → 사용자 PC (로컬 브릿지)
                                    ↓ API 키는 로컬에만
                                  키움 API 직접 호출
```

### 2.5 커뮤니티 서비스 이용약관
> 제정 2025.12.18

**성격**: 키움 HTS/MTS 내의 **소셜 커뮤니티 기능** (종목게시판, 실시간 채팅, 내 투자정보 공유) 약관.
**StockVision 핵심 기능(자동매매, API 연동)과 직접 관계 없음.**

**StockVision 관련 조항 (있다면)**:

**제4조①6**:
> "회사의 사전 동의 없이 **영리를 목적으로 커뮤니티 서비스를 사용하는 행위**"

→ 키움의 커뮤니티 서비스(HTS 게시판, 채팅)를 영리 목적으로 사용하면 안 된다는 것.
→ **StockVision은 키움 커뮤니티 기능을 사용하지 않음** — 자체 서비스 운영이므로 해당 없음.

**제9조②5**:
> "콘텐츠에 **(유사)투자자문업**, 집합투자업, 대부업, 투자조합 등에 대한 홍보, 소개, 광고 등이 포함된 경우"

→ 키움 커뮤니티 게시판에 유사투자자문업 홍보 글 게시 금지.
→ StockVision 서비스 자체를 금지하는 조항이 아님. 키움 게시판에 광고 올리지 말라는 것.

**제17조① — 면책**:
> "투자판단은 전적으로 회원의 권한과 책임 하에 이루어진다"

→ 투자 책임은 회원 귀속 — 이 원칙은 G1, G6와 일치. StockVision 설계 원칙에 부합.

**결론**: 이 약관은 **StockVision의 자동매매/자산관리 기능에 새로운 제약을 추가하지 않는다.**
기존 G5 (오픈 API 약관) 제5조②③가 핵심 제약이며, 이 문서는 그 범위 밖의 서비스.

## 3. 추가 근거

### 3.1 시스템매매 위험고지
> 출처: 키움증권 거래시 유의사항 → 시스템 매매 위험고지 (공식 거래 흐름 내)

**공식 정의**:
> "고객 자신의 판단을 배제하고 사전에 내장된 일련의 조건에 의하여 금융투자상품의 매매종목, 매매시점 또는 매매호가에 대한 **의사결정정보를 제공하거나 자동으로 매매주문을 내는** 전산소프트웨어에 의한 매매거래"

**의미**:
- 신호 제공형 AND 자동주문형 **모두** 공식 허용
- "시스템매매를 신청하기 전에" → 별도 신청 절차 존재 → 정규 서비스로 제도화
- 모든 손실은 고객 귀속 (3회 명시)

**StockVision 운영 요건**:
- 사용자에게 이 위험고지 내용 수락 받아야 함
- 사용자가 먼저 키움에서 시스템매매 신청 완료해야 함

### 3.2 금융위·금감원 유권해석 (2021.06.01)
> 출처: [CaseNote](https://casenote.kr/금융위원회·금융감독원/c83ec87490)

**결론**:
> "유사투자자문업자가 자동매매 프로그램을 연동하여 고객 계좌에서 자동으로 매매를 실행하는 방식은 **투자일임업에 해당**"

**판단 기준**: "투자판단 권한이 누구에게 있는가"

```
투자판단 → 고객  = 시스템매매 (합법, G6)
투자판단 → AI/서비스 = 투자일임업 (인가 필요)
```

**StockVision에 대한 함의**:
- AI가 자동으로 종목 선정 → 사용자 확인 없이 실행 = 위험
- 사용자가 조건을 사전 설정 + 시스템은 실행만 = 안전

### 3.3 젠포트 사례 (실증)
> 출처: [키움증권 젠포트 FAQ](https://www.kiwoom.com/inv/roboMarket/JP/faq)

**확인된 사실:**
- 키움증권 **공식 홈페이지**에서 직접 신청/해지
- 수수료: 가입 계좌 거래 시 0.1% (키움이 수취, 뉴지스탁에 배분)
- 제작사: 뉴지스탁 (기술) + 키움 (법적 라이선스 + 유통)
- NH투자증권, 이베스트증권도 연동

**핵심 의미**: 키움 라이선스 우산 아래 기술 제공사로 포지셔닝하는 것이 국내 합법 운영 경로.

**기술 아키텍처 (조사 결과):**

```
사용자 ─── AWS Windows VM (사용자별 1개) ───── 키움 서버
              ↑                    |
              |             키움 OpenAPI+ (로컬 COM)
     젠포트 서버 (전략 신호)    키움 HTS (백그라운드)
              |
     젠트레이더 (자동매매 엔진)
```

→ 사용자 자신의 PC가 아닌 **AWS Windows Server VM**에서 실행 가능.
→ 키움이 공식 매뉴얼에서 AWS 가상PC 설정 가이드 직접 제공.
→ VM에 키움 HTS + 젠트레이더 설치 → 24시간 클라우드 운영.
→ **핵심**: 각 사용자마다 독립 VM 1개 → API 키는 그 VM에만 → G5 제5조② 준수.

**비즈니스 모델 (조사 결과):**
- **젠마트** = 알고리즘 마켓플레이스. 개인이 전략 만들어 30~800만원에 판매.
- 뉴지스탁은 직접 알고리즘 설계/판매 X → **플랫폼 수수료** 수취.
- 키움 거래 수수료 0.1% (키움→뉴지스탁 배분).
- 2024년: 전략 480만 개, 연간 거래 5조원 규모.
- 뉴지스탁은 **투자자문업 등록 추진 중** (금융그룹 계열사 편입).

**StockVision에 대한 시사점:**
- 로컬 브릿지 = 사용자 PC뿐 아니라 **클라우드 Windows VM**도 가능.
- 각 사용자별 VM 비용이 발생하므로 유료 SaaS 수익모델 필요.
- 장기적으로는 마켓플레이스 모델(전략 공유/판매)이 젠포트의 핵심 차별점.

```
StockVision 기능별 법적 경로

읽기(조회)
└─ 코스콤 오픈플랫폼 등록 (문서 2)
   → 잔고·거래내역·자산구성 실시간 조회 가능

자동매매 실행

   → 사용자가 조건 직접 설정 (투자판단 = 사용자)
   → StockVision은 실행만

은행→증권 자금 이동
└─ 오픈뱅킹 경로 (문서 3)
   → 정기 투자금 자동 이체 가능

멀티유저 자동매매 SaaS (독자)
└─ ❌ 클라우드 SaaS 구조 = 명시적 약관 위반 (G5 제5조②③)
   → API 키 서버 보관 금지
   → 시세 데이터 서버 중계 금지
   → 유일한 합법 경로: 로컬 브릿지 아키텍처

로컬 브릿지 아키텍처 (합법 구조)
└─ ✅ 사용자 PC에 경량 클라이언트 설치
   → API 키는 로컬에만 저장 (서버 미전송)
   → StockVision 서버는 전략 신호(JSON)만 전송
   → 로컬 브릿지가 신호를 받아 키움 API 직접 호출
   → API 제한(초당 5건)도 사용자별 로컬에서 각자 부담
```

## 5. StockVision 운영 체크리스트

### 사용자 온보딩 필수 절차
- [ ] 키움 계좌 개설 (사용자 직접)
- [ ] 키움에서 시스템매매 신청 (사용자 직접)
- [ ] StockVision 시스템매매 위험고지 수락
- [ ] 코스콤 오픈플랫폼 통한 데이터 접근 동의 (연 1회 갱신)

### 서비스 설계 원칙
- ✅ 투자판단 = 사용자 (조건 설정은 반드시 사용자가)
- ✅ 돈은 사용자 키움 계좌에 (우리 통장 경유 없음)
- ✅ 손실 귀속 명시 (키움 위험고지 문구 준용)
- ❌ "AI가 알아서 종목 선정 후 자동 실행" 표현 금지
- ❌ 수익률 보장 또는 수익 배분 금지

### 아키텍처 필수 사항 (G5 기반)
- ❌ API 키를 StockVision 서버에 저장 금지
- ❌ 서버에서 사용자 대신 키움 API 호출 금지
- ❌ 시세 데이터를 서버에서 수집 후 다수 사용자에게 중계 금지
- ✅ 사용자 PC에 로컬 브릿지 설치 (Python/Electron 등)
- ✅ 서버는 전략 신호 JSON만 브릿지에 전송
- ✅ 브릿지가 신호 받아 로컬에서 키움 API 직접 호출

### 미결 확인 사항
- [ ] 코스콤 오픈플랫폼 핀테크기업 등록 요건 및 심사 기준
- [ ] 로컬 브릿지 배포 방식 결정 (Python 스크립트 vs Electron 앱 vs 웹소켓 데몬)
- [ ] 키움 OpenAPI+가 Windows 전용이므로 사용자 환경 제약 공지 방법

## 6. StockVision Phase 3 전략 방향

### 6.1 키움 OpenAPI가 원래 누구를 위한 것인가?

키움 OpenAPI의 본래 설계 목적: **개인 퀀트 개발자가 자기 계좌를 자동화**하기 위한 것.
약관의 "개인 업무에 한정" 문구가 이를 반영.

**실제 사용 계층:**

| 사용자 유형 | 사용 목적 | 합법성 |
|---|---|---|
| 개인 퀀트 개발자 | 자기 계좌 자동화 (Python 스크립트) | ✅ 원래 목적 |
| 젠포트/젠트레이더 | 타인 계좌 자동화 SaaS (키움 공식 파트너) | ✅ 파트너십으로 허용 |
| 퀀트킹, 씽크풀 등 | 신호 제공형 서비스 (실행은 사용자 직접) | ✅ 유사투자자문업 |
| 백테스팅/데이터 수집 | 과거 데이터 분석 | ✅ 개인 용도 |
| 독립 멀티유저 SaaS | API 키 서버 저장 후 여러 사용자 자동화 | ❌ G5 위반 |

### 6.2 StockVision이 선택할 수 있는 경로

**경로 A: 개인 퀀트 도구 (Phase 3 단기)**
- 대상: 직접 설치하고 운영하는 기술 지향 사용자
- 구조: 로컬 브릿지 (사용자 PC 또는 본인 클라우드 VM)
- 제약: 사용자 환경 설정 필요, Windows 전용
- 적합 시점: 지금 바로 구현 가능

**경로 B: 사용자별 관리형 VM (Phase 3 중기)**
- 대상: 설정 복잡도를 낮추고 싶은 일반 사용자
- 구조: StockVision이 사용자별 AWS Windows VM 프로비저닝
- 제약: VM 비용 ($15~30/월/사용자), 사용자가 키움 HTS 설정은 직접
- 적합 시점: 유료 구독 모델 도입 후

**경로 C: 키움/증권사 공식 파트너십 (장기)**
- 대상: 일반 대중
- 구조: 젠포트처럼 증권사 공식 채널로 배포
- 제약: 협상 및 계약 필요, 투자자문업 등록 필요
- 적합 시점: 사용자 기반 확보 후

### 6.3 Phase 3 기능별 법적 위험도

| 기능 | 위험도 | 비고 |
|------|--------|------|
| 백테스팅 엔진 | ✅ 안전 | 실거래 없음 |
| 포트폴리오 시각화 | ✅ 안전 | 조회만 |
| 시장 지표 대시보드 (yfinance) | ✅ 안전 | 퍼블릭 데이터 |
| AI 전략 추천 (사용자 확인 후 실행) | ✅ 안전 | 투자판단 = 사용자 |
| 로컬 브릿지 자동 실행 | ✅ 안전 | API 키 로컬 |
| API 키 서버 저장 | ❌ 위반 | G5 제5조② |
| 서버에서 직접 주문 API 호출 | ❌ 위반 | G5 제5조② |
| AI 자율 종목 선정 + 자동 실행 | ⚠️ 위험 | 투자일임 리스크 |

## 7. 금융법적 포지션 완전 정리

> 자본시장법 기준. StockVision이 어떤 기능을 만들 때 어떤 포지션에 해당하는지 판단하기 위한 레퍼런스.

---

### 7.1 법적 포지션별 정의 및 요건

#### A. 투자일임업 (Investment Discretionary Management)

**정의**: 투자자로부터 금융투자상품에 대한 **투자판단의 전부 또는 일부를 일임받아**, 투자자별로 구분하여 금융투자상품을 취득·처분하는 업.

> 핵심: "일임" = 판단 권한 자체를 넘겨받음

**근거**: 자본시장법 제6조 제6항, 제7조 제5항

**허가 요건**: 금융위원회 **인가** (인가제) — 최소 자기자본 75억원, 심사 수개월 소요

**해당하는 케이스**:
- AI가 종목 선정 → 사용자 확인 없이 자동 실행
- 서비스가 리밸런싱 시점·비중을 자율 결정
- 유사투자자문업자가 자동매매 프로그램으로 고객 계좌를 실행 (2021 금융위 유권해석)

**StockVision 회피 조건**: 종목 선정·매수·매도 시점 판단이 반드시 사용자에게 있어야 함.

---

#### B. 투자자문업 (Investment Advisory Business)

**정의**: 금융투자상품의 가치 또는 투자판단(종목, 매수·매도 여부, 투자 비중 등)에 관하여 **특정 투자자를 상대로** 자문에 응하는 업.

> 핵심: "특정 투자자" = 개인화된 자문

**근거**: 자본시장법 제6조 제5항, 제7조 제4항

**허가 요건**: 금융위원회 **등록** (등록제) — 최소 자기자본 5억원

**해당하는 케이스**:
- 개인 투자 성향 + 시장 상황을 결합해 "이 종목을 사야 합니다"라고 개인화 안내
- 사용자 포트폴리오를 분석해 특정 종목 추천 (LLM 기반 포함)
- "당신의 상황에 맞는 종목은 A, B, C입니다"

**StockVision 회피 조건**: LLM이 설명하되 종목 추천 표현 금지. "조건에 해당하는 종목입니다"(객관 사실)와 "이 종목을 매수하세요"(자문)는 다름.

---

#### C. 유사투자자문업 (Quasi-Investment Advisory Business)

**정의**: **불특정 다수인**을 대상으로 간행물·이메일·방송·인터넷 등을 통하여 금융투자상품의 가치나 투자판단에 관한 자문을 하는 업.

> 핵심: "불특정 다수" = 모든 사용자에게 동일한 내용

**근거**: 자본시장법 제101조

**허가 요건**: 금융위원회 **신고** (신고제, 가장 간소) — 최소 자기자본 2억5천만원, 결격 사유 없으면 수리

**해당하는 케이스**:
- 모든 사용자에게 동일한 "이번 주 주목 종목" 리스트 제공
- 시장 분석 뉴스레터에서 종목 언급
- 방송/유튜브에서 종목 추천

**StockVision 활용 가능 여지**: 개인화 없이 전체 사용자에게 동일하게 보이는 "시장 스크리너 결과" 형태라면 이 신고만으로 커버 가능.

---

#### D. 집합투자업 (Collective Investment Management)

**정의**: **2인 이상의 투자자로부터 금전을 모아** (펀드 구성) 공동 운용하는 업.

**근거**: 자본시장법 제6조 제4항

**허가 요건**: 금융위원회 **인가** — 최소 자기자본 80억원

**StockVision 해당 여부**: ❌ 없음 — 사용자 각자 자기 계좌에서 운용. 돈을 모으지 않음.

---

#### E. 시스템매매 (System Trading)

**정의**: 고객이 사전에 설정한 조건에 의하여 자동으로 매매주문을 내는 전산소프트웨어에 의한 매매거래.

**근거**: 키움증권 시스템매매 위험고지 (공식 거래 흐름)

**허가 요건**: ❌ **별도 허가 불필요** — 사용자가 키움에서 시스템매매 신청 후 이용

**핵심 조건**:
- 투자판단 주체 = 사용자 (조건을 사전에 직접 설정)
- 시스템은 실행만 담당
- 모든 손실은 사용자 귀속

**StockVision 핵심 경로**: 자동매매를 합법적으로 제공하는 유일한 무허가 경로.

---

#### F. 투자권유 (Investment Solicitation)

**정의**: 투자자를 상대로 금융투자상품의 매매 또는 투자자문을 **권유**하는 행위.

**근거**: 자본시장법 제9조 제4항

**업으로 하면**: 투자자문업(등록) 또는 유사투자자문업(신고) 필요

**표현 기준**:

| 표현 | 판단 |
|------|------|
| "현재 RSI가 27입니다" | ✅ 정보 제공 |
| "RSI가 낮아 과매도 구간입니다" | ✅ 설명 |
| "매수 고려해볼 만합니다" | ⚠️ 투자권유 경계 |
| "A 주식을 매수하세요" | ❌ 투자권유 |
| "AI가 이 종목을 추천합니다" | ❌ 투자권유 |

---

### 7.2 허가 없이 StockVision이 할 수 있는 최대치

> 핵심 원칙: **투자판단의 주체가 사용자**이면 허가 불필요.

```
                    투자판단 주체
                         │
             ┌───────────┴───────────┐
           사용자                  시스템/AI
             │                        │
      무허가 합법                  허가 필요
      (시스템매매)          (투자일임업/투자자문업)
```

**합법 기능 목록 (허가 없이)**:

| 기능 | 근거 |
|------|------|
| 주가 차트, 기술적 지표 표시 | 순수 정보 제공 |
| 백테스팅 엔진 | 실거래 없음, 도구 |
| 사용자 정의 규칙 자동매매 | 시스템매매 경로 (E) |
| 컨텍스트 변수를 조건 입력값으로 사용 | 판단 주체 = 사용자 |
| 조건 스크리너 (사용자 기준 기반) | 정보 표시 도구 |
| 포트폴리오 추적·손익 계산 | 내 데이터 집계 |
| LLM의 시장 상황 자연어 설명 | 설명 ≠ 권유 |
| 가격·지표 알림 (사용자 정의 조건) | 사용자가 조건 설정 |
| 가상 매매 시뮬레이션 | 실돈 없음 |

**절대 선 (허가 없이 불가)**:

| 기능 | 해당 포지션 |
|------|------------|
| AI가 종목 선정 + 자동 실행 | 투자일임업 |
| "이 종목을 사세요" 개인화 추천 | 투자자문업 |
| AI 추천 종목 목록 (서비스 명칭) | 유사투자자문업 신고 필요 |
| 여러 사용자 돈 모아 공동 운용 | 집합투자업 |

### 7.3 설계 판단 가이드

기능 아이디어가 생기면 이 3가지를 순서대로 확인:

```
1. 투자판단은 누가 하는가?
   → 사용자가 규칙/기준을 직접 설정했는가?
   → YES: 다음 단계
   → NO: 투자일임업 영역 — 설계 변경 필요

2. 표현이 권유인가, 설명인가?
   → "사야 합니다", "추천합니다", "유망합니다" 표현을 피하는가?
   → YES: 다음 단계
   → NO: 투자권유 — 표현 변경 필요

3. 불특정 다수 대상인가, 개인화인가?
   → 모든 사용자에게 동일 정보 → 유사투자자문업 신고 검토
   → 개인화 분석/권유 → 투자자문업 등록 필요
   → 도구 제공 (판단은 사용자) → 허가 불필요
```
