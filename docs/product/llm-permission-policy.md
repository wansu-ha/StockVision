# LLM 권한 정책

작성일: 2026-03-10
상태: 확정
관련 문서: `docs/product/product-direction-log.md`

## 1. 목적

이 문서는 StockVision에서 사용하는 LLM의 역할과 권한 범위를 명확히 정의하기 위한 정책 문서다.
핵심 목표는 다음 세 가지다.

- 사용자 경험을 해치지 않으면서도 법적/보안 리스크를 줄인다.
- LLM을 `직접 주문 주체`가 아니라 `전략/설명/보조 도구`로 제한한다.
- 프론트, 로컬 서버, 클라우드에서 어떤 LLM 동작이 허용되는지 공통 기준을 만든다.

## 2. 한 줄 원칙

- `LLM은 투자조언자나 주문 실행기가 아니라 전략 DSL 코파일럿이자 비서이다.`
- `LLM은 직접 주문하지 않는다.`
- `LLM이 만든 결과는 항상 초안, 설명, 계산, 비교, 준비 작업의 형태로만 실행 엔진에 전달된다.`

## 3. 전제

### 3.1 모델 제공 주체

현재 기본 가정은 `사용자 BYO LLM`이다.

- 사용자가 자신의 LLM API 키를 연결한다.
- 가능하면 LLM 호출은 `로컬 서버 -> 사용자 LLM 제공사`로 직접 보낸다.
- StockVision 클라우드는 기본적으로 사용자 LLM 키를 보관하지 않는다.

### 3.2 역할 분리

- 프론트: 입력 UI, 결과 표시, 워크플로우 연결
- 로컬 서버: 프롬프트 조립, 개인화 컨텍스트 결합, LLM 호출, 전략/주문 엔진 연계
- 클라우드: 동기화, 원격 접속, 리포트/운영 서비스

## 4. 허용 권한 레벨

### L0. 정보 / 설명

허용.

- 시장 요약
- 뉴스/공시 요약
- 전략 설명
- 주문 차단 사유 설명
- 백테스트 결과 설명
- 계좌 상태 요약

예시:

- `오늘 장에서 주의할 이벤트를 요약해줘`
- `이 전략의 손절 규칙을 설명해줘`
- `왜 이 주문이 차단됐는지 알려줘`

### L1. 코드 / DSL 초안 생성

허용.

- 자연어 -> 전략 DSL 변환
- 전략 템플릿 생성
- 문법 수정
- 전략 버전 차이 설명
- 백테스트용 설정 코드 생성

예시:

- `반도체 눌림목 전략 DSL 만들어줘`
- `현재 전략을 5분봉 버전으로 바꿔줘`
- `문법 오류를 고쳐줘`

### L2. 비활성 초안 저장 / 시뮬레이션 준비

허용.

- 전략 초안 저장
- 백테스트 준비
- 드라이런 준비
- 전략 비교표 생성
- 포트폴리오 규칙 초안 저장
- 리밸런싱 초안 계산

단, 이 단계의 결과는 `비활성 상태`여야 한다.

예시:

- `삼성전자용 전략 초안 3개를 만들어 저장해줘`
- `지난 6개월 백테스트 준비해줘`
- `목표 비중 기준 리밸런싱 초안 계산해줘`

### L3. 실행 준비 보조

조건부 허용.

- 주문 초안 계산
- 전략 무장(arm) 전 체크리스트 생성
- 활성화 후보 등록
- 외부 주문 감지 규칙 추가
- 사용자 승인이 필요한 액션의 준비 단계

단, 아래 조건이 필요하다.

- 항상 사용자의 명시적 승인 또는 별도 확인 단계를 둔다.
- 이 단계에서도 LLM이 직접 주문을 실행하지는 않는다.
- 자동전략의 활성화는 사용자가 저장한 규칙에 의해서만 진행된다.

예시:

- `삼성 100주 주문 초안 만들어줘`
- `오늘 엔진 시동 전에 점검해줘`
- `외부 주문 들어오면 정지 규칙 추가해줘`

## 5. 금지 권한 레벨

### X1. 직접 주문

금지.

- 시장가/지정가 주문 직접 실행
- 수량/가격/시점을 즉흥적으로 결정해서 주문 전송
- 대화 한 줄로 실주문 체결

예시:

- `지금 삼성 1000주 사`
- `하락하면 알아서 물타기해`

### X2. 개인별 투자조언

금지.

- 사용자 계좌, 보유종목, 성향을 읽고 `당신에게 맞는 종목/전략/비중`을 추천
- 특정 종목에 대해 `지금 사라 / 지금 팔아라`라고 말함
- 개인별 최적 포트폴리오 비중을 비서가 판단하여 제시

예시:

- `내 계좌 보면 지금 뭘 사야 해?`
- `삼성에 제일 좋은 전략 찾아줘`
- `내 포트폴리오를 최적으로 재분배해줘`

### X3. 자동 적용 / 자동 활성화

금지.

- LLM이 생성한 전략을 사용자 검토 없이 활성화
- LLM이 생성한 리밸런싱 방안을 자동 반영
- LLM 판단을 기반으로 엔진 설정을 몰래 변경

예시:

- `좋아 보이는 전략으로 알아서 갈아타`
- `시장 상황에 맞게 비중 자동 조절해`

## 6. 안전한 대체 작업으로의 변환 원칙

LLM은 금지 요청을 단순 거절하지 않고, 가장 가까운 안전한 작업으로 변환한다.

### 변환 예시

- `삼성 좋은 전략 적용해줘`
  - `삼성전자용 전략 초안 3개 생성 -> 비교표 작성 -> 1위 초안 비활성 저장`

- `오늘 뭐 사?`
  - `현재 활성 전략 기준 후보 목록 + 조건 충족률 + 리스크 상태 표시`

- `포트폴리오 재분배해줘`
  - `저장된 목표 비중 기준 드리프트 계산 + 리밸런싱 초안 계산`

- `삼성 100주 사`
  - `주문 초안 계산 + 잔고/손실 한도/중복 주문 점검`

## 7. 데이터 사용 원칙

### 로컬 우선

- 보유 종목, 전략 상세, 손익, 원장 데이터는 기본적으로 로컬 우선으로 처리한다.
- LLM 호출에 필요한 최소 데이터만 넘긴다.

### 클라우드 LLM 사용 시 주의

- StockVision이 호스팅하는 LLM 또는 외부 LLM에 사용자 데이터를 보내는 경우,
  개인정보 처리 목적, 위탁/제3자 제공 여부, 국외 이전 여부를 별도로 검토해야 한다.
- 가능하면 초기에는 BYO LLM + 로컬 호출을 기본값으로 둔다.

## 8. 제품 문구 원칙

### 권장 문구

- `전략 초안을 생성해드릴게요`
- `규칙을 DSL로 변환해드릴게요`
- `비활성 초안으로 저장해둘게요`
- `백테스트 준비를 해둘게요`
- `주문 초안을 계산해드릴게요`

### 피할 문구

- `당신에게 맞는 전략을 추천합니다`
- `오늘은 이 종목을 사세요`
- `내가 알아서 포트폴리오를 최적화할게요`
- `가장 좋은 전략으로 바로 적용할게요`

## 9. 제품 기능별 허용 여부 표

| 기능 | 허용 여부 | 조건 |
|---|---|---|
| 뉴스/공시 요약 | 허용 | 설명/요약 중심 |
| 자연어 -> DSL 변환 | 허용 | 초안 생성 |
| 전략 문법 수정 | 허용 | 코드 보조 |
| 백테스트 설정 생성 | 허용 | 실행 전 단계 |
| 전략 비교표 생성 | 허용 | 사용자 기준 기반 |
| 비활성 전략 저장 | 허용 | 활성화 금지 |
| 주문 초안 계산 | 조건부 허용 | 사용자 승인 전 단계 |
| 엔진 시동 체크리스트 | 조건부 허용 | 무장 전 점검 |
| 직접 주문 실행 | 금지 | 항상 금지 |
| 개인별 종목 추천 | 금지 | 투자자문 리스크 |
| 개인별 포트폴리오 추천 | 금지 | 투자자문 리스크 |
| 자동 리밸런싱 적용 | 금지 | 투자일임 리스크 |
| LLM 전략 자동 활성화 | 금지 | 승인 우회 금지 |

## 10. 결론

StockVision의 LLM은 `무엇을 살지 결정하는 AI`가 아니라,
`사용자가 정한 방향을 전략 코드, 설명, 초안, 검토 작업으로 구체화하는 도구`로 설계한다.

즉,

- `LLM은 추천하지 않고 초안을 만든다.`
- `LLM은 주문하지 않고 준비한다.`
- `LLM은 판단을 대신하지 않고 이해와 실행 준비를 돕는다.`

## 11. 구현 정책 (2026-03-17 확정)

### 금지 권한은 코드 하드코딩
X1(직접 주문), X2(개인별 투자조언), X3(자동 활성화)는 설정으로 해제할 수 없다.
아키텍처 수준에서 LLM → 주문 실행 경로를 차단한다.
사용자 config에 `max_permission_level` 같은 옵션을 제공하지 않는다.

### 사용자 설정 가능 범위
`~/.stockvision/config.json`에 아래만 노출:

```json
{
  "llm": {
    "provider": "openai",
    "api_key_ref": "keyring",
    "daily_call_limit": 50
  }
}
```

이유: "가능하게 만든 것" 자체가 금투법상 투자일임 리스크. 면책 고지로도 불충분.
