## 2. DSL v2 설계 결정 기록

이번 세션에서 내린 핵심 설계 결정들:

| 결정 | 이유 |
|------|------|
| DSL은 IR, 차별화는 AI+UX+투명성 | DSL 문법으로는 EasyLanguage/Pine과 수렴. 차별화는 위(AI 생성, 카드 UI)와 아래(한국 특화, 로컬 실행)에서 나온다 |
| (조건→행동) × N 플랫 규칙 | 매수/매도/청산 블록 분리 폐지. 대조건/상태블록 폐지. 순서 = 우선순위 |
| 수식어 폐지 → 함수 통합 | [1회],[래치],[유지N봉],[N봉내] → 횟수(), 연속(), 실행횟수 |
| @ 접두어 제거 | 상수와 함수가 같은 네임스페이스. 이름 = 값은 상수, 변하는 건 함수 |
| 무인자 함수 괄호 선택 | 골든크로스 = 골든크로스(). "이름 쓰면 값이 나오는 것" 통일 |
| 달성+카운트 → 횟수 통합 | 횟수(조건, 보유봉) >= 1 로 래치 대체. 함수 3개로 축소 |
| BETWEEN P2→P1 | 레인지 조건 빈번, syntactic sugar로 가치 충분 |
| 매도 나머지 = 매도 전량 별칭 | 사이클당 1개 실행 원칙과 양립 |
| 전량 매도 > 부분 매도 > 매수 | 손절이 1분 늦어지는 것 방지 |
| 보유봉 초기값 = 1 | 0이면 횟수(조건, 보유봉)이 첫 봉에서 항상 0 |

## 6. AI 자연어 → DSL 생성 — 다음 작업 방향

### 결정된 사항 (이번 세션 논의)

- **운영자 API 제공** (BYO LLM 아님) — 비용은 요청당 ~$0.01~0.05
- **무료: 시간당 N회 제한**, Pro: 무제한
- **구조**: 프론트 → 클라우드 서버 → Claude API → DSL 반환 → parse_v2() 검증 → 프론트 표시

### 필요한 것

1. **프롬프트 설계** — v2 DSL 문법 레퍼런스 + 예시를 포함한 시스템 프롬프트
2. **클라우드 API 엔드포인트** — POST /api/v1/ai/generate-strategy
3. **프론트 UI** — 자연어 입력 → "생성 중..." → DSL 결과 → 카드 UI 표시 → 수정/저장
4. **검증 루프** — parse_v2() 실패 시 AI에 에러 전달 → 재생성 (최대 3회)
5. **사용량 제한** — 무료 사용자 시간당 N회 (Redis counter 또는 DB)
6. **대화형 수정** — "트레일링 추가해줘" → 기존 DSL + 요청 → 수정된 DSL (코파일럿)
