# 원격 권한 모델 표

작성일: 2026-03-10
상태: 초안
관련 문서:
- `docs/product/product-direction-log.md`
- `docs/product/llm-permission-policy.md`

## 1. 목적

이 문서는 StockVision의 접속 환경별 권한 범위를 명확히 정의하기 위한 문서다.
핵심 목표는 다음과 같다.

- `조회는 넓게`, `실행은 좁게` 원칙을 제품에 반영한다.
- 같은 PC, 모바일, 다른 PC, 메신저 등 접속 채널별 권한 차이를 분명히 한다.
- 킬스위치와 위험 차단은 어디서나 쉽게, 실행 재개는 더 어렵게 설계한다.

## 2. 핵심 원칙

- `실주문은 항상 로컬 서버에서만 실행된다.`
- `클라우드는 중계, 인증, 동기화, 리포트 역할만 맡는다.`
- `정지는 쉽게, 재개는 어렵게` 설계한다.
- `원격 자연어 즉시 주문`은 자동전략 주문보다 더 엄격하게 다룬다.
- `자동 전략 주문`과 `원격 수동 주문`은 같은 권한으로 취급하지 않는다.

## 3. 접속 환경 정의

### A. 같은 PC 로컬 모드

- 프론트가 같은 PC의 로컬 서버에 직접 접속
- 가장 강한 권한을 가질 수 있는 환경
- 사용자 설정과 엔진 운영의 기준 환경

### B. 원격 앱/웹 모드 (Pro)

- 모바일 또는 다른 PC에서 StockVision 웹/앱으로 로그인
- 클라우드 릴레이를 통해 로컬 서버와 통신
- 로컬 서버가 온라인일 때 실시간 정보 접근 가능

### C. 메신저 모드 (선택 채널)

- 텔레그램 등 외부 채널을 통한 간단한 질의/알림 수신
- 가벼운 조회와 초안 요청에 적합
- 구조적 설정/승인 UX는 프론트보다 약함

### D. 로컬 서버 오프라인 원격 모드

- 사용자는 원격 접속 중이지만 로컬 서버가 꺼져 있음
- 마지막 동기화 정보만 사용 가능
- 실시간 조회와 실행 계열은 제한됨

## 4. 권한 구분

### 4.1 조회

- 계좌 요약
- 전략 상태
- 최근 브리핑/리포트
- 최근 경고/로그
- 외부 주문 감지 내역

### 4.2 설정

설정은 두 단계로 나눈다.

- `일반 설정`
  - 관심 종목
  - 비서 성향
  - 브리핑 스타일
  - 알림 채널
  - 보기 옵션

- `민감 설정`
  - 브로커 키 등록/삭제
  - LLM 키 등록/삭제
  - 손실 한도
  - 자동 실행 전략 활성화/비활성화
  - 주문 허용 범위
  - 외부 주문 감지 시 자동 정지 여부

### 4.3 주문 초안

- 수량/가격/예상 금액 계산
- 잔고, 중복 주문, 손실 한도 점검
- 초안 저장

### 4.4 실주문

실주문은 두 종류로 나눈다.

- `자동 전략 주문`
  - 사용자가 미리 저장하고 승인한 전략이 조건 만족 시 자동 실행

- `원격 수동 주문`
  - 사용자가 대화나 UI에서 즉시 주문을 요청하는 경우

### 4.5 킬스위치

- 즉시 정지
- 신규 주문만 정지
- 손실 잠금 해제 해제 여부
- 엔진 재개/무장(arm)

## 6. 세부 정책

### 6.1 조회

기본 원칙:

- 조회는 가능한 넓게 열어준다.
- 단, 로컬 서버가 오프라인이면 마지막 동기화 기준임을 분명히 표시한다.

권장 정책:

- 같은 PC: 전체 조회 가능
- 원격 앱/웹: 전체 조회 가능
- 메신저: 요약형 조회만 제공
- 오프라인 원격: 실시간 표시 금지, 마지막 동기화 시간 명시

### 6.2 설정

일반 설정은 원격에서도 비교적 넓게 허용할 수 있다.
민감 설정은 더 좁게 가져간다.

권장 정책:

- 일반 설정: 원격 앱/웹 허용
- 민감 설정: 로컬 우선, 원격은 추가 확인 또는 제한적 허용
- 키 등록/삭제: 원칙적으로 같은 PC 로컬 모드만

### 6.3 주문 초안

권장 정책:

- 같은 PC: 허용
- 원격 앱/웹: 허용
- 메신저: 단순한 형태만 허용
- 오프라인 원격: 금지

이유:

- 주문 초안은 실행이 아니라 준비 단계이므로 사용자 경험상 가치가 크다.
- 다만 실시간 계좌 상태가 필요하므로 로컬 서버 온라인이 중요하다.

### 6.4 실주문

#### 자동 전략 주문

권장 정책:

- 사용자가 미리 전략을 저장하고 승인했을 것
- 손실 한도, 종목당 한도, 총 주문금액 상한이 설정되어 있을 것
- 엔진이 무장(arm) 상태일 것
- 실제 실행은 계속 로컬 서버가 담당할 것

#### 원격 수동 실주문

권장 정책:

- 초반 제품에서는 원칙적으로 금지 또는 매우 제한적으로만 허용
- 허용하더라도 별도 강한 확인이 필요
- 메신저에서는 금지
- 같은 PC 로컬 모드 외 채널에서는 매우 보수적으로 다룰 것

## 7. 킬스위치 정책

### 정지

정지는 넓게 허용한다.

- 같은 PC: 허용
- 원격 앱/웹: 허용
- 메신저: 허용 가능
- 오프라인 원격: 클라우드에 stop 요청 저장 후 로컬 서버 복귀 시 즉시 적용하는 방식 검토 가능

### 재개 / 무장

재개는 좁게 허용한다.

- 같은 PC: 허용
- 원격 앱/웹: 추가 확인이 있을 때만 조건부 허용
- 메신저: 금지
- 오프라인 원격: 금지

## 8. 권장 확인 규칙

| 행동 | 기본 확인 수준 |
|---|---|
| 조회 | 추가 확인 없음 |
| 일반 설정 변경 | 저장 전 1회 확인 |
| 민감 설정 변경 | 2단계 확인 |
| 주문 초안 생성 | 추가 확인 없음 |
| 자동 전략 활성화 | 저장 + 활성화 전 확인 |
| 원격 수동 실주문 | 강한 추가 확인 또는 초기 금지 |
| 킬스위치 정지 | 즉시 실행 |
| 엔진 재개 / 무장 | 추가 확인 필요 |

## 9. 제품 UX 문장 예시

- `조회는 어디서나, 실행은 로컬에서`
- `정지는 쉽게, 재개는 신중하게`
- `원격에서도 같은 비서를 쓰되, 위험한 실행은 더 엄격하게 관리합니다`

## 10. 초기 권장안

초기 출시 기준으로는 아래 구성이 가장 안전하다.

- 같은 PC 로컬 모드: 전체 기능 허용
- 원격 앱/웹 모드: 조회, 일반 설정, 주문 초안, 킬스위치 정지 허용
- 메신저 모드: 요약 조회, 경고 확인, 간단 질의, 초안 요청 정도만 허용
- 원격 수동 실주문: 초기에는 금지
- 자동 전략 주문: 사용자가 로컬에서 무장한 전략만 로컬 서버가 자동 실행

## 11. 한 줄 정리

StockVision의 원격 권한 모델은
`모든 곳에서 같은 비서를 쓰되, 실행 권한만 로컬 중심으로 끝까지 좁게 유지하는 구조`로 설계한다.
## 부록: 엔진 무장(arm) 세션 원칙

자동 전략 주문은 `매 주문 승인` 방식이 아니라 `엔진 무장 세션` 안에서만 허용한다.

핵심 원칙:

- 조회는 넓게 허용
- 정지는 쉽게 허용
- 재개와 무장은 더 어렵게 허용
- 원격 수동 즉시 주문은 자동전략 주문과 같은 권한으로 취급하지 않음
- 장 종료, 킬스위치, 손실 락, 보안 사고 의심 시 세션은 자동 해제 또는 중단

상세 규칙은 `spec/system-trader/spec.md`를 따른다.
