# 무료 / Pro / Team 플랜 초안

작성일: 2026-03-10
상태: 초안
관련 문서: `docs/product/product-direction-log.md`

## 1. 목적

이 문서는 StockVision의 초기 유료화 방향을 정리하기 위한 플랜 초안이다.
핵심 방향은 `무료 오픈소스 트레이딩 엔진 + 유료 개인 비서 클라우드`이며,
요금은 거래량이나 투자 성과가 아니라 `편의`, `지속성`, `운영 서비스`에 대해 받는다.

## 2. 플랜 설계 원칙

1. 신뢰와 안전에 필요한 핵심 기능은 무료 또는 오픈소스로 제공한다.
2. 유료 기능은 사용자의 시간을 아껴주는 `클라우드 편의 기능` 중심으로 설계한다.
3. 실제 매수/매도 판단과 실행은 계속 로컬 서버 중심으로 유지한다.
4. LLM 비용은 원칙적으로 `사용자 BYO LLM` 기준이며, 별도 과금과 혼동하지 않는다.
5. 사용자는 언제든 전략과 데이터를 export할 수 있어야 한다.
6. 기본 브로커 연결, 안전장치, 로컬 실행 엔진은 유료 잠금 대상으로 두지 않는다.

## 2.1 BYO LLM 관련 원칙

- 사용자 AI API 키 연결 자체는 무료/오픈소스 영역으로 본다.
- 오픈소스 공개 시 `모델 연결`과 `기본 프롬프트 실행`은 누구나 복제 가능하다고 가정한다.
- 따라서 `내 LLM 연결` 자체를 유료 기능으로 삼지 않는다.
- 유료 가치는 개인화 메모리, 관리형 컨텍스트, 브리핑/알림 전달, 버전 관리, 협업 기능에서 만든다.
- 프롬프트 자체는 중요하지만, 단독으로는 강한 방어력이 되지 않는다.
## 2.2 클라우드 과금 원칙

- 오픈소스로도 구현 가능한 `로컬 기능 자체`에 과금하려고 하지 않는다.
- 우리가 실제로 운영하는 `클라우드 서버 기능`을 유료화의 중심으로 둔다.
- 예를 들면 전략 동기화, 백업, 원격 메모리, 브리핑 생성, 알림 전달, 관리형 컨텍스트 파이프라인이 여기에 해당한다.
- 같은 `알림`이라도 로컬 알림은 무료일 수 있고, 우리가 운영하는 메신저 전달 인프라는 유료가 될 수 있다.
- 퍼널이 필요하면 무료 영구 제공보다 `기간 제한 체험판`이 경계가 더 깔끔하다.
## 2.3 기억 과금 원칙

- `비서가 사용자를 잊지 않게 하는 최소 기억`은 무료에 가깝게 둔다.
- 사용자가 로그인하면 기본 프로필, 관심종목, 비서 성향, 핵심 설정은 그대로 복원되는 것이 자연스럽다.
- 유료는 `기억 그 자체`보다 `기억을 더 오래, 더 깊게, 더 편하게 쓰는 기능`에 둔다.
- 예를 들면 장기 메모리 검색, 회상 품질 강화, 버전 히스토리, 복구, 팀 공유 메모리, 관리형 브리핑이 유료 후보가 된다.
## 3. 대상 사용자

### 무료

- 혼자 써보는 개인 사용자
- 한 대의 PC에서 로컬 엔진을 돌리고 싶은 사용자
- 전략 실험과 백테스트를 먼저 해보고 싶은 사용자

### Pro

- 실제로 매일 쓰는 개인 투자자
- 브리핑, 알림, 백업, 동기화, 메신저 편의가 필요한 사용자
- 사용자 맞춤 비서 경험을 원하는 사용자

### Team

- 2인 이상이 전략과 리포트를 함께 관리하는 팀
- 소규모 리서치 팀, 지인 투자 스터디, 내부 운용 실험 조직
- 권한 분리, 공유 전략, 운영 기록이 필요한 사용자

## 5. 플랜별 상세 기능

### 5.1 무료

핵심 메시지:

> 혼자서도 충분히 시작할 수 있는 오픈소스 로컬 엔진

포함 기능:

- 로컬 서버 설치 및 실행
- 브로커 연동
- 전략 작성 및 로컬 실행
- 전략 DSL / 규칙 엔진
- 기본 백테스트
- 로컬 로그 저장 및 조회
- 킬 스위치, 손실 제한, 중복 주문 방지 등 기본 안전장치
- 사용자 AI API 키 연결 및 사용자 LLM 직접 연결
- 기본 로컬 대시보드
- 전략 import/export
- 비서 기본 메모리 동기화

제한 사항:

- 전략 클라우드 동기화 없음 (체험판 제외)
- 다기기 자동 연동 없음
- 메신저 알림 없음
- 전략 버전 히스토리 없음 또는 최소 제공
- 관리형 브리핑/리포트 없음

무료 플랜의 역할:

- 신뢰 구축
- 설치 저변 확대
- 커뮤니티 형성
- 브로커/LLM 연결 장벽 제거

### 5.2 Pro

핵심 메시지:

> 매일 쓰는 개인 투자자를 위한 개인 비서 플랜

포함 기능:

- 무료 플랜의 모든 기능
- 전략 클라우드 동기화
- 전략 버전 히스토리 및 롤백 전체 제공
- 클라우드 백업
- 아침 브리핑
- 장중 리스크 알림
- 장마감 요약 리포트
- 메신저 알림 연동
- 비서 메모리 동기화 (기본 프로필, 관심종목, 비서 설정은 무료 기반)
- 다기기 사용
- 개인용 프롬프트/비서 설정 저장
- 프롬프트 버전 관리 및 개인화 설정 동기화
- 관리형 뉴스/공시/요약 컨텍스트
- 업데이트 채널 및 기본 운영 지원

Pro 플랜의 핵심 가치:

- `저장`이 아니라 `매일 챙겨주는 운영 경험`
- `내가 매번 켜서 확인하지 않아도 되는 편의`
- `PC를 바꿔도 이어지는 지속성`

### 5.3 Team

핵심 메시지:

> 전략과 리포트를 함께 운영하는 팀을 위한 협업 플랜

포함 기능:

- Pro 플랜의 모든 기능
- 팀 워크스페이스
- 공유 전략 폴더
- 권한 관리(읽기/편집/실행 승인)
- 팀용 비서 메모리 및 공용 플레이북
- 팀 활동 로그
- 전략 변경 이력 감사 로그
- 팀 단위 리포트
- 공유 메신저 채널 알림
- 좌석 관리
- 우선 지원

Team 플랜의 핵심 가치:

- `협업`
- `권한 분리`
- `운영 기록`
- `팀 단위 지속성`

## 6. 가격 가설

아직 확정 가격은 아니며, 아래는 검토용 범위다.

### 무료

- 0원

### Pro

권장 시작 구간:

- 월 12,900원 ~ 19,900원

추천 시작값:

- 월 14,900원 또는 월 16,900원

이유:

- 개인 투자자가 `브리핑 + 알림 + 동기화`에 지불 가능한 심리선
- 너무 낮으면 운영 비용과 제품 가치가 약해 보일 수 있음
- 너무 높으면 아직 초기 제품에서 진입 장벽이 커짐

### Team

권장 시작 구간:

- 월 69,000원 ~ 149,000원

추천 구조:

- 기본 3석 포함 월 79,000원
- 추가 좌석당 월 15,000원 ~ 25,000원

이유:

- 팀 플랜의 핵심은 단순 사용자 수가 아니라 `공유/권한/감사` 가치
- 좌석형 구조가 확장성과 설명력이 좋음

## 7. 출시 전략 제안

### 1단계

- 무료 + Pro만 먼저 출시
- Team은 실제 협업 수요가 보일 때 출시

이유:

- 초기에 가장 중요한 것은 `개인 사용자가 매일 쓰게 만드는 것`
- Team은 기능 폭이 넓고 지원 부담이 커서 너무 빨리 열면 집중력이 흐려질 수 있음

### 2단계

- Pro에서 메신저 알림과 아침 브리핑을 핵심 유료 포인트로 검증
- 전략 동기화와 버전 히스토리를 함께 묶어 `계속 쓰는 이유`를 강화

### 3단계

- 실제 협업 사용자 패턴이 보이면 Team 출시
- 이때 권한, 감사 로그, 공유 채널, 승인 플로우를 전면에 둠

## 8. 추천 유료 포인트 우선순위

1. 전략 클라우드 동기화
2. 전략 버전 히스토리/롤백 전체 제공
3. 아침 브리핑 + 장중 알림
4. 비서 메모리 클라우드 동기화
5. 관리형 뉴스/공시/요약 컨텍스트

이 순서가 좋은 이유:

- 처음 돈을 내는 이유를 가장 쉽게 설명할 수 있음
- 로컬 오픈소스와 충돌하지 않음
- 서버 비용 대비 사용자 체감 가치가 큼

## 9. 유료로 팔면 안 되는 것

- 기본 브로커 연결
- 기본 로컬 엔진 실행
- 핵심 안전장치
- 전략 export 기능
- 사용자 자신의 LLM 연결 권한

이 항목들을 잠그면 신뢰를 잃고,
오픈소스 기반 제품으로서의 매력이 크게 약해질 수 있다.

## 10. 추가 수익화 후보

### 가능성 높음

- 연간 결제 할인
- 고급 팀 기능
- 고급 리포트 패키지
- 관리형 컨텍스트 팩
- 향후 전략 마켓/템플릿 스토어

### 주의 필요

- 서버가 직접 투자 판단을 내리는 구독 모델
- 수익률 연동 수수료
- 과도하게 공격적인 자동매매 프리미엄 판매
