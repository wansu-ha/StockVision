# Support Policy (초안)

이 문서는 StockVision 오픈소스 프로젝트의 지원 범위와 응답 원칙 초안입니다.
이 프로젝트는 1인 유지관리 체계로 운영되며, 모든 지원은 best-effort 기준으로 제공됩니다.

## 지원 채널

기본 채널은 아래와 같습니다.

- 버그 리포트: GitHub Issues
- 기능 제안: GitHub Issues 또는 Discussions
- 보안 취약점: `SECURITY.md`의 비공개 채널
- 일반 사용 문의: README 또는 문서 우선

## 응답 원칙

- 모든 이슈와 PR에 대한 응답을 보장하지 않습니다.
- 일반 triage는 주 1회 수준을 목표로 합니다.
- 응답이 늦더라도 방치가 아니라 우선순위 조정일 수 있습니다.
- 유지관리자 일정, 건강, 본업 상황에 따라 대응 속도는 달라질 수 있습니다.

## 우선 지원 범위

아래 항목은 상대적으로 우선순위가 높습니다.

- 최신 안정 버전에서 재현되는 버그
- 설치와 실행 자체를 막는 문제
- 로컬 엔진 안전성과 관련된 문제
- 인증, 권한, 토큰, localhost 보안 문제
- 문서와 실제 동작이 심하게 어긋나는 문제

## 우선순위가 낮은 항목

아래 항목은 답변이나 수정이 늦을 수 있습니다.

- 오래된 버전의 문제
- 개별 사용자 환경에 강하게 의존하는 문제
- 대규모 UI 재설계 제안
- 특정 종목/전략 성과 관련 질문
- 유지관리 비용이 큰 신규 브로커 추가 요청

## 지원하지 않는 항목

아래는 기본적으로 지원 범위 밖입니다.

- 투자 자문, 종목 추천, 수익률 상담
- 개별 전략의 수익성 검토
- 사용자의 API Key 또는 계좌 정보를 받아 직접 디버깅
- 포크/개조판에 대한 호환성 지원
- 구버전 무기한 지원
- 사용자 PC 원격 접속을 전제로 한 문제 해결

## 이슈를 열기 전에 확인해 주세요

가능하면 아래를 먼저 확인해 주세요.

- 최신 문서와 README를 읽었는지
- 최신 안정 버전에서 재현되는지
- 민감정보를 제거했는지
- 이미 같은 이슈가 있는지
- 로컬 설정 문제인지, 클라우드 문제인지 구분했는지

## 좋은 버그 리포트에 포함될 내용

- OS 버전
- 앱 버전
- 브로커 종류
- 재현 단계
- 기대 결과
- 실제 결과
- 에러 로그 또는 스크린샷
- 민감정보 제거 여부

## 이슈 종료 원칙

유지관리자는 아래 경우 이슈를 닫을 수 있습니다.

- 재현 정보가 부족하고 추가 정보가 장기간 오지 않는 경우
- 문서에 이미 답이 있는 경우
- 프로젝트 방향과 맞지 않는 기능 요청인 경우
- 보안 이슈를 공개로 게시한 경우
- 공격적이거나 무례한 커뮤니케이션이 있는 경우

## PR 지원 원칙

- PR 리뷰와 병합은 보장되지 않습니다.
- 프로젝트 방향과 유지관리 가능성을 우선합니다.
- 테스트가 없거나 리스크가 큰 변경은 보류될 수 있습니다.
- 보안, 법적 리스크, 증권사 약관 이슈가 보이면 병합하지 않을 수 있습니다.

## 상용/운영 지원과의 구분

오픈소스 저장소의 이슈 응답은 커뮤니티 지원입니다.
향후 운영, 배포, 조직 맞춤 지원이 필요하면 별도 유료 범위로 분리될 수 있습니다.
현재 문서는 무료 무제한 지원을 약속하지 않습니다.

## 한 줄 원칙

이 프로젝트의 지원은 `가능한 범위에서 돕는다`가 기본이며, `모든 문제를 해결해 준다`는 약속은 아닙니다.
