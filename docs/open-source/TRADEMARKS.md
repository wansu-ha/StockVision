# StockVision 상표 및 브랜드 사용 정책 초안

이 문서는 StockVision 프로젝트의 이름, 로고, 아이콘, 브랜드 표시에 대한 사용 원칙 초안입니다.
코드 라이선스와 상표 정책은 서로 다른 문서이며, 코드가 오픈소스라고 해서 브랜드 사용 권한이 자동으로 주어지지는 않습니다.

## 1. 대상

이 정책에서 다루는 표시는 아래를 포함합니다.

- `StockVision` 이름
- 공식 로고, 워드마크, 앱 아이콘
- 유지관리자가 공식 제품명으로 지정한 명칭

## 2. 기본 원칙

1. 코드에는 `MPL-2.0`이 적용될 수 있습니다.
2. 이름과 로고는 별도의 권리이며, 자동으로 허가되지 않습니다.
3. 사용자는 사실을 설명하는 범위에서 프로젝트 이름을 언급할 수 있습니다.
4. 수정본, 포크, 재배포본은 공식판으로 오인되지 않게 표시해야 합니다.

## 3. 별도 허가 없이 가능한 사용

아래 사용은 사실 설명 목적이라면 별도 허가 없이 가능합니다.

- `이 프로젝트는 StockVision을 기반으로 한다`와 같은 출처 표기
- 버그 리포트, 기사, 블로그, 강연 자료에서 프로젝트를 지칭하는 사용
- 공식 저장소, 공식 릴리스, 공식 문서를 링크하기 위한 사용
- 수정하지 않은 공식 배포본을 그대로 전달하면서 원저작자와 출처를 명확히 표시하는 사용

## 4. 별도 허가가 필요한 사용

아래 사용은 유지관리자의 사전 허가가 필요합니다.

- `StockVision`을 제품명, 회사명, 서비스명, 패키지명, 앱 이름으로 사용하는 경우
- 공식 로고나 아이콘을 포크, 파생 서비스, 상용 서비스의 대표 표식으로 사용하는 경우
- `공식`, `인증`, `파트너`, `승인`처럼 오인을 유도하는 마케팅 표현
- 상표가 포함된 도메인, 소셜 계정, 앱스토어 등록명 사용
- 유료 지원, 컨설팅, 호스팅 서비스를 공식 서비스처럼 보이게 홍보하는 경우

## 5. 포크와 수정 배포본에 대한 규칙

수정된 포크나 파생 배포본은 아래 원칙을 따라야 합니다.

1. 원본과 구별되는 별도 이름을 사용합니다.
2. 공식 로고와 아이콘을 그대로 사용하지 않습니다.
3. 설명 문구에 `StockVision 기반`, `StockVision fork`처럼 관계를 사실대로 표시할 수는 있습니다.
4. 사용자가 공식 유지관리자가 배포한 버전으로 오해하지 않게 해야 합니다.

## 6. 허용되지 않는 사용 예시

- 수정한 포크를 `StockVision`이라는 동일한 이름으로 재배포하는 행위
- 공식 로고를 그대로 넣고 상용 서비스를 운영하는 행위
- 공식 유지관리자의 승인 없이 `StockVision 공식 한국 파트너`처럼 표기하는 행위
- 사용자 지원, 호환성, 보안 검토를 공식 팀이 제공하는 것처럼 오인시키는 행위

## 7. 권리 유보

이 정책에서 명시적으로 허용하지 않은 모든 상표 사용 권한은 보유자에게 유보됩니다.
코드에 대한 오픈소스 라이선스는 상표, 로고, 아이콘, 브랜드 자산의 사용 허가로 해석되지 않습니다.

## 8. 실제 적용 시 보완할 항목

최종 공개 전에 아래 항목을 채워 넣어야 합니다.

- 상표 보유 주체명
- 공식 문의 이메일 또는 연락 경로
- 공식 로고 파일 목록
- 공식 배포 채널 목록
- 상표 사용 허가 요청 절차
