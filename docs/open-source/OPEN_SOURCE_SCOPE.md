# StockVision 오픈소스 공개 범위 초안

이 문서는 StockVision이 `오픈소스`라고 말할 때 정확히 무엇을 뜻하는지 정리하기 위한 초안입니다.
이 문서는 공개 저장소와 공개 릴리스에만 적용됩니다.
명시적으로 공개되지 않은 코드, 문서, 서비스, 인프라는 이 문서의 대상이 아닙니다.

## 기본 원칙

1. 사용자가 자신의 PC에서 실제로 실행되는 로직은 최대한 검증 가능해야 합니다.
2. 키 보관 방식, 주문 안전장치, 로컬 실행 흐름은 투명성이 중요합니다.
3. 반대로 운영 자동화, 클라우드 관리 기능, 내부 연구 자산은 공개 대상과 분리합니다.
4. 오픈소스의 가치는 `코드 은닉`이 아니라 `신뢰 가능한 구조`에 둡니다.

## 제품 방향과의 정합성

이 문서는 제품 방향 로그의 아래 판단을 그대로 따른다는 전제를 둡니다.

- 무료 오픈소스 트레이딩 엔진 + 유료 개인 비서 클라우드
- 오픈소스는 실행 구조와 안전성에 대한 신뢰를 만든다
- 유료 가치는 저장, 동기화, 알림, 브리핑, 메모리, 운영 편의에 둔다
- 핵심 거래 기능은 클라우드가 없어도 로컬에서 동작할 수 있어야 한다

## 이 초안의 전제

현재 StockVision은 하나의 모노레포 안에 공개 후보와 비공개 후보가 섞여 있습니다.
따라서 이 문서는 `현재 저장소 전체를 그대로 공개한다`는 뜻이 아닙니다.

이 초안은 아래 둘 중 하나를 전제로 합니다.

- 공개 범위만 분리한 별도 공개 저장소를 운영한다.
- 현재 저장소에서 공개하지 않을 디렉터리와 파일을 제거한 뒤 공개한다.

## 공개 대상

아래 항목은 공개 저장소에 포함할 수 있는 1차 후보입니다.

### 코드

- `local_server/`
- `sv_core/`
- 로컬 상태 확인과 제어에 필요한 `frontend/` 코드
- 위 코드에 직접 대응하는 테스트
- 공개 가능한 설치 스크립트와 개발 보조 스크립트

### 문서

- 공개 가능한 설치 문서
- 공개 가능한 아키텍처 개요
- 공개 가능한 API/파일 포맷/프로토콜 문서
- 오픈소스 정책 문서와 기여 문서

### 기능 범위

- 로컬 실행 엔진
- 브로커 어댑터
- 전략 DSL과 평가 프레임워크
- 백테스트 러너
- 로컬 로그 및 상태 모니터링
- 손실 제한, 킬 스위치, 중복 주문 방지 같은 안전장치
- 사용자 로컬 환경에서 동작하는 BYO LLM 연결 레이어

## 비공개 대상

아래 항목은 공개 저장소에 포함하지 않는 것을 기본 원칙으로 합니다.

### 운영 코드와 서비스

- `cloud_server/`
- `backend/`
- 관리자 전용 기능
- 내부 운영 자동화
- 알림 라우팅, 전략 동기화, 버전 히스토리, 백업 같은 관리형 서비스

### 내부 문서와 검토 자료

- `docs/legal/`
- `docs/research/`
- 외부 공개용으로 정리되지 않은 내부 spec 및 메모
- 법률 검토 초안, 운영 정책 초안, 사고 대응 자료

### 비밀정보와 인프라

- API 키, 브로커 키, 서명키, 인증서
- 배포 파이프라인 구성
- 모니터링 설정, 운영 대시보드, 내부 헬스체크
- 관리자 계정, 운영용 seed 데이터, 내부 테스트 자격증명

### 권리 정리가 끝나지 않은 자산

- 재배포 권한이 불명확한 데이터셋
- 재배포 권한이 불명확한 모델 파일
- 상용 또는 별도 계약이 필요한 SDK/에셋
- 사용권 검토가 끝나지 않은 이미지, 폰트, 아이콘

## `MPL-2.0` 적용 단위

`MPL-2.0`은 공개 저장소에 실제로 포함되어 배포되는 파일에 적용됩니다.
비공개 저장소, 비공개 서비스, 내부 인프라에 자동으로 확장되지 않습니다.

공개 저장소에 포함된 파일을 수정해 재배포하는 경우, 그 수정 파일은 `MPL-2.0`에 따라 공개되어야 합니다.
하지만 별도 파일로 결합된 더 큰 작업물 전체가 자동으로 공개되는 것은 아닙니다.

## 사용자에게 약속하는 것

공개 저장소에서는 아래를 확인할 수 있어야 합니다.

- 로컬에서 어떤 판단과 실행 흐름이 동작하는지
- 브로커 연동이 어떤 안전장치를 거치는지
- 로컬 키 보관이 어떤 방식으로 이루어지는지
- 사용자가 스스로 수정하고 빌드하고 검증할 수 있는지

## 사용자에게 약속하지 않는 것

아래는 오픈소스 공개만으로 보장하지 않습니다.

- 클라우드 운영 내부 구현 전체 공개
- 관리형 알림/동기화/백업 서비스의 내부 코드 공개
- 공식 상표 사용 권한
- 투자 성과 보장 또는 법률상 책임 면제

## 공개 전 체크리스트

1. 공개 저장소에서 비공개 디렉터리가 완전히 제외되었는지 확인
2. 공개 파일의 제3자 라이선스와 재배포 권한 점검
3. 각 소스 파일에 `MPL-2.0` 고지 또는 SPDX 헤더 반영
4. 루트 `LICENSE`, `README`, `TRADEMARKS.md` 반영
5. 실행 파일 배포 시 소스 코드 접근 경로 고지
6. 비공개 서비스와 공개 저장소의 경계 문구 검토
