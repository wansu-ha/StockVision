## 주요 내용 요약

### 온보딩 플로우 (6단계)

| 순번 | 단계 | 소요 시간 | 목적 |
|------|------|---------|------|
| 0 | 환경 사전 검사 | 1분 | Windows, .NET 확인 |
| 1 | 계정 생성/로그인 | 2분 | 매직 링크 인증 (또는 무계정) |
| 2 | 위험고지 수락 | 3분 | 법적 책임 명확화 |
| 3 | 로컬 브릿지 설치 | 5분 | 1-클릭 설치 or 수동 설치 |
| 4 | API 키 설정 | 10분 | 키움 OpenAPI+ 설정 + 암호화 저장 |
| 5 | 연결 테스트 | 2분 | 서버 ↔ 브릿지 ↔ 키움 검증 |
| 6 | 첫 전략 만들기 | 5분 | 샘플 전략 선택 또는 직접 작성 |

**총 소요 시간**: 25분 (평균) ~ 35분 (수동 설치)

### 핵심 설계 원칙

1. **마찰 최소화**
   - 각 단계가 명확한 목적과 완료 기준을 가짐
   - "나중에 계속하기" 옵션으로 부분 완료 지원
   - 자동 검증 + 1-클릭 재시도

2. **보안 강화**
   - API 키는 **로컬에만 저장** (서버 미저장)
   - 클라이언트 측 암호화 (XChaCha20)
   - 위험고지 수락을 통한 법적 책임 명확화

3. **사용자 선택지**
   - 1-클릭 설치 (권장) vs 수동 설치 (고급 사용자)
   - 모의투자 (안전) vs 실거래 선택 가능
   - 샘플 전략 로드 vs 커스텀 작성

4. **진행도 추적**
   - 각 단계별 완료 상태 및 타임스탬프 저장
   - 이탈 사용자에 대한 리텔게팅 (이메일 리마인더)
   - 완료율 및 단계별 통계 수집 가능

### 기술 구현 범위

#### 백엔드 (FastAPI)
- `OnboardingProgress` 모델: 사용자별 진행 상태 추적
- 10개 엔드포인트: 환경 검사, 인증, 설정, 테스트 등
- 자동 재시도 로직 및 에러 복구

#### 프론트엔드 (React/TypeScript)
- 6개 Step 페이지 (Step0 ~ Step6)
- 진행도 바 + 네비게이션
- 로컬 저장소 암호화 (클라이언트 측)
- 상태 체크 스피너 및 오류 핸들링

#### 로컬 브릿지
- 헬스체크 API (`/health` 엔드포인트)
- 설치 감지 및 상태 보고
- WebSocket 재연결 로직

- 소요 시간 요약 (최소/평균/최대)

### 3. 단계별 상세 ✅
각 단계마다:
- **UI 목업**: ASCII 아트로 화면 표현
- **구현 로직**: 프론트엔드/백엔드 코드 예시
- **API 정의**: 요청/응답 형식 명시
- **데이터 흐름**: 단계별 처리 과정

#### Step 0: 환경 검사
- 자동 OS 감지 및 .NET Framework 확인
- 경고 메시지 및 대응 링크 제공

#### Step 1: 계정 생성
- **경로 A**: 매직 링크 (권장)
- **경로 B**: 무계정 UUID (선택)
- 기존 auth spec과 통합

#### Step 2: 위험고지
- 3개 필수 체크박스 (이해, 책임, 자산 보호)
- 법적 면책사항 및 다운로드 링크

#### Step 3: 브릿지 설치
- **옵션 A**: 1-클릭 자동 설치
- **옵션 B**: 수동 설치 (고급)
- **옵션 C**: Docker (클라우드, Phase 4)

#### Step 4: API 키 설정
- 클라이언트 측 암호화 (XChaCha20)
- 로컬 스토리지에만 저장
- 자동 검증

#### Step 5: 연결 테스트
- 3단계 병렬 테스트 (서버, 브릿지, 키움)
- 레이턴시 측정 및 상세 정보 표시

#### Step 6: 첫 전략
- 3가지 템플릿 제공 (RSI+EMA, MACD, 커스텀)
- 백테스팅 유도 (선택)
- 지금 시작 옵션

### 4. 진행 상태 추적 ✅

- 단계별 완료 여부 + 타임스탬프

### 5. 이탈 처리 ✅
