## 1. 목표

Proto A(리스트+슬라이드), Proto B(3컬럼 워크벤치) 프로토타이핑 결과를 반영하여,
**종목 중심 싱글뷰** 메인 화면의 UX 패턴을 확정한다.

핵심 설계 원칙:
- **싱글 페이지**: 사이드바 없음, `/` 하나에서 동작
- **종목 → 규칙**: 종목을 먼저 선택, 규칙은 종목의 하위 개념
- **점진적 공개**: 목록 → 아코디언 확장(미리보기) → 상세 뷰(전체 정보)

### 2.2 계좌 요약

헤더 바로 아래, 종목 리스트 위에 배치. 항상 표시.

| 항목 | 데이터 소스 |
|------|-----------|
| 증권사명 + 계좌번호 (마스킹) | 로컬 서버 → 브로커 |
| 총 평가금액 | 브로커 잔고 조회 |
| 주문가능금액 (예수금) | 브로커 잔고 조회 |
| 보유 종목 수 | 브로커 잔고 조회 |
| 일간 수익률 | 계산 (전일 대비) |
| 장 상태 (장전/장중/장후/휴장) | 시간 기반 + 브로커 API |

### 2.3 헤더

```
[ StockVision ● ]  [──── 종목 검색... ────]  [ 🔔¹ ] [ ⚙ ]
```

| 영역 | 내용 |
|------|------|
| 좌측 | 로고 + 상태 신호등 (§5) |
| 중앙 | 검색바 (§2.5 참조) |
| 우측 | 알림 벨 (§2.6) + 톱니바퀴 (§2.4) |

### 2.5 검색 오버레이

헤더 검색바에 입력 시, 페이지 전환 없이 오버레이로 결과 표시.

| 항목 | 동작 |
|------|------|
| 트리거 | 검색바 포커스 + 입력 (300ms 디바운스) |
| 데이터 | `cloudStocks.search()` (클라우드 서버 API) |
| 결과 표시 | 드롭다운 오버레이 (최대 10건) |
| 종목 선택 시 | 상세 뷰로 전환 (페이지 이동 없음) |
| 닫기 | 외부 클릭 또는 ESC |

> 기존 `StockSearch.tsx` 컴포넌트는 HeroUI + 라이트 테마 기반.
> 구현 시 Proto C 다크 테마에 맞게 재작성 필요. API 호출 로직(`cloudStocks.search`)은 재사용.

### 2.6 알림 드롭다운

알림 벨(🔔) 클릭 시 드롭다운.

```
┌─────────────────────────────┐
│ 알림                모두 읽음│
├─────────────────────────────┤
│ ● 삼성전자 매수 10주 체결    │
│   10:30                     │
├─────────────────────────────┤
│   SK하이닉스 RSI(14) 30 이하 │
│   10:15                     │
└─────────────────────────────┘
```

| 항목 | 설명 |
|------|------|
| 미읽음 | 파란 점(●) + 배경 하이라이트 |
| 배지 | 헤더 벨 아이콘에 미읽음 수 표시 |
| 알림 종류 | 체결, 규칙 트리거, 연결 상태 변경 |

## 3. 인터랙션 패턴

### 3.1 목록 뷰 → 아코디언 확장 → 상세 뷰

3단계 점진적 공개:

| 단계 | 트리거 | 결과 | 되돌리기 |
|------|--------|------|----------|
| 1. 목록 | 기본 상태 | 종목명, 현재가, 등락률, 규칙 수, 최근 체결 | — |
| 2. 확장 | 행 클릭 | 주요 지표 4개 미리보기 (아코디언) + "상세 보기 →" 텍스트 링크 | 같은 행 다시 클릭 |
| 3. 상세 | 오른쪽 `>` 버튼 **또는** "상세 보기 →" 클릭 | 목록 사라지고 상세 뷰 (전체 지표, 규칙, 체결 내역) | `← 목록` 버튼 |

**핵심 원칙**: 클릭 = 안전한 확장/축소 (되돌릴 수 있음), 명시적 버튼 = 뷰 전환 (의도적 액션)

**이중 진입점**: 상세 뷰로의 진입은 2가지 경로 제공 — 오른쪽 `>` 셰브론 (항상 보임) + 확장 영역 하단 "상세 보기 →" 텍스트 (확장 시 보임). `>` 버튼만으로는 용도를 모를 수 있으므로 텍스트로 보완.

### 3.2 시각적 피드백

**색상 체계**: UI 액센트는 `indigo` 계열 사용. 금융 색상(`red`=상승, `blue`=하락)과의 충돌 방지.

| 상태 | 시각적 표현 |
|------|------------|
| 호버 | 왼쪽 인디고 액센트 바 (`indigo-500/40`) + **하단 발광** (그라디언트 `via-indigo-500/30`) |
| 확장 | 왼쪽 인디고 액센트 바 (진하게, `indigo-500`) + 배경 하이라이트 |
| 상세 버튼 (평소) | 어두운 배경 (`gray-800/30`), 회색 셰브론 |
| 상세 버튼 (확장 시) | 인디고 톤 배경 (`indigo-500/10`), 인디고 셰브론 |
| 뷰 전환 | 목록↔상세 fade + translateY 애니메이션 (200ms) |
| 아코디언 | 확장/축소 max-height transition (150ms) |

**첫 방문 온보딩**: 첫 번째 행을 자동 확장 + 힌트 텍스트 "행을 탭하면 지표를 볼 수 있습니다" (1회, localStorage 플래그).

### 3.3 상세 뷰 구성

목록이 사라지고 전체 너비를 사용. 구성:

1. **헤더**: `← 목록` 뒤로가기 + 종목명 + 현재가/등락률
2. **가격 차트**: 캔들스틱 + 볼륨 (§3.4 참조)
3. **기술적 지표**: 8개 카드 그리드 (4열)
4. **시장 컨텍스트**: KOSPI/KOSDAQ RSI, 시장 추세, 변동성
5. **규칙 목록**: ON/OFF 토글 + 인라인 편집 (§3.5 참조)
6. **이 종목 체결 내역**: 해당 종목만 필터링
7. **하단 액션**: 관심 종목 해제 등

### 3.4 가격 차트

Lightweight Charts (TradingView) 기반. 캔들스틱 + 볼륨 히스토그램.

| 항목 | 설명 |
|------|------|
| 라이브러리 | `lightweight-charts` (TradingView 오픈소스) |
| 차트 타입 | 캔들스틱 (상승 빨강, 하락 파랑 — 한국식) |
| 볼륨 | 하단 히스토그램 (캔들 색상과 동기화, 30% 투명도) |
| 높이 | 300px |
| 크로스헤어 | Normal 모드 (십자선 + 가격/시간 표시) |

**기간 선택 버튼** (차트 상단):

| 버튼 | 범위 |
|------|------|
| 1W | 최근 7일 (영업일 ~5개) |
| 1M | 최근 30일 |
| 3M | 최근 90일 (기본값) |
| 6M | 최근 180일 |
| 1Y | 최근 365일 |

**드래그 확대**: 차트 위에서 마우스 드래그로 영역 선택 → 해당 구간 확대. 커서 `crosshair` 표시. 최소 20px 이상 드래그 시에만 동작.

**기타 인터랙션**:
- 마우스 휠: 줌 인/아웃
- "전체 보기" 버튼: 줌인 상태에서 나타남, 클릭 시 fitContent 복귀
- 드래그 선택 오버레이: 반투명 파란 영역으로 선택 범위 시각화

### 3.5 규칙 ON/OFF 토글

규칙 카드 왼쪽에 슬라이드 토글 배치.

| 상태 | 시각적 표현 |
|------|------------|
| 활성 (ON) | 초록색 (`green-500`) 배경, 흰색 동그라미 오른쪽 |
| 비활성 (OFF) | 회색 (`gray-600`) 배경, 흰색 동그라미 왼쪽 |
| 비활성 규칙 | 카드 전체 `opacity-50` |

토글 클릭 → 규칙 활성/비활성 전환 (API: 로컬 서버 규칙 업데이트).
수정/삭제 버튼은 카드 오른쪽에 배치.

### 3.6 반응형 동작

배경은 항상 전체 화면을 채우고, 콘텐츠만 `max-w-6xl` 중앙 정렬.

| 구간 | 브레이크포인트 | 변경 사항 |
|------|--------------|----------|
| **데스크톱** | ≥ 1024px | 기본 레이아웃. 지표 카드 4열. 종목 행에 모든 정보 표시. |
| **태블릿** | 768 ~ 1023px | 지표 카드 2열. 종목 행에서 `최근 체결` 컬럼 숨김. 체결 테이블 컬럼 축소. |
| **모바일** | < 768px | 지표 카드 1열. 종목 행: 이름+가격만 표시, 규칙 수는 작은 배지. 오른쪽 `>` 버튼 유지 (터치 영역 w-12). 헤더 검색바 → 아이콘 클릭 시 확장. |

**공통 원칙**:
- 아코디언 + 오른쪽 `>` 패턴은 모든 화면 크기에서 동일하게 동작 (터치 친화적 설계)
- 상세 뷰는 항상 전체 너비 사용 (화면 크기 무관)
- 헤더는 sticky, 스크롤 시 항상 접근 가능

## 4. 탭 구성

탭은 **종목 리스트 카드 내부**에 배치 (카드 헤더). 탭의 영향 범위가 이 카드 안에 한정됨을 시각적으로 명확히 한다.

| 탭 | 데이터 소스 | 설명 |
|----|------------|------|
| 내 종목 | 규칙이 걸린 종목 | 엔진이 감시 중인 종목 |
| 관심 종목 | 사용자 등록 | 아직 규칙 없음, 모니터링용 |

종목 카드 아래의 미체결 주문·체결 내역은 탭과 무관하게 항상 표시.

### 4.1 미체결 주문

미체결 건이 있을 때만 표시. 노란색 테두리로 "대기 중" 상태 강조.

| 컬럼 | 내용 |
|------|------|
| 시각, 종목, 방향, 수량, 가격, 유형 | 주문 정보 |
| 취소 | 미체결 주문 취소 버튼 |

### 4.2 체결 내역

항상 표시. 전체 종목의 최근 체결 기록.

| 컬럼 | 내용 |
|------|------|
| 시각, 종목, 방향, 수량, 가격 | 체결 정보 |
| 상태 | 체결/거부 배지 |

## 5. 상태 표시 (신호등)

### 5.1 위치

**로고 오른쪽**에 작은 원(dot, 2.5 크기)으로 표시: `StockVision ●`

선정 이유:
- 로고는 항상 시야에 들어오는 첫 번째 영역
- 읽는 순서가 `로고 → 상태`로 자연스러움
- 우측 영역(알림/톱니바퀴)은 이미 정보가 많아서 묻힘

### 5.2 색상 로직

| 색상 | 조건 | 의미 |
|------|------|------|
| 🟢 녹색 | 로컬 + 브로커 + 클라우드 정상 | 거래 가능 |
| 🟡 노란색 | 일부 연결 불안정 | 제한적 동작 |
| 🔴 빨간색 | 로컬 서버 미연결 | 규칙 관리만 가능 |

## 6. 툴팁 (용어 설명)

### 6.1 대상

사용자가 모를 수 있는 금융/기술 용어:
- 지표: RSI, MACD, MACD Signal, 볼린저 밴드, 거래량배수
- 상태: 규칙 활성/비활성, 시장가/지정가, 체결/거부
- 시장: KOSPI RSI, 시장 추세, 변동성

### 6.2 표현 방식 (후보)

| 방식 | 장점 | 단점 |
|------|------|------|
| **(A) 점선 밑줄** (dotted underline) | 깔끔, 금융 앱 표준 (TradingView 등), 이미 아는 유저에게 방해 없음 | 터치에서 발견 어려울 수 있음 |
| (B) `?` 아이콘 | 명시적, 초보자 친화 | 지표가 많으면 `?` 범람 → 노이지 |
| (C) `ⓘ` 아이콘 | `?`보다 모던 | B와 같은 문제 |

> **권장**: (A) 점선 밑줄 기본 + 최초 방문 시 "용어를 클릭하면 설명이 나옵니다" 힌트 한 번 표시.
> → 프로토에서 비교 후 확정.

### 6.3 툴팁 내용

간결하게 1~2문장. 예:
- **RSI(14)**: 14일 상대강도지수. 30 이하 = 과매도(매수 기회), 70 이상 = 과매수(매도 기회).
- **MACD**: 이동평균수렴확산. 양수 = 상승 추세, 음수 = 하락 추세.
- **거래량배수**: 최근 평균 대비 현재 거래량 비율. 2x 이상 = 급등/급락 가능성.

## 7. 수용 기준

### 7.1 계좌 요약

- [ ] 증권사명 + 계좌번호 (마스킹) 표시
- [ ] 총 평가금액(강조) + 수익률 배지 1행, 보조 정보 2행 레이아웃
- [ ] 장 상태 (장전/장중/장후/휴장) 표시

### 7.2 목록 뷰

- [ ] 탭 전환 (내 종목 ↔ 관심 종목) — 카드 내부 배치, indigo 액센트
- [ ] 행 클릭 → 아코디언 확장/축소 (max-height transition)
- [ ] 확장 영역에 지표 4개 카드 + "상세 보기 →" 텍스트
- [ ] 왼쪽 인디고 액센트 바 + 호버 시 하단 발광
- [ ] 오른쪽 `>` 버튼 클릭 → 상세 뷰 전환
- [ ] 첫 방문 시 첫 행 자동 확장 + 힌트 (1회, localStorage)
- [ ] 미체결 주문 테이블 (미체결 시만, 취소 버튼)
- [ ] 체결 내역 테이블 항상 표시
- [ ] 빈 상태 UI (종목 0개, 체결 0건 시 안내 카드)

### 7.3 상세 뷰

- [ ] `← 목록` 버튼 → 목록 뷰 복귀 (fade 애니메이션)
- [ ] 가격 차트: 캔들스틱 + 볼륨 (Lightweight Charts)
- [ ] 차트 기간 선택 (1W/1M/3M/6M/1Y)
- [ ] 차트 드래그 확대 + 휠 줌 + "전체 보기" 리셋
- [ ] 전체 기술적 지표 (8개) 표시
- [ ] 시장 컨텍스트 표시
- [ ] 규칙 ON/OFF 토글 (활성/비활성 전환, `role="switch"`)
- [ ] 규칙 인라인 편집 — IF/THEN 구조화 (수정/삭제/추가)
- [ ] 이 종목 체결 내역 (필터링)

### 7.4 헤더

- [ ] 검색 오버레이 (입력 → 드롭다운 결과 → 선택 시 상세 뷰)
- [ ] 알림 드롭다운 (읽음/미읽음, 배지, `aria-haspopup`)
- [ ] 톱니바퀴 드롭다운 (사용자+증권사, 엔진, 연결 상태, 메뉴)
- [ ] 엔진 중지/Kill 인라인 확인
- [ ] 상태 신호등 (로고 오른쪽)

### 7.5 툴팁

- [ ] 금융 용어에 점선 밑줄 (또는 확정된 방식) 적용
- [ ] 클릭/호버 시 설명 표시
- [ ] 최초 방문 시 안내 힌트

### 7.6 반응형

- [ ] 데스크톱 (≥1024px): 기본 레이아웃 그대로
- [ ] 태블릿 (768~1023px): 지표 카드 2열, 체결 테이블 축소
- [ ] 모바일 (< 768px): 종목 행 정보 축소, 지표 카드 1열, 터치 최적화

### 7.7 비주얼 통일

- [ ] 로그인/회원가입 페이지 스타일을 Proto C 다크 테마에 맞춤
