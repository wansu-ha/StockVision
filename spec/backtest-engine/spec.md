# 백테스트 엔진 — Backtest Engine

> 작성일: 2026-03-26 | 상태: 초안

---

## 목표

사용자가 작성한 DSL 전략을 과거 데이터로 시뮬레이션하여 수익률, 최대 낙폭, 승률 등을 확인할 수 있는 백테스트 엔진.
**클라우드에서 실행** → 로컬 설치 없이 전략 체험 가능.

---

## 범위

### 포함

1. **멀티 타임프레임** — 1분/5분/15분/1시간/일봉 단위 백테스트
2. **DSL 규칙 평가** — 기존 `sv_core/parsing/evaluator` 재사용
3. **포지션/P&L 추적** — MockAdapter 기반
4. **수수료/세금/슬리피지** — 현실적 시뮬레이션
5. **결과 리포트** — 수익 곡선, 거래 목록, 요약 지표
6. **API 엔드포인트** — `POST /api/v1/backtest/run`
7. **프론트엔드 UI** — 기간/타임프레임 선택 + 차트

### 미포함 (v2)

- 멀티 종목 포트폴리오 백테스트
- 파라미터 최적화 (그리드 서치)
- 공매도
- 마진/레버리지

## BT-1: BacktestRunner 코어 엔진

**설명**: 과거 데이터를 순회하며 DSL 규칙을 평가하고 가상 매매를 실행하는 코어 로직.

**수용 기준**:
- [ ] 입력: rule (DSL script), symbol, start_date, end_date, timeframe, initial_cash
- [ ] 지원 타임프레임: `1m`, `5m`, `15m`, `1h`, `1d`
- [ ] 바 데이터 로드: DB 우선, 부족하면 yfinance on-demand (일봉만)
- [ ] 분봉 → 상위 타임프레임 집계 (1m → 5m/15m/1h)
- [ ] 각 바마다: 지표 계산 → DSL 평가 → 매수/매도 시그널 → 주문 실행
- [ ] look-ahead bias 방지: `as_of_date` 기준으로 데이터 제한
- [ ] 결과: equity_curve, trades, summary_metrics

**파일**:
- `cloud_server/services/backtest_runner.py` (신규)
- `sv_core/indicators/calculator.py` (신규) — 지표 계산 공유 모듈 (local engine + backtest 공용)

**재사용 컴포넌트**:

| 기존 코드 | 용도 | 수정 필요 |
|----------|------|----------|
| `sv_core/parsing/evaluator.py` | DSL 평가 | 없음 |
| `sv_core/parsing/parser.py` | DSL 파싱 | 없음 |
| `local_server/engine/evaluator.py` | RuleEvaluator (DSL+JSON) | context 빌드 로직을 `sv_core`로 추출 |
| `local_server/broker/mock/adapter.py` | MockAdapter | 수수료/세금 추가 |

**멀티 타임프레임 데이터 로드**:
- DSL에 타임프레임이 명시된 경우 (예: `RSI(14, "5m") AND MA(20, "1d")`)
- 규칙에서 참조하는 **모든 타임프레임**의 바 데이터를 로드해야 함
- DSL AST를 순회하여 사용된 타임프레임 목록을 추출하는 유틸 필요

## BT-2: 지표 계산 (멀티 타임프레임)

**설명**: 백테스트 루프 내에서 각 바 시점의 지표를 계산. 일봉/분봉 모두 지원.

**수용 기준**:
- [ ] 롤링 윈도우 방식 (매 바마다 전체 재계산 하지 않음)
- [ ] 지원 지표: RSI(n), MA(n), EMA(n), MACD, 볼린저밴드(n), 평균거래량(n)
- [ ] 타임프레임별 독립 계산 (5분봉 RSI ≠ 일봉 RSI)
- [ ] look-ahead bias 없음 (현재 바까지의 데이터만 사용)
- [ ] numpy 벡터화로 성능 최적화

**성능 목표**:
- 1종목 × 1년 일봉 (~250바) → < 1초
- 1종목 × 1년 1분봉 (~95,000바) → < 10초
- 1종목 × 5년 5분봉 (~95,000바) → < 10초

## BT-3: 수수료/세금/슬리피지 시뮬레이션

**설명**: 백테스트 결과가 과대평가되지 않도록 현실적 비용 반영.

**수용 기준**:
- [ ] 매수 수수료: 0.015% (증권사 기본)
- [ ] 매도 수수료: 0.015%
- [ ] 매도 세금: 0.18% (증권거래세 0.18%, 2025년 기준)
- [ ] 슬리피지: 설정 가능 (기본 0.1% — 호가 1틱 가정)
- [ ] 비용 항목별 분리 리포트

**파일**:
- `cloud_server/services/backtest_cost.py` (신규)

## BT-4: 결과 리포트

**설명**: 백테스트 결과를 정량적으로 요약.

**수용 기준**:
- [ ] 수익 곡선 (equity curve) — 시계열
- [ ] 거래 목록 — 진입/청산 일시, 가격, 수량, P&L, 보유기간
- [ ] 요약 지표:

| 지표 | 설명 |
|------|------|
| 총 수익률 (%) | (최종 자산 / 초기 자산 - 1) × 100 |
| 연환산 수익률 (CAGR) | 기간 보정 |
| 최대 낙폭 (MDD) | 고점 대비 최대 하락률 |
| 승률 (%) | 수익 거래 / 전체 거래 |
| 손익비 (Profit Factor) | 총이익 / 총손실 |
| 평균 보유기간 | 거래당 평균 바 수 |
| 거래 횟수 | |
| 샤프 비율 | (수익률 - 무위험) / 변동성 |

## BT-5: API 엔드포인트

**수용 기준**:
- [ ] `POST /api/v1/backtest/run`
- [ ] 요청:
  ```json
  {
    "rule_id": 42,           // 또는 inline script
    "script": "매수: RSI(14) <= 30\n매도: RSI(14) >= 70",
    "symbol": "005930",
    "start_date": "2025-01-01",
    "end_date": "2025-12-31",
    "timeframe": "1d",       // 1m, 5m, 15m, 1h, 1d
    "initial_cash": 10000000,
    "commission_rate": 0.00015,
    "tax_rate": 0.0018,
    "slippage_rate": 0.001
  }
  ```
- [ ] 응답: equity_curve, trades, summary (BT-4 형식)
- [ ] inline script 지원 (rule_id 없이 DSL 직접 전달 → 저장 전 테스트)
- [ ] 타임아웃: 60초 (분봉 5년은 오래 걸릴 수 있음)
- [ ] 인증 필요 (current_user)
- [ ] rate limit: 사용자당 분당 5회

## 성능 고려사항

### 데이터 로딩

| 시나리오 | 바 수 | 예상 로드 시간 |
|---------|-------|--------------|
| 1종목 1년 일봉 | ~250 | < 0.1초 |
| 1종목 1년 1분봉 | ~95,000 | ~1초 (DB), ~5초 (yfinance) |
| 1종목 5년 5분봉 | ~95,000 | ~1초 (DB) |
| 1종목 5년 1분봉 | ~475,000 | ~3초 (DB) |

### 최적화 전략

1. **numpy 벡터화** — 지표 계산을 루프 대신 배열 연산
2. **사전 계산** — 전체 바에 대해 지표를 한 번에 계산 후 인덱싱
3. **DB 쿼리 최적화** — symbol + timestamp 인덱스 활용, 페이징
4. **캐싱** — 같은 종목/기간 재실행 시 바 데이터 캐시 (Redis)
