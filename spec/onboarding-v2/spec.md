# 온보딩 신뢰 강화 (Onboarding v2)

> 작성일: 2026-03-12 | 상태: 구현 완료 | Phase C (C5)

## 1. 배경

현재 온보딩(`OnboardingWizard.tsx`)은 4단계(위험 고지 → 브릿지 설치 → 브로커 키 → 요약)로 구성되어 있다. 기본 흐름은 작동하지만, 설치형 자동매매 제품 특유의 불안감을 해소하기에 피드백이 부족하다.

특히 브릿지 설치 단계(`BridgeInstaller.tsx`)에서:
- exe 다운로드 → 실행 → 연결 감지까지 사용자에게 "지금 뭐가 되고 있는지" 상태 전환이 불명확
- 딥링크 실패 시 대안 안내 부족
- "왜 로컬 서버가 필요한지" 설명 부족

## 2. 목표

온보딩 각 단계에서 사용자가 "왜 이것이 필요한지", "지금 어디까지 됐는지", "막히면 뭘 해야 하는지"를 즉시 파악할 수 있다.

## 3. 범위

### 3.1 포함

**A. 단계별 맥락 설명**
- 각 단계 진입 시 "왜 필요한지" + "완료하면 얻는 것" 설명
- 예: 브릿지 설치 → "주문은 PC에서만 실행됩니다. 외부 서버에 비밀번호가 전달되지 않습니다."

**B. 브릿지 설치 단계 세분화**
- 현재 3단계 (download → run → connect)를 더 명확하게 시각화
- 각 하위 단계에 체크마크 + 진행 표시
- 딥링크 실패 시 수동 실행 안내 (exe 경로, 명령어)
- 포트 충돌 감지 시 해결 안내

**C. 위험 고지 시각화 강화**
- "로컬 실행" → 자물쇠 아이콘 + "API 키는 이 PC에만 저장됩니다"
- "비밀번호 미보관" → 방패 아이콘 + "클라우드 서버는 비밀번호를 저장하지 않습니다"
- "사용자 책임" → 체크리스트 형태로 동의 항목 명시

**D. 요약 단계 강화**
- 모든 연결 상태를 실시간으로 표시 (로컬/브로커/클라우드)
- 미완료 항목에 "돌아가기" 링크
- "설정 완료 → 대시보드로" 전환 시 자연스러운 애니메이션

### 3.2 제외

- 온보딩 단계 순서 변경
- 새로운 단계 추가
- 모바일 온보딩 (PWA는 Phase C6)

## 6. 수용 기준

- [ ] 각 온보딩 단계에 "왜 필요한지" 설명이 표시된다
- [ ] 브릿지 설치 단계에서 하위 진행 상태(다운로드/실행/연결)가 각각 표시된다
- [ ] 딥링크 실패 시 수동 실행 경로가 안내된다
- [ ] 위험 고지에서 "로컬 실행", "비밀번호 미보관", "사용자 책임"이 시각적으로 구분된다
- [ ] 요약 단계에서 모든 연결 상태가 실시간으로 표시된다
- [ ] 사용자는 각 단계가 왜 필요한지 설명할 수 있다 (UX PRD 수용 기준)
