## 1. 목표

관리자(운영팀)가 시스템 상태, 사용자 활동, 시세 데이터를 모니터링하고
전략 템플릿·서비스 키를 관리하는 웹 UI를 구현한다.

**핵심 원칙:**
- 개인 금융정보 접근 불가 (체결, 잔고, 수익률 = 로컬에만)
- 유저 API Key 접근 불가 (Credential Manager에만)

## 2. 요구사항

### 2.1 기능적 요구사항

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| F1 | 유저 목록 (이메일, 닉네임, 역할, 이메일 인증 여부, 활성/비활성, 가입일, 마지막 로그인) | P0 |
| F2 | 유저 비활성화/삭제 | P1 |

| F4 | 시스템 통계 (유저 수, 활성 유저 수, 규칙 수, 활성 클라이언트 수) | P0 |

| F6 | AI 분석 모니터링 (일별/월별 분석 수, 토큰 사용량/비용 추정, 최근 결과 샘플, 에러율) | P1 |
| F7 | 서비스 증권사 키 등록/삭제/상태 확인 | P1 |
| F8 | 전략 템플릿 CRUD (디폴트 전략 양식) | P1 |
| F9 | 에러 로그 뷰어 | P2 |

| 항목 | 목표 |
|------|------|
| 페이지 로딩 | < 2초 |
| 통계 갱신 주기 | 10초 (폴링) |

## 3. 페이지 구조

```
/admin/login               → 어드민 전용 로그인 (흰색 테마, 공개)
/admin                     → 대시보드 (통계 요약 + 클라우드 서버 상태 + AI 요약 + 최근 에러)
/admin/users               → 유저 목록/관리
/admin/stats               → 접속 통계 (차트)
/admin/service-keys        → 서비스 증권사 키 관리
/admin/ai                  → AI 분석 모니터링

/admin/errors              → 에러 로그
```

## 5. API 연동

모든 어드민 API는 `Authorization: Bearer <jwt>` + `role == "admin"` 필수.

```
GET    /api/v1/admin/users              → 유저 목록
PATCH  /api/v1/admin/users/:id          → 유저 상태 변경
GET    /api/v1/admin/stats              → 시스템 통계
GET    /api/v1/admin/stats/connections  → 접속 통계 (하트비트)
GET    /api/v1/admin/collector-status   → 클라우드 서버 수집기 상태
GET    /api/v1/admin/service-keys       → 서비스 키 목록
POST   /api/v1/admin/service-keys       → 키 등록
DELETE /api/v1/admin/service-keys/:id   → 키 삭제
GET    /api/v1/admin/templates          → 템플릿 목록
POST   /api/v1/admin/templates          → 템플릿 생성
PUT    /api/v1/admin/templates/:id      → 템플릿 수정
DELETE /api/v1/admin/templates/:id      → 템플릿 삭제
GET    /api/v1/admin/ai/stats           → AI 분석 통계 (일별/월별 분석 수, 토큰, 비용)
GET    /api/v1/admin/ai/recent         → AI 최근 분석 결과 샘플
GET    /api/v1/admin/errors?limit=100   → 에러 로그
```

## 6. 데이터 접근 정책

### 볼 수 있는 것

| 카테고리 | 항목 |
|----------|------|
| 유저 메타 | 이메일, 닉네임, 역할, 이메일 인증 여부, 활성/비활성, 가입일, 마지막 로그인 |
| 운영 통계 | 총 유저 수, 활성 유저 수, 규칙 수, 활성 클라이언트 수 |
| 하트비트 | 로컬 앱 UUID, 버전, OS, 브로커 연결 여부, 엔진 실행 여부, 활성 규칙 수 |
| 서버 상태 | 업타임, CPU/메모리 (psutil), 시세 수집 상태 (구독 종목 수, 마지막 수집, 오늘 건수) |
| AI 분석 | 일별/월별 분석 수, 토큰 사용량, 비용 추정, 최근 결과 샘플, 에러율 |
| 서비스 키 | 키 목록 (secret은 마스킹), 상태, 마지막 사용 시각 |
| 전략 템플릿 | 이름, 설명, 카테고리, 공개 여부, 생성자, 생성일 |
| 관심종목 집계 | 종목별 관심 등록 수 (집계만, 개별 유저 열람 불가) |
| 에러 로그 | 서버 에러 목록, 시각, 레벨 |

### 조심해야 하는 것 (접근 제외)

| 항목 | 이유 |
|------|------|
| 개별 유저 규칙 내용 (buy/sell_conditions) | 투자 전략 = 개인정보 취급. 규칙 "수"만 집계 |
| 개별 유저 관심종목 | 투자 성향 유추 가능. 집계 통계만 허용 |
| AI 분석 결과의 유저별 이력 | 관심 종목/분석 패턴 노출 우려. 전체 샘플만 열람 |

### 볼 수 없는 것 (아키텍처상 불가)

| 항목 | 이유 |
|------|------|
| 체결 내역, 잔고, 보유 종목/수량 | 로컬에만 저장, 클라우드 미수집 |
| 수익률, 손익액 | 로컬에만 저장 |
| 증권사 API Key/Secret | 로컬 Credential Manager에만 저장 |
| 거래 금액/거래량 집계 | 로컬 데이터 → 클라우드 미전송 (opt-in 없이 불가) |

## 8. 범위

### 포함

- 어드민 라우팅 + 레이아웃 (사이드바 7개 메뉴)
- 대시보드 (통계 카드 + 클라우드 서버 상태 + AI 요약 + 최근 에러)
- 유저 관리 (목록, 검색, 페이지네이션, 역할/인증/마지막로그인 표시)
- 접속 통계 차트 (7/30/90일)
- 서비스 키 관리 (등록/삭제, api_key/api_secret/app_name 스키마)
- 전략 템플릿 CRUD (name/description/category/is_public 스키마)
- AI 분석 모니터링 (토큰/비용 추적, 최근 분석 결과)
- 에러 로그 뷰어 (레벨 필터, 페이지네이션, 상세 모달)

### 미포함

- 어드민 API 백엔드 (Unit 4에 포함)
- APM 수준 인프라 모니터링 (v2, Sentry/Grafana)
- 개별 유저 전략 규칙 열람 (접근 제외 정책)
- 커뮤니티 관리 (v2)
