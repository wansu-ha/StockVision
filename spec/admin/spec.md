## 1. 목표

관리자(운영팀)가 시스템 상태, 사용자 활동, 시세 데이터를 모니터링하고
전략 템플릿·서비스 키를 관리하는 웹 UI를 구현한다.

**핵심 원칙:**
- 개인 금융정보 접근 불가 (체결, 잔고, 수익률 = 로컬에만)
- 유저 API Key 접근 불가 (Credential Manager에만)

---

## 2. 요구사항

### 2.1 기능적 요구사항

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| F1 | 유저 목록 (이메일, 닉네임, 가입일, 접속 상태) | P0 |
| F2 | 유저 비활성화/삭제 | P1 |
| F3 | 접속 통계 (하트비트 기반: 온라인 수, DAU) — MAU 차트는 v2 | P0 |
| F4 | 시스템 통계 (규칙 수, 전략 수, 에러율) | P0 |
| F5 | 시세 데이터 모니터링 (클라우드 서버 상태) | P1 |

> AI 분석 모니터링은 v2 범위 (AI 자체가 v2)

| F6 | 서비스 키움 키 등록/삭제/상태 확인 | P1 |
| F7 | 전략 템플릿 CRUD (디폴트 전략 양식) | P1 |
| F8 | 에러 로그 뷰어 | P2 |

### 2.2 비기능적 요구사항

| 항목 | 목표 |
|------|------|
| 페이지 로딩 | < 2초 |
| 통계 갱신 주기 | 10초 (폴링) |
| 접근 제어 | role=admin만 (JWT 검증) |

## 3. 페이지 구조

### 3.1 라우팅

```
/admin                     → 대시보드 (통계 요약)
/admin/users               → 유저 목록/관리
/admin/stats               → 접속 통계 (차트)
/admin/data                → 시세 데이터 상태
/admin/service-keys        → 서비스 키움 키 관리
/admin/templates           → 전략 템플릿 관리
/admin/errors              → 에러 로그
```

### 3.2 레이아웃

```
┌──────────────────────────────────────────────────────────┐
│  StockVision Admin ●●●  │                │  🔔  admin ▼  │
├──────────┬───────────────────────────────────────────────┤
│          │                                               │
│ 대시보드  │  메인 콘텐츠                                   │
│ 유저     │                                               │
│ 시세     │                                               │
│ 템플릿   │                                               │
│ 에러 로그 │                                               │
│ 설정     │                                               │
│          │                                               │
└──────────┴───────────────────────────────────────────────┘

상단 바:
  좌측: "StockVision Admin" + 신호등 ●●● (유저 화면과 동일 컴포넌트)
  우측: 알림 벨 + admin 유저명 (클릭 → 유저 화면으로 전환, 로그아웃)
  중앙: 비움 (검색 불필요)

사이드바: 메뉴 많으므로 유저 화면과 달리 사이드바 사용
```

## 5. API 연동

모든 어드민 API는 `Authorization: Bearer <jwt>` + `role == "admin"` 필수.

```
GET    /api/v1/admin/users              → 유저 목록
PATCH  /api/v1/admin/users/:id          → 유저 상태 변경
GET    /api/v1/admin/stats              → 시스템 통계
GET    /api/v1/admin/stats/connections  → 접속 통계 (하트비트)
GET    /api/v1/admin/data/status        → 클라우드 서버 상태
GET    /api/v1/admin/service-keys       → 서비스 키 목록
POST   /api/v1/admin/service-keys       → 키 등록
DELETE /api/v1/admin/service-keys/:id   → 키 삭제
GET    /api/v1/admin/templates          → 템플릿 목록
POST   /api/v1/admin/templates          → 템플릿 생성
PUT    /api/v1/admin/templates/:id      → 템플릿 수정
DELETE /api/v1/admin/templates/:id      → 템플릿 삭제
GET    /api/v1/admin/errors?limit=100   → 에러 로그
```

## 6. 볼 수 있는 것 vs 없는 것

| 볼 수 있음 | 볼 수 없음 |
|------------|-----------|
| 유저 이메일, 닉네임, 가입일 | 키움 API Key/Secret |
| 접속 상태 (온/오프라인) | 체결 내역, 잔고, 수익률 |
| 저장된 규칙 수 (내용은 정책에 따라) | 거래량, 거래 종목 |
| 시세 데이터 (클라우드 서버) | — |
| 전략 템플릿 내용 | — |
| 에러 로그 | — |

## 7. 수용 기준

- [ ] admin 계정으로 `/admin` 접근 → 대시보드 표시
- [ ] 일반 유저로 `/admin` 접근 → 403 또는 리다이렉트
- [ ] 유저 목록 조회 (이메일, 닉네임, 가입일, 접속 상태)
- [ ] 시스템 통계 (유저 수, 규칙 수, 에러 수) 표시
