## 1. 목표

관리자(운영팀)가 시스템 상태, 사용자 활동, 시세 데이터를 모니터링하고
전략 템플릿·서비스 키를 관리하는 웹 UI를 구현한다.

**핵심 원칙:**
- ✅ **규칙 내용 열람 가능** — 어드민이 유저의 규칙 내용(매수/매도 조건)을 조회 가능
- ❌ **체결 내역·잔고·수익률 접근 불가** — 로컬 PC에만 저장 (구조적 보장)
- ❌ **유저 API Key 접근 불가** — Credential Manager에만 저장

> 어드민은 "규칙 설계"를 모니터링할 수 있지만, "실행 결과(체결/잔고)"는 볼 수 없음.

## 2. 요구사항

### 2.1 기능적 요구사항

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| F1 | 유저 목록 (이메일, 닉네임, 가입일, 접속 상태, 규칙 수) | P0 |
| F2 | 유저 비활성화 (soft delete: is_active = false) | P1 |
| F3 | **유저 규칙 열람** — 개별 유저의 규칙 목록 + 조건 내용 조회 | P0 |
| F4 | 접속 통계 (하트비트 기반: 온라인 수, DAU) | P0 |
| F5 | 시스템 통계 (전체 유저, 활성 규칙, 에러 수) | P0 |
| F6 | 시세 데이터 모니터링 (클라우드 서버 상태) | P1 |
| F7 | 서비스 키 등록/삭제/상태 확인 | P1 |
| F8 | 전략 템플릿 CRUD (디폴트 전략 양식) | P1 |
| F9 | 에러 로그 뷰어 | P2 |

## 5. API 연동

모든 어드민 API는 `Authorization: Bearer <jwt>` + `role == "admin"` 필수.

```
GET    /api/v1/admin/stats              → 시스템 통계
GET    /api/v1/admin/stats/connections  → 접속 통계 (하트비트)
GET    /api/v1/admin/data/status        → 클라우드 서버 상태
GET    /api/v1/admin/users              → 유저 목록
PATCH  /api/v1/admin/users/:id          → 유저 상태 변경
GET    /api/v1/admin/users/:id/rules    → 유저 규칙 열람 (NEW)
GET    /api/v1/admin/service-keys       → 서비스 키 목록
POST   /api/v1/admin/service-keys       → 키 등록
DELETE /api/v1/admin/service-keys/:id   → 키 삭제
GET    /api/v1/admin/templates          → 템플릿 목록
POST   /api/v1/admin/templates          → 템플릿 생성
PUT    /api/v1/admin/templates/:id      → 템플릿 수정
DELETE /api/v1/admin/templates/:id      → 템플릿 삭제
GET    /api/v1/admin/errors?limit=100   → 에러 로그
```

## 6. 볼 수 있는 것 vs 없는 것

| 볼 수 있음 | 볼 수 없음 |
|------------|-----------|
| 유저 이메일, 닉네임, 가입일 | KIS API Key/Secret |
| 접속 상태 (온/오프라인) | 체결 내역 (로컬 PC에만 저장) |
| **유저 규칙 내용 (읽기 전용)** | 잔고, 수익률 (로컬 PC에만 저장) |
| 규칙 수 (활성/비활성) | 거래량, 거래 종목 |
| 시세 데이터 (클라우드 서버) | — |
| 전략 템플릿 내용 | — |
| 에러 로그 | — |

> **보안 원칙**: 체결·잔고·수익률은 로컬 PC의 SQLite(logs.db)에만 저장되므로
> 클라우드 서버에 데이터 자체가 존재하지 않음 → 구조적으로 접근 불가.

## 7. 수용 기준

- [ ] admin 계정으로 `/admin` 접근 → 대시보드 표시
- [ ] 일반 유저로 `/admin` 접근 → 403 또는 리다이렉트
- [ ] 유저 목록 조회 (이메일, 닉네임, 가입일, 접속 상태, 규칙 수)
- [ ] 유저 [상세] 클릭 → 규칙 목록 + 조건 내용 열람 (읽기 전용)
- [ ] 유저 비활성화 동작
- [ ] 시스템 통계 (유저 수, 온라인 수, 활성 규칙, 에러 수) 표시
- [ ] 접속 통계 차트 (DAU, 기간 선택)
- [ ] 전략 템플릿 CRUD 정상 동작
- [ ] 서비스 키 등록/삭제 정상 동작
- [ ] 클라우드 서버 상태 표시
- [ ] 에러 로그 조회 + 필터
- [ ] 개인 금융정보 접근 불가 (구조적 확인)
