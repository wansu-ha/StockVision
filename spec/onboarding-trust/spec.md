# 온보딩 신뢰 강화 (Onboarding Trust)

> 작성일: 2026-03-12 | 상태: 구현 완료 | Phase C

## 1. 배경

현재 온보딩은 회원가입 → 설정(API 키 입력) → 엔진 시작의 분산된 수동 과정이다.
`BridgeInstaller` 컴포넌트가 존재하지만 메인 플로우에 통합되지 않았다.
설치형 자동매매 제품 특성상, 각 단계의 피드백이 부족하면 "이게 제대로 된 건지" 불안감이 남는다.

UX PRD에서 P1으로 분류: "온보딩은 설치 절차가 아니라 신뢰 형성 과정이어야 한다."

## 2. 목표

설치 과정에서 각 단계가 왜 필요한지 알고, 막혀도 다음 행동이 즉시 보인다.

## 3. 범위

### 3.1 포함

- 온보딩 위저드 (단계별 가이드 플로우)
- 각 단계별 피드백 강화: 왜 필요한지, 완료 조건, 다음에 얻는 것
- 로컬 서버(브릿지) 연결 단계: 다운로드 → 실행 → 연결 감지 상태 분리
- 실패 시 재시도 + 도움말 안내
- 위험고지 시각화 강화: 로컬 실행, 비밀번호 미보관, 사용자 책임

### 3.2 제외

- 모바일 전용 온보딩 UI
- 가입 절차 변경 (이메일/비번 기반 유지)
- 튜토리얼/교육 콘텐츠

## 4. 의존성

| 의존 대상 | 상태 | 비고 |
|-----------|------|------|
| BridgeInstaller 컴포넌트 | 존재 (미통합) | 3단계 안내 + WS 폴링 구현 완료 |
| 클라우드 인증 | 완료 | 로그인/회원가입 동작 |
| 로컬 서버 health API | 완료 | `GET /health` |
| 브로커 키 등록 API | 완료 | `POST /api/config/broker-keys` |

**선행 조건 없음 — 바로 착수 가능**

## 5. 온보딩 플로우 설계

### 5.1 단계 구성

| 단계 | 제목 | 설명 | 완료 조건 |
|------|------|------|----------|
| 1 | 계정 만들기 | 이메일 인증으로 계정 보호 | 이메일 인증 완료 |
| 2 | 로컬 서버 연결 | 내 PC에서 실행되는 비서 연결 | 로컬 서버 health 응답 |
| 3 | 증권사 연결 | API 키로 증권사 계좌 연결 | 브로커 연결 상태 확인 |
| 4 | 시작 준비 완료 | 설정 요약 확인 + 다음 단계 안내 | 사용자가 "시작" 클릭 |
| 5 | 완료 | 대시보드로 이동 | - |

### 5.2 각 단계 카드 구성

모든 단계 카드에 3가지 정보 표시:

```
[왜 필요한가]  이메일 인증은 계정을 보호합니다. 비밀번호는 서버에 저장되지 않습니다.
[완료 조건]    인증 메일의 링크를 클릭하면 완료됩니다.
[다음 단계]    계정이 만들어지면 로컬 서버를 연결합니다.
```

### 5.3 단계 4: 시작 준비 완료

현재 기본 전략/템플릿이 없으므로, 이 단계에서는:
- 설정 요약: 연결된 브로커, 모의/실전 모드, 로컬 서버 상태
- 다음 행동 안내: "대시보드에서 종목을 선택하고 전략 규칙을 추가하세요"
- 모의투자 권장: "먼저 모의투자로 시작하는 것을 권장합니다"

### 5.4 단계 2: 로컬 서버 연결 (상세)

> **미결**: 로컬 서버 인스톨러 배포 방식 미정 (BridgeInstaller의 다운로드 URL이 "#" 플레이스홀더).
> GitHub Releases, 자체 CDN, 또는 수동 안내 중 결정 필요.

기존 BridgeInstaller를 확장하여 3단계로 분리 표시:

| 하위 단계 | 상태 표시 | 동작 |
|-----------|----------|------|
| 다운로드 | ⬜ 대기 → ✅ 완료 | 다운로드 버튼 클릭 → 파일 저장 확인 |
| 실행 | ⬜ 대기 → 🔄 감지 중 | "설치 후 실행하세요" 안내 + 자동 감지 대기 |
| 연결 | 🔄 감지 중 → ✅ 연결됨 | health 폴링 → 성공 시 자동 전환 |

실패 시:
- "연결을 찾을 수 없습니다" + 체크리스트:
  - 로컬 서버가 실행 중인지 확인
  - 방화벽에서 포트 4020이 차단되지 않았는지 확인
  - [다시 시도] [도움말]

### 5.5 단계 3: 증권사 연결

기존 Settings 페이지의 API 키 등록 폼을 온보딩 위저드 내에 임베드.
추가 안내:
- "API 키는 내 PC의 로컬 서버에만 저장됩니다. 클라우드로 전송되지 않습니다."
- 모의투자/실전 선택 안내
- 키 발급 방법 가이드 링크

## 6. 위험고지 강화

### 6.1 표시 위치

온보딩 시작 전 또는 단계 2(로컬 서버 연결) 시점에 위험고지 화면.

### 6.2 고지 항목

시각적 아이콘과 함께 3가지 핵심 고지:

| 항목 | 설명 |
|------|------|
| 🖥️ 로컬 실행 | 모든 주문은 내 PC에서 실행됩니다. PC가 꺼지면 자동매매도 멈춥니다. |
| 🔒 키 미전송 | API 키와 비밀번호는 내 PC에만 저장됩니다. 클라우드로 전송되지 않습니다. |
| ⚠️ 사용자 책임 | 자동매매의 손익은 사용자에게 귀속됩니다. 모의투자로 먼저 확인하세요. |

"이해했습니다" 체크박스 → 다음 단계 활성화.

## 7. 프론트엔드 설계

### 7.1 라우팅

```
/onboarding          → 온보딩 위저드 (신규 사용자)
/onboarding?step=2   → 특정 단계로 이동 (재방문)
```

### 7.2 컴포넌트 구조

```
OnboardingWizard
├── OnboardingStep (공통 카드 레이아웃)
│   ├── StepIndicator (진행 표시)
│   ├── StepContent (단계별 내용)
│   └── StepNavigation (이전/다음)
├── BridgeInstaller (기존 컴포넌트 확장)
├── BrokerKeyForm (Settings에서 추출)
└── RiskDisclosure (위험고지 카드)
```

### 7.3 온보딩 완료 판정

온보딩 완료 여부를 localStorage에 저장.
첫 로그인 + 미완료 → /onboarding으로 리다이렉트.
완료 후 → / (대시보드)로 이동.

## 8. 수용 기준

- [ ] 신규 사용자가 첫 로그인 시 온보딩 위저드를 만난다
- [ ] 각 단계에서 "왜 필요한지", "완료 조건", "다음에 얻는 것"이 보인다
- [ ] 로컬 서버 연결 단계에서 다운로드/실행/연결 상태가 분리 표시된다
- [ ] 연결 실패 시 재시도 + 도움말 안내가 보인다
- [ ] 위험고지에서 로컬 실행, 키 미전송, 사용자 책임을 이해하고 동의한다
- [ ] 온보딩 완료 후 대시보드로 자연스럽게 전환된다
- [ ] 온보딩을 중단했다가 재방문하면 마지막 단계부터 이어진다
