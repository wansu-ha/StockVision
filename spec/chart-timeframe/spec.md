# 차트 타임프레임 & 줌 시스템

> 작성일: 2026-03-15 | 상태: 확정 | 갱신: 2026-03-18 (Stage 1+2 구현 완료, Stage 3 부분 구현)

## 목표

종목 차트에서 **봉 해상도(일봉/시봉/분봉)** 선택과 **줌 기반 기간 탐색**을 지원한다.
현재는 일봉만 가능하지만, 사용자가 줌 레벨과 봉 타입을 조합하여 원하는 시야를 확보할 수 있어야 한다.

---

## 1. 현황

| 항목 | 현재 상태 |
|------|----------|
| 봉 타입 | 일봉만 (candle/hollow/heikin/ohlc/line 5종 차트 스타일) |
| 기간 선택 | 1W, 1M, 3M(기본), 6M, 1Y |
| 데이터 소스 | 클라우드 DailyBar (yfinance → DB 캐시) |
| 분봉 | MinuteBar DB 모델 존재, 수집/API 미구현 |
| 줌 | lightweight-charts 기본 줌 (드래그/스크롤), 데이터 해상도 전환 없음 |
| 로컬 서버 | BarBuilder가 WS 틱 → 1분봉 집계, HTTP API 미노출 |

## 2. 타 플랫폼 조사

### 비교표

| 기능 | TradingView | 영웅문(키움) | 토스증권 | 카카오페이 |
|------|-------------|-------------|---------|-----------|
| 봉 옵션 | 1m,5m,15m,30m,1h,2h,4h,D,W,M + 커스텀 | 틱,분,시,일,주,월 | TradingView 동일 | TradingView 동일 |
| 줌→해상도 자동전환 | 선택적 (Auto-magnifier) | X | X | X |
| 해상도 선택 UI | 버튼 + 즐겨찾기 + 키보드 | 버튼/숫자 입력 | 큰 버튼 그리드 | TradingView |
| 기본 표시량 | ~50봉 | 설정 가능 | ~50봉 | ~50봉 |
| 과거 데이터 로딩 | 좌측 스크롤 lazy load | 드래그 | lazy load | lazy load |
| 장중↔일봉 전환 | 연속 (동일 캔버스) | 연속 | 연속 | 연속 |

### 핵심 인사이트

1. **줌 ≠ 해상도**: 모든 플랫폼이 줌(확대/축소)과 봉 해상도(1분/5분/일봉)를 **독립적으로** 제어함. 줌하면 자동으로 분봉으로 바뀌는 건 TradingView의 선택적 기능뿐.
2. **해상도는 사용자가 선택**: 버튼으로 명시적 선택이 표준.
3. **Lazy Load**: 모든 플랫폼이 좌측 스크롤 시 과거 데이터 추가 로딩.
4. **한국 모바일 증권사**: 토스, 카카오 모두 TradingView 차트 임베드.

## 3. 설계

### 3.1 봉 해상도 옵션

| 해상도 | 소스 | 과거 조회 가능량 |
|--------|------|-----------------|
| 1분봉 | 로컬 서버 (KIS REST: 최근 30일) | ~30일 |
| 5분봉 | 로컬 서버 (1분봉 집계) | ~30일 |
| 15분봉 | 로컬 서버 (1분봉 집계) | ~30일 |
| 시봉 (1h) | 로컬 서버 (1분봉 집계) | ~30일 |
| 일봉 | 클라우드 서버 (yfinance → DB) | 수년 |
| 주봉 | 클라우드 서버 (일봉 집계) | 수년 |
| 월봉 | 클라우드 서버 (일봉 집계) | 수년 |

**법적 제약 참조**: `docs/research/krx-data-legal-research.md`
- 분봉 데이터는 로컬 서버에서 **사용자 본인 브로커 키**로 조회 → 본인에게만 표시 (약관 범위 내)
- 클라우드에서 분봉 수집/재배포 불가 (증권사 약관 위반)

### 3.2 데이터 흐름

```
┌─────────────────────────────────────────────────────┐
│  Frontend (PriceChart)                              │
│                                                     │
│  [1m] [5m] [15m] [1h] [1D] [1W] [1M]  ← 해상도 버튼 │
│  [1W] [1M] [3M] [6M] [1Y] [ALL]       ← 기간 버튼   │
│                                                     │
│  해상도 ≥ 일봉 → 클라우드 요청                         │
│  해상도 < 일봉 → 로컬 요청 (+ 클라우드 일봉 fallback)  │
└───────┬──────────────────────┬───────────────────────┘
        │                      │
   Cloud Server            Local Server
   GET /api/v1/stocks/     GET /api/v1/bars/
     {symbol}/bars           {symbol}
   ?start=&end=            ?resolution=1m
   &resolution=1d            &start=&end=
        │                      │
   DailyBar (DB)           KIS REST API
   yfinance fallback       (사용자 본인 키)
```

### 3.3 기본 동작

1. **종목 선택 시**: 클라우드에서 **일봉 3개월** 로드 (현행 유지)
2. **동시에**: 로컬 서버에 **1분봉 당일분** 요청 (프리페치)
3. **사용자가 해상도 변경 시**: 해당 해상도 데이터 요청
4. **좌측 스크롤 시**: 이전 기간 데이터 lazy load

### 3.4 해상도 선택 UI

```
┌──────────────────────────────────┐
│ [1m] [5m] [15m] [1h] │ [D] [W] [M] │   ← 해상도 버튼 (장중 | 장기)
│                                  │
│ 장중 버튼: 로컬 서버 미연결 시 비활성 │
│ (회색 + "브릿지 연결 필요" 툴팁)    │
└──────────────────────────────────┘
```

- 장중 해상도(1m~1h)는 로컬 서버 연결 상태에 따라 활성/비활성
- 일봉 이상은 항상 사용 가능 (클라우드 데이터)
- 현재 선택된 해상도 버튼 하이라이트

### 3.5 기간 선택과 해상도 연동

해상도별 기본 기간 및 최대 기간:

| 해상도 | 기본 기간 | 최대 기간 | 기간 버튼 |
|--------|----------|----------|----------|
| 1분 | 1일 | 30일 | 1D, 3D, 1W, 2W, 1M |
| 5분 | 3일 | 30일 | 1D, 3D, 1W, 2W, 1M |
| 15분 | 1주 | 30일 | 1D, 3D, 1W, 2W, 1M |
| 1시간 | 2주 | 30일 | 1W, 2W, 1M |
| 일봉 | 3개월 | 전체 | 1M, 3M, 6M, 1Y, ALL |
| 주봉 | 1년 | 전체 | 6M, 1Y, 3Y, ALL |
| 월봉 | 3년 | 전체 | 1Y, 3Y, 5Y, ALL |

해상도 변경 시 기간 버튼도 해당 세트로 전환.

### 3.6 줌과 해상도의 관계

**줌은 해상도를 변경하지 않는다** (TradingView 표준 패턴).

- 줌 인/아웃은 보이는 봉 수만 조절
- 해상도는 사용자가 버튼으로만 변경
- 줌으로 봉이 3개 이하로 보이면 "더 세밀한 봉 사용 가능" 툴팁 표시 (선택적)

## 4. API 변경

### 4.1 클라우드 서버 (기존 확장)

```
GET /api/v1/stocks/{symbol}/bars
  ?start=2026-01-01
  &end=2026-03-15
  &resolution=1d          # 신규 파라미터 (기본값: 1d)
                          # 허용값: 1d, 1w, 1mo
```

- `1w`, `1mo`: 서버 측에서 DailyBar 집계하여 반환
- 기존 API 호환 유지 (resolution 미지정 시 일봉)

### 4.2 로컬 서버 (신규)

```
GET /api/v1/bars/{symbol}
  ?resolution=1m          # 1m, 5m, 15m, 1h
  &start=2026-03-15T09:00
  &end=2026-03-15T15:30
```

응답:
```json
{
  "success": true,
  "data": [
    {
      "time": "2026-03-15T09:00:00",
      "open": 55000,
      "high": 55100,
      "low": 54900,
      "close": 55050,
      "volume": 12500
    }
  ],
  "count": 360,
  "resolution": "1m",
  "source": "kis_rest"
}
```

데이터 소스 우선순위:
1. 로컬 DB (BarBuilder가 수집한 분봉)
2. KIS REST API (과거 분봉 조회, 최대 30일)
3. 데이터 없으면 빈 배열 + `"source": "none"`

## 5. 분봉 수집/보관 구조 (로컬 서버)

### 5.1 틱 데이터 특성

- 틱은 **WS 실시간 스트림 전용** — 지나간 틱의 과거 조회 API 없음
- 분봉은 KIS REST로 **최근 30일** 과거 조회 가능
- 따라서 분봉 저장이 핵심 (틱 전량 저장은 비현실적)

### 5.2 수집 전략

| 경로 | 트리거 | 저장 위치 |
|------|--------|----------|
| WS 틱 → BarBuilder → 1분봉 | 장중 자동 (WS 연결 시) | 로컬 SQLite |
| KIS REST → 과거 분봉 | 차트 요청 시 (캐시 미스) | 로컬 SQLite |

### 5.3 보관 정책

- 로컬 SQLite에 분봉 저장 (종목별)
- 30일 이상 된 분봉은 자동 정리 (디스크 절약)
- 5분/15분/시봉은 1분봉에서 실시간 집계 (별도 저장 X)

## 7. 범위

### 포함
- 해상도/기간 선택 UI
- 클라우드 주봉/월봉 집계
- 로컬 서버 분봉 API + KIS REST 연동
- 로컬 분봉 캐시 (SQLite)
- lazy load (좌측 스크롤)

### 미포함
- 틱 차트 (틱 전량 저장 비현실적, 추후 검토)
- 자동 해상도 전환 (Auto-magnifier, 추후 선택적 추가)
- TradingView 라이브러리 교체 (현행 lightweight-charts 유지)
- 클라우드 분봉 수집/재배포 (법적 제약)

## 8. 구현 단계

### Stage 1: 로컬 서버 분봉 API
- `GET /api/v1/bars/{symbol}` 엔드포인트
- KIS REST 분봉 조회 연동
- 로컬 SQLite 캐시 (같은 구간 재요청 방지)
- 5분/15분/시봉 집계 로직

### Stage 2: 클라우드 서버 주봉/월봉
- 기존 bars API에 `resolution` 파라미터 추가
- 일봉 → 주봉/월봉 집계 로직

### Stage 3: 프론트엔드
- 해상도 버튼 (1m/5m/15m/1h/D/W/M)
- 해상도↔기간 연동 (해상도 변경 시 기간 + 기간 버튼 세트 자동 전환)
- 해상도별 클라우드/로컬 분기 호출
- lazy load (onVisibleTimeRangeChanged + 디바운스 200~300ms)
- 로드 범위 추적 — 이미 로드된 구간 재요청 방지
- 로컬 미연결 시 분봉 버튼 비활성 + "브릿지 연결 필요" 툴팁

### 의존성

```
Stage 1 (로컬 API) ──┐
                     ├→ Stage 3 (프론트)
Stage 2 (클라우드) ──┘
```

Stage 1, 2는 병렬 가능. Stage 3은 둘 다 완료 후 진행.
