## 1. 목표

React SPA로 StockVision의 사용자 인터페이스를 구현한다.
**듀얼 연결**(클라우드 서버 + localhost)로 클라우드 데이터와 로컬 실행을 통합.

**핵심 역할: 순수 UI**
1. 사용자 인증 (클라우드 서버) → JWT를 localhost에 전달
2. 전략 규칙 관리 (클라우드 서버 CRUD → 저장 후 localhost sync)
3. 실시간 시세 + 체결 모니터링 (localhost WS)
4. 실행 로그 열람, 설정 변경

> 컨텍스트 fetch, 하트비트, 템플릿은 로컬 서버가 직접 통신.
> 프론트엔드의 브릿지 역할은 **JWT 전달 + 규칙 sync**로 축소.

## 3. 페이지 구조

### 3.1 라우팅

```
/                           → 메인 (로그인 필요, 싱글 페이지)
/login                      → 로그인
/register                   → 회원가입
/verify-email               → 이메일 인증
/forgot-password            → 비밀번호 재설정
/settings                   → 설정 (API Key, 모드, 프로필)
/logs                       → 실행 로그
/admin/*                    → 어드민 (Unit 6)
```

> **싱글 페이지 원칙**: 대시보드·내 종목·종목 상세·규칙 편집이 모두 `/` 하나에서 동작.
> 사이드바 없음. 설정·로그는 유저 메뉴에서 접근.

### 3.2 레이아웃

```
┌──────────────────────────────────────────────────────────┐
│  StockVision ●●●  │  🔍 종목 검색...    │  🔔  홍길동 ▼  │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  [엔진 시작/중지]                                         │
│                                                          │
│  ── 내 종목 ──────────────────────────────────────────── │
│  삼성전자  72,400 +1.2%  규칙 3개 활성                    │
│  SK하이닉스  185,000 -0.5%  규칙 1개                      │
│                                                          │
│  (v2: 주목 종목 — docs/v2-ideas.md 참조)                 │
│                                                          │
└──────────────────────────────────────────────────────────┘

상단 바 구성:
  좌측: 로고 + 신호등 ●●● (hover → 툴팁: "클라우드: 연결됨")
  중앙: 검색바 (클릭 → 오버레이, 페이지 전환 없음)
  우측: 알림 벨 + 유저명 (클릭 → 드롭다운: 실행 로그, 설정, 로그아웃)
```

### 3.3 종목 중심 UX

```
종목 검색/클릭 → 종목 상세 패널:
  - 현재가 + 등락률
  - 현재 지표 값 (RSI, MACD 등 — 조건 설정 시 참고)
  - 이 종목에 걸린 규칙 목록
  - [규칙 추가] ← 메인 CTA
  - [관심 종목 등록/해제]

규칙은 종목의 하위 개념:
  종목 → 규칙 (1:N)
```

## 5. 연결 구조 + JWT 전달

### 5.1 연결 구조

```
프론트엔드
├── 클라우드 서버 연결 (HTTPS)
│   ├── 인증 (JWT)
│   ├── 규칙 CRUD
│   ├── 종목 검색 (StockMaster)
│   ├── 관심종목 CRUD
│   └── 어드민
│
└── localhost 연결 (HTTP + WS)
    ├── JWT 전달 (로그인 직후)
    ├── 규칙 sync (저장 직후)
    ├── API Key 등록
    ├── 상태 조회
    ├── 로그 조회
    └── WS: 실시간 시세 + 체결
```

> 컨텍스트, 하트비트, 템플릿은 로컬 서버가 직접 클라우드 서버와 통신.

### 5.2 JWT 전달 + 규칙 sync

```typescript
// 로그인 성공 시 — JWT를 localhost에 전달
async function onLoginSuccess(jwt: string, refreshToken: string) {
  await localClient.setAuthToken(jwt, refreshToken);
  // 이후 로컬 서버가 자립적으로 클라우드 서버와 통신
}

// 규칙 변경 시 — 저장 후 localhost에 sync
async function onRuleSave(rule: Rule) {
  // 1. 클라우드 서버에 저장
  await apiClient.saveRule(rule);

  // 2. localhost에 즉시 sync
  await localClient.syncRules(await apiClient.getRules());
  // (브라우저 닫혀도 WS push로 로컬 서버가 자동 re-fetch)
}
```

### 5.3 신호등 구현

```typescript
type TrafficLightColor = "green" | "yellow" | "red";

// 단일 신호등: "지금 거래 가능한가?"
// 🟢 로컬 + 키움 정상 (거래 가능)
// 🟡 로컬만 연결 (키움 안 됨 — 툴팁으로 원인 안내)
// 🔴 로컬 미연결 (규칙 관리만 가능)

function getTrafficLight(localStatus: LocalStatus | null): TrafficLightColor {
  if (!localStatus) return "red";           // 로컬 서버 미연결
  if (!localStatus.kiwoom_connected) return "yellow";  // 키움 안 됨
  return "green";                           // 전부 정상
}

// 5초마다 갱신
useInterval(() => {
  localClient.getStatus()
    .then(s => setTrafficLight(getTrafficLight(s)))
    .catch(() => setTrafficLight("red"));
}, 5000);
```

### 5.4 오프라인 대응

클라우드 서버 미연결 시 프론트엔드 동작:
- 종목 검색: 로컬 서버의 stock_master.json 캐시에서 검색 (fallback)
- 규칙 생성/수정: 로컬 서버에 저장 → 복구 후 sync
- 관심종목: 로컬 서버의 watchlist.json 캐시 사용
- 클라우드 장애 감지 시 에러 배너:
  "서버 점검 중 — 자동 매매는 정상 작동합니다. 규칙 변경은 복구 후 동기화됩니다."

- [ ] 매수/매도 조건을 UI로 구성 가능
- [ ] AND/OR 연산자 선택 가능

- [ ] 로그인 후 → JWT + Refresh Token을 localhost에 전달
