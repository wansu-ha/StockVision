## 6. UI 법적 요소 분포 점검

### 6.1 현재 구현 상태

| 전략 활성화 | 면책 고지 팝업 | ❌ 없음 | 전략 활성화 시 별도 팝업 없음 |

### 6.2 법적으로 필요한 UI 요소 (한국법 기준)

#### 필수 (P0)
1. **회원가입 시 이용약관 + 개인정보처리방침 개별 동의 체크박스**
   - 개인정보보호법 제15조: 개인정보 수집 전 **별도 동의** 필수
   - 현재: Register.tsx에 없음 → 법적 흠결
   - 위치: `frontend/src/pages/Register.tsx`

2. **투자 위험고지 동의 (전략 최초 활성화 시)**
   - 자본시장법 시행령: 시스템매매 이용 시 위험고지 수령 확인
   - 현재: 온보딩에만 있음, 전략 활성화 시 없음
   - 위치: 전략 활성화 버튼 클릭 시 모달

3. **개인정보처리방침 상시 열람 가능**
   - 개인정보보호법 제30조: 처리방침 공개 의무
   - 현재: 어디서도 열람 불가
   - 위치: 설정 페이지 또는 Footer 링크

#### 권장 (P1)
4. **Footer에 법적 링크**
   - 이용약관, 개인정보처리방침, 투자 위험고지 링크
   - 일반적인 웹서비스 관례

5. **약관 변경 시 재동의 메커니즘**
   - 로그인 시 최신 약관 버전 체크 → 변경 시 재동의 팝업
   - plan.md Step 6에 정의됨 (미구현)

6. **계정 삭제 시 데이터 삭제 안내**
   - 개인정보처리방침 §6과 연동
   - "로컬 데이터는 직접 삭제" 안내 팝업

#### 고려 (P2)
7. **전략 빌더 하단 상시 면책 문구**
   - "과거 성과는 미래 수익을 보장하지 않습니다" 한 줄 표시
   - 백테스팅 결과 화면에도 동일 문구

8. **백테스팅 결과 페이지 면책 문구**
   - "이 결과는 과거 데이터 기반이며, 실제 거래 성과를 보장하지 않습니다"

## 10. 점검 결론

### 잘 된 부분
- 4개 법무 문서의 내용 충실도가 높음 (총 ~62KB)
- 아키텍처와 법무 문서 간 핵심 원칙(로컬/클라우드 분리, Credential Manager)이 일관됨
- 키움 약관 분석 리서치가 매우 상세함 (kiwoom-legal-analysis.md)
- 온보딩 RiskDisclosure 컴포넌트가 이미 구현됨

### 개선 필요
1. **KIS 반영** — 코드에는 있으나 법무 문서에 없음 (가장 큰 갭)
2. **진행도 동기화** — spec/plan 체크박스가 실제와 불일치
3. **UI 법적 요소** — 가입 시 약관 동의 체크박스 없음 (법적 흠결)
4. **형식 표준화** — "법률 검토 전 초안" 헤더, 효력 발생일 조건부 변경
5. **법명 오류** — "증권거래법" → "자본시장법" 수정
