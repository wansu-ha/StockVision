## 3. 이용약관

### 3.1 핵심 조항

**서비스 성격:**
- StockVision은 **시스템매매 도구**이다 (투자자문/투자일임 아님)
- 사용자가 직접 규칙을 정의하고, 사용자의 PC에서 자동 실행된다
- StockVision은 매매 판단/추천을 하지 않으며, AI/LLM은 데이터 제공자 역할만 한다

**투자 손실 면책:**
- 사용자가 정의한 규칙에 의한 매매 결과는 **사용자 책임**
- 시스템 오류, 네트워크 장애, 키움 API 장애로 인한 손실에 대해 제한적 면책
- "과거 성과는 미래 수익을 보장하지 않습니다" 명시

**서비스 중단:**
- 클라우드 장애 시 로컬 서버는 캐시 기반 독립 동작 (규칙 지속 실행)
- 예정된 점검은 사전 공지
- 불가항력 사유로 인한 중단에 대해 면책

**커뮤니티 데이터 활용 (v2 추가):**
- 사용자가 **자발적으로 공유**한 전략/수익 데이터는 서비스 개선, 통계, AI 학습에 활용 가능
- 공유하지 않은 데이터는 수집/활용 대상 아님
- 공유 철회 시 합리적 기간 내 삭제

**증권사 API 사용:**
- 사용자는 증권사(키움증권/한국투자증권) 계좌를 직접 개설하고 API를 신청한다
- StockVision은 사용자의 API Key를 서버에 저장하지 않는다
- API Key는 사용자 PC의 Windows Credential Manager에만 저장된다

## 4. 개인정보처리방침

### 4.1 수집 항목

| 항목 | 수집 여부 | 보관 위치 | 법적 근거 |
|------|----------|---------|----------|
| 이메일 | ✅ 수집 | 클라우드 DB | 서비스 제공 (회원 식별) |
| 닉네임 | ✅ 수집 (선택) | 클라우드 DB | 커뮤니티 기능 |
| 비밀번호 | ✅ 수집 (해시) | 클라우드 DB | Argon2id 단방향 해싱 |
| 전략 규칙 | ✅ 수집 | 클라우드 DB | 서비스 제공 (동기화) |

### 4.2 수집하지 않는 항목

| 항목 | 미수집 사유 |
|------|-----------|
| 키움 API Key/Secret | 사용자 PC에만 저장 (구조적 미수집) |
| 체결 내역 | 로컬 SQLite에만 저장 |
| 잔고, 수익률 | 로컬 SQLite에만 저장 |
| 거래 종목, 거래량 | 로컬 SQLite에만 저장 |
| 계좌번호 | 사용자 PC에만 저장 |

### 4.3 하트비트 데이터

```
수집: 익명 UUID + 앱 버전 + OS + 키움 연결 상태 + 타임스탬프
→ 개인정보보호법상 개인정보 해당 없음 (식별 불가)
→ 서비스 운영 통계 목적
```

### 4.4 법적 준수 사항

| 조항 | 요건 | 우리 구현 |
|------|------|---------|
| 개인정보보호법 제29조 | 안전조치의무 | HTTPS, Argon2id (클라우드) |
| 안전성확보조치기준 §8 | 비밀번호 일방향 암호화 | Argon2id ✅ |
| 안전성확보조치기준 §9 | 접속기록 6개월 보관 | 서버 액세스 로그 보관 |
| 안전성확보조치기준 §10 | 민감정보 암호화 저장 | 클라우드 DB: AES-256-GCM ✅ |

**로컬 데이터 보호:**
- 체결 로그(logs.db), 전략 캐시(strategies.json)는 **사용자 PC에만 저장**되며 클라우드 미전송
- 이들은 §4.2 "수집하지 않는 항목"에 해당 → 개인정보보호법 적용 대상 아님
- 보호 수단: OS 레벨 (NTFS 파일 권한, BitLocker 디스크 암호화)
- API Key/Secret: Windows Credential Manager (OS DPAPI 암호화)

### 4.5 데이터 삭제

- 회원 탈퇴 시 30일 유예 후 완전 삭제
- 로컬 데이터는 사용자가 직접 삭제 (우리 서버에 없음)
- .exe 삭제 시 `%APPDATA%\StockVision\` 삭제 안내

## 5. 키움 약관 준수 확인서

### 5.1 준수 사항

| 키움 약관 | 조항 | 우리 대응 |
|---------|------|---------|
| 제3자 위임 금지 | 제5조② | API Key는 사용자 PC에만 저장. 서버 미보관. |
| 시세 중계 금지 | 제5조③ | 서비스 키 시세는 내부 분석용. 유저 재배포 안 함. |
| ID/PW 보관 금지 | — | REST API 방식 → HTS 불필요. ID/PW 미수집. |
| IP 화이트리스트 | — | 유저 키: 유저 PC IP. 서비스 키: 클라우드 IP. |

### 5.2 키 분리 원칙

```
서비스 키 (우리 소유)
├── 위치: 클라우드 서버
├── 용도: 시세 수집 (내부 분석)
└── 유저 재배포 안 함

유저 키 (각 사용자 소유)
├── 위치: 사용자 PC (Credential Manager)
├── 용도: 시세 수신 + 주문 실행
└── 서버에 전송 안 함
```

## 6. 투자 면책 고지

### 6.1 문구

```
StockVision은 시스템매매 자동화 도구입니다.
투자자문 또는 투자일임 서비스가 아닙니다.

- 모든 매매 규칙은 사용자가 직접 정의합니다.
- AI/LLM이 제공하는 정보는 참고용 데이터이며, 투자 추천이 아닙니다.
- 과거 성과는 미래 수익을 보장하지 않습니다.
- 시스템 오류, 네트워크 장애로 인한 손실이 발생할 수 있습니다.
- 투자에 따른 모든 손익은 사용자 본인에게 귀속됩니다.

투자는 원금 손실의 위험이 있으며, 그 책임은 투자자 본인에게 있습니다.
```

### 6.2 표시 위치

- 회원가입 시 동의 체크박스
- 전략 빌더에서 첫 규칙 활성화 시 확인 팝업
- 설정 페이지 하단 고정 표시
- 앱 최초 실행 시 온보딩 고지
