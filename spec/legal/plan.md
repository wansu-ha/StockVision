## 1. 구현 단계

### Step 1 — 이용약관 작성

**목표**: StockVision의 법적 포지션을 명확히 하는 이용약관 문서 작성

**파일**: `docs/legal/terms-of-service.md` (신규)

**구현 내용:**

1. **서비스 성격**
   - StockVision은 시스템매매 자동화 도구임을 명시
   - 투자자문, 투자일임 서비스가 아님
   - 사용자가 매매 규칙을 직접 정의하고 자동 실행하는 방식

2. **투자 손실 면책**
   - 사용자가 정의한 규칙에 의한 매매 결과는 사용자 책임
   - AI/LLM은 데이터 제공자 역할만 함 (매수/매도 추천 아님)
   - 과거 성과는 미래 수익을 보장하지 않음

3. **시스템 오류 및 장애**
   - 시스템 오류, 네트워크 장애로 인한 손실에 대해 제한적 면책
   - 클라우드 장애 시 로컬 서버 캐시 기반 독립 동작 설명
   - 예정된 점검은 사전 공지, 불가항력 사유로 인한 중단에 대해 면책

4. **키움 API 사용 약관**
   - 사용자는 키움증권 계좌를 직접 개설하고 API 신청
   - StockVision은 사용자 API Key를 서버에 저장하지 않음
   - API Key는 사용자 PC의 Windows Credential Manager에만 저장
   - 키움증권 OpenAPI 이용약관 준수

5. **커뮤니티 데이터 활용** (v2 추가 예정)
   - 사용자가 자발적으로 공유한 전략/수익 데이터는 서비스 개선, 통계, AI 학습에 활용 가능
   - 공유하지 않은 데이터는 수집/활용 대상 아님
   - 공유 철회 시 합리적 기간 내 삭제

6. **약관 버전 관리**
   - 약관 변경 시 사용자에게 재동의 요청
   - 변경 이력 및 효력 발생일 명시

**검증:**
- [ ] 문서 한국어로 작성 완료
- [ ] 4개 섹션(서비스 성격, 면책, 장애, 키움 API, 커뮤니티) 포함 확인
- [ ] 법적 용어 검수 (변호사 검수 전 셀프 체크)
- [ ] 아키텍처 문서와 일관성 확인

### Step 2 — 개인정보처리방침 작성

**목표**: POPIA(개인정보보호법) 준수하는 개인정보처리방침 작성

**파일**: `docs/legal/privacy-policy.md` (신규)

**구현 내용:**

1. **수집 항목 명시**
   - 수집 항목: 이메일, 닉네임(선택), 비밀번호(해시), 전략 규칙
   - 각 항목의 보관 위치(클라우드 DB) 명시
   - 수집 법적 근거 명시 (서비스 제공, 커뮤니티 기능 등)

2. **수집하지 않는 항목 명시**
   - API Key/Secret: 사용자 PC 저장, 서버 미수집
   - 체결 내역, 잔고, 거래량: 로컬 SQLite만 저장
   - 계좌번호: 사용자 PC만 저장
   - **구조적 미수집** 원칙 강조

3. **하트비트 데이터 설명**
   - 익명 UUID + 앱 버전 + OS + 키움 연결 상태 + 타임스탬프만 수집
   - 개인정보보호법상 개인정보 해당 없음 (식별 불가)
   - 서비스 운영 통계 목적만 사용

4. **보안 조치**
   - HTTPS 전송 (클라우드 API)
   - Argon2id 비밀번호 해싱
   - 접속 기록 6개월 보관 (안전성확보조치기준 §9)
   - 민감정보 AES-256-GCM 암호화 저장
   - 로컬 데이터: NTFS 파일 권한, BitLocker 디스크 암호화, Windows Credential Manager (DPAPI)

5. **데이터 삭제 정책**
   - 회원 탈퇴 시 30일 유예 후 완전 삭제
   - 로컬 데이터는 사용자가 직접 삭제
   - .exe 삭제 시 `%APPDATA%\StockVision\` 삭제 안내 제공

6. **법적 준수 사항**
   - 개인정보보호법 제29조 안전조치의무 준수
   - 안전성확보조치기준 §8(비밀번호), §9(접속기록), §10(민감정보) 준수
   - 제3자 제공 없음 (커뮤니티 공유 데이터는 사용자 자발적 선택)

**검증:**
- [ ] 문서 한국어로 작성 완료
- [ ] 수집/미수집 항목 명확히 구분
- [ ] 보안 조치 섹션에 기술 스택 명시 (Argon2id, AES-256-GCM 등)
- [ ] 개인정보보호법 조항 번호 정확성 확인

### Step 4 — 키움 약관 준수 확인서 작성

**목표**: 키움 OpenAPI 이용약관 준수 사항을 내부 문서로 정리

**파일**: `docs/legal/kiwoom-compliance.md` (신규, 내부 문서)

**구현 내용:**

1. **제3자 위임 금지 준수 (제5조②)**
   - API Key 사용자 PC 저장 (서버 미보관)
   - API Key는 사용자가 직접 관리
   - 서버가 대신 주문하지 않음 (API 호출은 로컬 서버만)
   - 법적 책임: 사용자 → 키움 (StockVision 중개 아님)

2. **시세 중계 금지 준수 (제5조③)**
   - 서비스 키: 클라우드 서버에만 저장 (내부 분석)
   - 유저 키: 로컬 PC에만 저장 (사용자가 직접 수신)
   - 시세 데이터를 외부로 재배포하지 않음
   - 주문 전 로컬에서 키움 직접 재확인 (가공 의혹 방지)

3. **키 분리 원칙**
   ```
   서비스 키 (우리 소유)
   ├── 위치: 클라우드 서버 (environment variable)
   ├── 용도: 시세 수집 (내부 분석만)
   └── 유저 재배포 금지

   유저 키 (각 사용자 소유)
   ├── 위치: 사용자 PC (Windows Credential Manager)
   ├── 용도: 시세 수신 + 주문 실행
   └── 서버 전송 금지
   ```

4. **IP 화이트리스트 설정**
   - 유저 키: 사용자 PC IP만 허용
   - 서비스 키: 클라우드 서버 IP만 허용

5. **준수 확인 체크리스트**
   - [ ] API Key 서버 저장 안 함
   - [ ] 시세 내부 분석용만 사용
   - [ ] ID/PW 수집 안 함 (REST API 방식)
   - [ ] 키움 API 버전, 엔드포인트 문서화

**검증:**
- [ ] 키움 OpenAPI 약관 최신 버전 확인
- [ ] 아키텍처 문서와 일관성 확인 (로컬 vs 클라우드 역할)
- [ ] 구현 단계 전에 법무팀 검수 권장

### Step 6 — 백엔드 약관 버전 관리

**목표**: 약관 버전 관리 및 사용자 동의 기록 저장 기능 구현

**파일**: `backend/app/models/legal.py` (신규)

**구현 내용:**

1. **약관 버전 모델**
   ```python
   class LegalDocument(SQLAlchemy Model):
       id: int (PK)
       document_type: str  # 'terms', 'privacy', 'disclaimer'
       version: str  # '1.0', '1.1', ...
       content: str  # markdown
       effective_date: datetime
       created_at: datetime
   ```

2. **사용자 동의 기록 모델**
   ```python
   class UserLegalConsent(SQLAlchemy Model):
       id: int (PK)
       user_id: int (FK)
       document_id: int (FK)
       version: str
       agreed_at: datetime
       ip_address: str
   ```

3. **API 엔드포인트**
   - `GET /api/v1/legal/{document_type}/{version}` — 약관 조회
   - `POST /api/v1/legal/consent` — 동의 기록 저장
   - `GET /api/v1/legal/consent/status` — 사용자 동의 상태 조회

4. **로직**
   - 약관 변경 시 새 버전 생성
   - 사용자가 구 버전으로 가입했을 경우, 신버전 재동의 요청
   - 동의 기록에 IP, 타임스탬프 저장 (감시 추적)

**파일 변경:**
- `backend/app/api/auth.py` — 회원가입 시 동의 기록 저장
- `backend/app/services/user_service.py` — 동의 상태 확인 로직

**검증:**
- [ ] 약관 생성 후 조회 가능 확인
- [ ] 사용자 동의 기록 저장 확인
- [ ] 신버전 약관 → 기존 사용자 재동의 요청 로직 동작 확인
