# Phase B 백로그 — 추후 논의 사항

Phase A 완료 후 검토할 기능/설계 주제 목록.

## 원격 접속 (클라우드 릴레이)

- 다른 PC/모바일에서 로그인 → 내 집 로컬 서버 제어
- 설계 방향: heartbeat 연결을 활용한 클라우드 릴레이
  - 읽기: 로컬 서버 → 클라우드에 상태 동기화 (heartbeat에 포함)
  - 쓰기: 클라우드 → 로컬 서버 커맨드 채널 (WS or long-poll)
- 프론트 연결 흐름:
  ```
  로그인 → 클라우드에 "내 로컬 서버 heartbeat 있어?" 확인
    ├─ 있음 → 클라우드 릴레이로 연결
    └─ 없음 → 딥링크로 로컬 서버 실행 요청 (같은 PC일 때)
  ```
- P2P (WebRTC) vs 릴레이: JSON 수 KB 수준이라 릴레이 부하 무시 가능. P2P는 NAT 문제로 과도한 엔지니어링. 트래픽 증가 시 재검토
- **보안 필수 조건**: heartbeat 유효 시간 내에서만 커맨드 전달. heartbeat 만료 상태에서 API 요청 오면 거부 + "로컬 서버 오프라인" 응답 (비정상 접근 차단)
- **API 분류 (heartbeat 필요 여부)**:
  - 프론트 전용 (heartbeat 불필요): 로그인, 규칙 CRUD, 종목 검색, AI 분석 — 로컬 서버 상태와 무관
  - 로컬 서버 전용: heartbeat 전송, 체결 로그 업로드 — 자기 자신이 heartbeat
  - 릴레이 (heartbeat 필수): 엔진 시작/중지, 포지션 조회, 주문 — 로컬 서버 경유 명령, heartbeat 없으면 거부
- 프론트 API 추상화: 로컬/원격 투명 전환 (같은 API, 경로만 다름)

## 인스톨러

- exe 패키징 완료 후, MSI 또는 NSIS로 감싸기
- 포함: exe 설치 + 딥링크 등록 + 시작 메뉴 + 바탕화면 바로가기 + 언인스톨러
- 약관 동의 UI (설치 시 표시, 레지스트리 변경 고지, 면책 조항)
- 라이선스 고지 (번들 OSS 의존성 NOTICE 파일)
- 크래시 자동 복구: Windows Task Scheduler에 "프로세스 종료 시 재시작" 트리거 등록 (인스톨러에서 설정)
- 자동 업데이트 (Squirrel, electron-updater 참고)
- 코드 서명: 오픈소스 공개 후 SignPath.io 무료 서명 + 주요 백신사 false positive 신고

## 모바일 대응

- 원격 접속과 원리 동일 (클라우드 릴레이)
- 프론트엔드 반응형 UI
- PWA or 네이티브 앱은 별도 판단

## SyncQueue 데드코드

- 오프라인 큐 관련 코드 정리 (B part 제거 후 잔여)
- 비긴급, 별도 이슈로 추적

## 프론트엔드 데이터 소스 전략

- 현재: 프론트 → 로컬 서버 (실시간), 프론트 → 클라우드 (규칙/인증/AI)
- 원격 시: 로컬 서버 데이터를 어디서 가져올지 결정 필요
  - 옵션 A: 클라우드 릴레이 (프록시)
  - 옵션 B: 로컬 → 클라우드 동기화 (읽기), 릴레이 (쓰기)
